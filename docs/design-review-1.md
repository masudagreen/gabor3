## 1. 総合判定

**結果：NG（軽微な不足あり）**

カバー率は約 **94%**（仕様の受け入れ基準 60 件中 56 件をカバー、4 件が部分的／未カバー）。重大な構造欠落は無いが、Sprint 6（バッジ）と Sprint 5（結果サマリ）に **観察可能な挙動レベルで仕様文言と表現が一致していない／対応画面の指示が不在** の項目が残っており、これらは Generator が実装時に解釈ぶれを起こしうるため、Round 2 で Designer に補完依頼することを推奨する。

カバー率の高さと整合性は良好だが、下表の 4 項目を埋めると合格レベルに到達する。

## 3. 不足項目（Round 2 で Designer に補完依頼）

### 3.1【Sprint 5】F-11「スキップは『未挑戦』として記録され、サマリには表示されない」の表現が無い

- **仕様の文言（spec.md L195）**：
  > [ ] スキップ操作は「未挑戦」として記録され、サマリには表示されない
- **Designer の現状**：S5-01 結果サマリ完成版にはスキップ操作の場合の表示分岐（「未挑戦」状態でサマリを出さない、あるいは『未挑戦』ラベルを出す）が screens.md に書かれていない。
- **影響**：Generator は「スキップ時にサマリを出すか／出さないか」「未挑戦記録の可視化方法」を独自判断することになる。
- **補完依頼内容**：S5-01 の「未挑戦時の状態」セクションを追加し、観察可能な挙動を明示する。例：
  - ゲーム開始後に何もタップせず時間切れ／全試行未回答 → サマリ画面では「未挑戦」と表示するのか／結果サマリ自体をスキップしてホームへ戻すのか、を 1 つに決めて screens.md に書く。

### 3.2【Sprint 6】F-14「未獲得バッジには獲得条件のヒント」のうち、B-08（全方位改善）のヒント方針が観察可能でない

- **仕様の文言（spec.md L225）**：
  > [ ] 未獲得バッジには獲得条件のヒントが表示される
- **Designer の現状**：S6-03 で「全方位改善（B-08）：『全 3 ゲームの前週比を見ています』（数値ヒントなし、抽象度を保つ）」と記載されているが、これだけだとユーザーは「あと何が起きれば取れるか」が観察不能。他の B-02〜B-07 は「あと N 日」「あと N 点」「累計 87/100」と進捗を可視化しているのに対し、B-08 だけ非対称な扱い。
- **影響**：仕様の「獲得条件のヒント」要件は満たすが、「観察可能性」が他バッジと比べて著しく低く、ユーザー体感が一貫しない。
- **補完依頼内容**：B-08 のヒントを「3 ゲームのうち N ゲームが先週比で改善中」のように、ゲーム単位の進捗状況を可視化する文言に変更するか、あるいは「現状ヒントなしで OK」と明示的に意図を screens.md に記述する（仕様の意図と整合的か Designer の判断）。

### 3.3【Sprint 2】Game 1（F-07）の「タップ済みパッチは選択状態が視覚的に分かる（色／枠）」は OK だが、「未挑戦のままタイムアップも記録される（ただし staircase は up=易方向）」の表現が screens.md 内で観察可能になっていない

- **仕様の文言（spec.md L307）**：
  > [ ] 「未挑戦のままタイムアップ」も記録される（ただし staircase は up=易方向）
- **Designer の現状**：S2-03 Game 1 プレイ画面の状態遷移（時系列）には「60 秒経過 → 完了ボタン強調（Sprint 内では時間切れで自動採点に進める）」と書かれているが、「タップ無しで時間切れ」のときに何が画面に出るか（採点画面に進むのか／「未挑戦」表示か／staircase 上昇の旨をユーザーに伝えるか）が未記述。
- **影響**：Generator は「ユーザーが何もタップせずに 60 秒経過した場合の画面遷移と表示」を独自判断する必要が生じる。
- **補完依頼内容**：S2-03 / S5-01 のいずれかに「未挑戦時の動作（自動採点 → 結果サマリで未挑戦記録の表示有無）」を 1〜2 行で明記する。

### 3.4【Sprint 5】F-05/F-11 の自動進行 10 秒タイマー：単体プレイモード時の挙動

- **仕様の文言（spec.md L148, L194）**：F-05「ゲーム間は 10 秒以内の結果サマリで自動進行」／F-11「サマリ画面は『次へ』を押すまで自動で進まない（OPT-7）」
- **Designer の現状**：S5-01 §2 で「単体プレイモードでは自動進行しない」と一文だけ言及されているが、screens.md の S5-01 メインのレイアウト図には「コースモード時のみ：あと 8 秒で次のゲームへ」が書かれているのに対し、**単体プレイモード時の S5-01 の見た目（カウントダウン非表示、「次へ」のみ）が画面例として示されていない**。
- **影響**：Generator が「単体プレイ時のレイアウト」をコースモード版から自分で派生させる必要があり、解釈ぶれの余地が残る。
- **補完依頼内容**：S5-01 に「単体プレイモード時の表示（カウントダウン無し、『次へ』ボタンのみで自動進行しない）」のレイアウトを 1 ブロック追加するか、または「コースモードとの差分」を明示的に箇条書きする。

## 5. 全体所感

**Designer の良かった点**：

- 仕様 §1 の OPT-1〜10 を system.md §0 の冒頭で「実装上の規範」として組み込み、トークン値（`font.body.min`、`tappable.min`、`DistanceCalibrator` ノッチ強制等）に**構造的に違反不能な形で**展開した点は秀逸。Generator が個別判断で外す余地が小さい。
- コントラスト比検証表（§1.4）を 8 ペア計算済みで明示しており、ダークモード時の 7:1 維持の根拠が揃っている。
- スプリント別 screens.md に **「受け入れ基準カバレッジ表」** を冒頭に置き、仕様 ID と画面 ID をクロスリファレンスしている運用が、レビュー作業を大幅に効率化した。
- F-05（10 秒自動進行）と F-11（自動で進まない）の仕様矛盾を Sprint 5 §2 で「OPT-7 の解釈」として明示的に整理し、Generator の解釈ぶれを未然に防いでいる点は良い設計判断。
- Game 3 のレスポンシブ問題（時計 8 ボタン + ガボール領域 + ヘッダーで縦が詰まる）を S3-09 / S7-10 で予測し、縮小規範を先回りで定義している。

**改善余地（補完依頼の背景）**：

- 「未挑戦／スキップ」「自動進行モード差分」「B-08 のヒント非対称」など、**例外パスや非対称な挙動の明示化**で軽い不足が目立つ。本流の挙動は揃っているが、観察可能性の網羅という観点で 4 件残った。これは Designer のクリエイティブ余地を超えた「仕様準拠の明示」レベルの修正で、難しくない範囲。

総じて、Round 2 で 4 項目を補完すれば人間レビュー（デザイン凍結）に進める品質。
