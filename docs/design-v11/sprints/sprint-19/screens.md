> **Sprint 22 v1.2 改訂注記（2026-05-10、最重要）**：本スプリントは **v1.2 で大規模改訂**された。
> - **F-18 `releaseEnabled` 機構は完全廃止**。7 ゲームをハードコード。

> - **設定画面**：F-19 音 / 振動の独立トグルを追加（`SoundHapticsToggle`、components.md §30）。staircase リセット項目は維持。

> - **NF-27 / NF-28 ガボールクリッピング**：実サイズ 1.5 倍生成 + 矩形クリッピング（system.md §1.12）。

> - 最新仕様は `docs/design-v11/sprints/sprint-22/screens.md` §11 設定 / §12 バッジ を参照。
>
> **Sprint 20 改訂注記（v1.1.1、2026-04-30）**：本スプリントの **a11y 監査・レスポンシブ確認は Sprint 20 改訂後に再実施が必要**。Sprint 20 で導入された ResultOverlay（◯/✕ 重畳・メトリクスバー・選択枠改訂）が a11y / レスポンシブ規範を満たすか、Sprint 20 完了後に Evaluator が再監査する。バッジ獲得演出（F-13）は ResultOverlay 上で 1 度だけ流れる仕様に変わるため、`AchievementBadgeBurst` の表示位置を ResultOverlay 上層レイヤに対応させる必要がある（`sprint-20/screens.md` §2 / components.md §23 参照）。

## スプリントの目的（spec-v11.md §13）

13 種バッジが定義通り獲得できる。設定全項目動作。`releaseEnabled` を false にしたゲームがホーム・コース・グラフ・バッジから動的除外される。最小 360px〜最大 1280px でレイアウト崩れなし。a11y 監査をパス。

含む機能：F-13、F-14、F-15、F-18、a11y 全件再監査、レスポンシブ確認

### 8.3 フェーズタイミング表

| 時刻（秒） | 表示 / 動作 |
|---|---|
| 0.0s（結果サマリマウント） | 通常の結果サマリ要素（タイトル / 正解 / 回答 / MetricCard ×2）が順次フェードイン（既存タイミング） |
| 2.0s | バッジ獲得演出開始：scale(0.6, 0.6) で出現（不透明度 0 → 1）、`aria-live="assertive"` で読み上げ開始 |
| 2.0s〜2.4s | scale(0.6 → 1.05) overshoot 5%、easing `cubic-bezier(0.34, 1.56, 0.64, 1)` |
| 2.4s〜3.5s | scale(1.05 → 1.0) settle、計 1.5 秒で完了 |
| 3.5s | 演出完了、静止表示維持 |
| 任意 / コース時 10s | 「次へ」タップまたは自動進行 |

### 8.4 複数バッジ同時獲得時の挙動

1 セッションで複数バッジを同時獲得した場合（例：B-01 はじめの一歩 + B-09 探検家 が同時条件達成）、**1.5 秒ずつ順次表示**：

| 時刻 | 表示 |
|---|---|
| 2.0s | バッジ 1 獲得演出開始 |
| 3.5s | バッジ 1 静止 + 直後にバッジ 2 へ切替 |
| 3.5s〜5.0s | バッジ 2 演出（同じ scale-up） |
| 5.0s | バッジ 2 静止表示 |
| 任意 | 「次へ」タップ |

複数獲得時の SR：1 つずつ順に「B-01 はじめの一歩 を獲得しました」「B-09 探検家 を獲得しました」とアナウンス。

> コース時自動進行（10 秒）との整合：複数バッジ表示で 5 秒消費しても 10 秒以内に収まるため `countdownSeconds` の調整は不要。3 個以上の同時獲得（最大 13 種だが現実的には 1〜2 個）は仕様上発生しにくいが、Generator は表示が 10 秒を超える場合「3 個目以降は静止表示のみで通り過ぎる」フォールバックを設けてよい。

### 8.5 演出仕様（既存）

- 1.5 秒の scale-up（overshoot 5% 以下）
- **フラッシュ・点滅なし**（NF-11）
- バッジアイコン Bold + 色強調（装飾文脈なので brand.accent 6.40:1 許容、ただし大型化で識別性確保）
- 背景：通常の結果サマリの背景（`color.bg.surface`）に `palette.semantic.success` の薄塗（10% alpha）+ outlined border `palette.semantic.success` 7:1 適合色
- アイコン 80×80px、テキスト font.h2 30px Bold

### 8.6 prefers-reduced-motion: reduce 時

scale アニメーションは即時化（0ms で 1.0 表示）、`aria-live` は通常通り動作。バッジカード自体は表示されるが演出フェーズタイミングは 2.0s = 即表示 → 3.5s 同タイミングで完了。

### 8.7 a11y

- バッジカードは `role="status"`, `aria-live="assertive"`, `aria-atomic="true"`
- SR：「{バッジ名} を獲得しました」（1 度だけ）
