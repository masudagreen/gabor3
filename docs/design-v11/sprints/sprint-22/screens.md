## 3. S22-G01-PLAY：G-01 ゆっくり回転検出（コンセプト刷新）

> **コンセプト**：3×3 グリッドの中で、ゆっくり回転しているパッチが複数（ランダム個数）混じっている。それらを 60 秒間注視しながらすべてタップで選ぶ。

### 3.1 仕様（spec §7.1、A2 確定）

- **グリッドサイズ**：3×3 固定
- **回転アイテム個数**：1〜3 個ランダム（**Designer 提案分布：1 個 40% / 2 個 40% / 3 個 20%**、system.md §1.17.5）
- **回転方向**：時計回り / 反時計回り の両方向ランダム（アイテムごとに独立）
- **回転速度**：staircase 対象（**Designer 提案値：易 6°/s → 難 1°/s、初期 3°/s、step 0.5°/s**、system.md §1.17.4）
- **静止アイテムの角度**：それぞれ異なるランダム角（角度自由化、最小可視差は system.md §1.13 表）。**互いに最低 10° 離す**（Designer 提案、system.md §1.17.3、NF-28b 整合）
- **選択肢**：3×3 の各パッチ 9 個（ImageChoiceCell）。複数選択
- **メッセージ**：「**回転しているパッチをタップ**」（方向情報・個数情報なし、A2 確定）
- **採点**：TP+1 / FP-1 / FN 0 の部分点。staircase 駆動の正誤判定は「TP - FP > 0 なら正解」

### 3.2 スマホ縦（375×667）レイアウト

```

│ [ステータスバー領域：背景色 #808080] │ ← top inset 背景延長
├─────────────────────────────────────┤
│  ✕     残り 47 秒                    │ ← GameStatusBarV12 (64px + topInset)
│                                     │   CountdownTimer size=lg、tabular-nums
│                                     │
│  回転しているパッチをタップ           │ ← ガイド文 font.body 24px center
│                                     │
│  ┌────────────────────────────────┐ │
│  │  ▦   ▦   ▦                    │ │ ← MorphGridStimulus (3×3)
│  │                                │ │   全体辺 320×320、隙間 8px
│  │  ▦   ▦   ▦                    │ │   各セル 約 101×101px
│  │                                │ │   ImageChoiceCell × 9
│  │  ▦   ▦   ▦                    │ │
│  └────────────────────────────────┘ │
│                                     │
│                                     │
└─────────────────────────────────────┘
```

### 3.3 PC 横（1280×800）レイアウト

- 中央寄せ最大幅 720px
- グリッド全体辺 480px（隙間 8px、各セル約 153×153px）
- ガイド文は刺激領域上に配置（font.body 24px、center）

### 3.4 状態

| 状態 | 描画 |
|---|---|
| 通常（注視中） | 各セルが ImageChoiceCell。回転アイテムは staircase 値の速度で連続回転、静止アイテムはランダム角で固定 |
| 選択中 | 選択枠 2px `color.selection.subtle`（中性グレー、components.md §4） |
| 試行終了（採点直後） | ResultOverlay 重畳（§13）、回転は停止 |
| 中断確認 | AbortConfirmDialog（v1 既存） |

### 3.5 a11y

- ImageChoiceCell に `role="checkbox"`, `aria-checked` で選択状態
- ガイド文は `aria-describedby` で radiogroup（実際は checkboxgroup）に関連付け
- カウントダウンは `CountdownTimer` 経由（aria-live polite/assertive 切替）
- 各 ImageChoiceCell に `data-target-id="g01-cell-{row}-{col}"`（ResultOverlay の MarkBadge 配置用）
- prefers-reduced-motion: reduce 時：回転をフレームレート 5fps に落とす（停止はしない、回転判定が課題のため）。Designer 注：完全停止すると課題不成立になるため低速描画で対応

### 3.6 受け入れ基準（追加）

- [ ] 静止アイテム同士の角度差は最低 10°（Designer 提案、回転 1 秒分の最大値 6°の 1.67 倍）
- [ ] 回転アイテム個数は試行ごとにランダム抽選（同セッション内では固定）
- [ ] 回転方向はアイテムごとにランダム
- [ ] 「現在の回答：◯◯」テキスト表示なし
- [ ] 試行中の正誤フィードバックなし

## 4. S22-G03-PLAY / S22-G03-RESULT：G-03 周辺視野ハント（中央線方式廃止）

> **コンセプト**：固視点を中心に円周上に並ぶ 8 個のガボールから「異質な向き」のものを見つける。60 秒間ずっと中央を見続ける訓練。

### 4.1 仕様（spec §7.3、A10 確定）

- **構造**：中央固視点（+ アイコン、選択不能）+ 周囲 8 個のガボール（離心角 8° 固定、ImageChoiceCell × 8）
- **選択方式**：単一選択（radio）。タップで切替、別を押すと前の選択は解除
- **staircase パラメータ**：odd one の向き差（°、最小単位は system.md §1.13 表、易：大角度差 → 難：小角度差、Designer 提案初期値 30°、min 5°、max 60°、step 2°）
