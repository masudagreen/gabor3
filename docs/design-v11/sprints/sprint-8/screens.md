# Sprint 8 — v1.1 基盤と起動時データリセット

## スプリントの目的（spec-v11.md §13）

起動時に v1 由来データが消去される。新ホームに「全ゲーム連続プレイ」CTA がある（連続実行は次スプリント）。空のゲームレジストリと改訂版 staircase ロジック（直近 5 セッション平均）が動作する。

### F-18 動的反映
- 「全ゲーム連続プレイ（約 N 分）」の N は `enabledGameCount` から動的計算（13 ゲーム時 13 分、N が 7〜13 で揺れる前提）

- HomeNavGrid のバッジ進捗 `{earned} / 13` は固定で 13 種表示（バッジ自体は disabled に依存しても項目数は変わらない）

### スプリント 8 時点での暫定
- gameRegistry はまだ「ゲーム実装空」のため、HeroCTA タップは「準備中：Sprint 9 以降で実装」を表示する Toast に遷移する仮実装

- 進捗グラフ・バッジは EmptyState 表示で OK（次スプリント以降で実装）

## 9. テスト観点（Generator 向け）

- `dataResetNoticeShown` フラグの永続化
- 初回起動 → 通知表示 → OK 押下 → 2 回目起動で通知非表示

- DistanceReminderV11 の 3 秒タイマー終了で `onCountdownComplete` コールバック発火
