# Sprint 1 — 描画基盤と Game 2 最小骨格

> **2026-04-29 amendment（Game 2 再設計反映）**：Game 2 を「左右並び 5 秒同時提示型 2AFC」に変更。点滅・マスクは廃止し、固視点 → 左右並び 5 秒提示 → 3 秒回答受付 → 1 秒フィードバック → 次試行のループに変更。本書は新挙動に合わせて差分更新済み。

## スプリントの目的（spec.md §13 より）
起動すると左右並びのガボール 2 つが 5 秒間同時に見え、「どちらが時計回りに傾いているか」を 2AFC で回答できる。staircase が動く。1 試行ループが完結する。視聴距離は暫定固定 40cm。

含む機能：F-08（Game 2）／F-10（staircase 最小）／F-03 部分（暫定固定 40cm のみ）／ガボール描画基盤

## 4. S1-03：Game 2 プレイ画面（amendment 後：左右並び 5 秒同時提示型）

固視点 500ms → 左右ガボール 5 秒同時提示 → 回答受付（最大 3 秒）→ フィードバック 1 秒 → 次試行。
**点滅・マスクは廃止**。提示中はガボールがそのまま画面に居続けるため、注視を保ちやすい。

### スマホ縦（375×667）

```
┌─────────────────────────────────────┐
│  ✕    残り 47 秒    1 / 30 試行     │ ← GameStatusBar 64px
│                                     │
│                                     │
│   ┌──────────┐  ┌──────────┐         │
│   │          │  │          │         │
│   │ 縞模様   │  │ 縞模様   │         │ ← 中性グレー領域 #808080
│   │ 左       │  │ 右       │         │   各 ~140×140px
│   │          │  │          │         │   中央ギャップ 24px
│   └──────────┘  └──────────┘         │   5 秒間そのまま静止表示
│                                     │
│                                     │
│  ─────────────────────────────────  │ ← 区切り線
│                                     │
│   どちらが時計回りに傾いていますか？  │ ← font.body（24px）guide
│                                     │
│   ┌──────────────┐  ┌──────────────┐ │
│   │ ← 左         │  │ 右 →         │ │ ← Button secondary lg×2、64pt
│   └──────────────┘  └──────────────┘ │   間隔 16px
│                                     │
└─────────────────────────────────────┘
```

### PC 横
- 中央 720px に同構成
- ガボールサイズは画面幅 / 2 - パディングで決まる（最大 240px、最小 96px）
- 回答ボタンは横並び、各 280×64px、中央 24px ギャップ

### 状態機
| フェーズ | 持続時間 | 表示内容 | 回答ボタン |
|---|---|---|---|
| fixation | 500ms | 中央に固視点のみ | disabled |
| presentation | 5000ms | 左右並びガボール（同時静止表示、点滅なし） | disabled |
| answer | 最大 3000ms | 左右並びガボール継続表示 | enabled |
| feedback | 1000ms | ガボール継続表示（最終試行のみ正解側枠ハイライト） | disabled |

### 回答ボタンの状態
- presentation 中は disabled
- answer フェーズ（5 秒提示完了直後）= 回答可能（enabled）
- 3 秒以内に押下無し → **不正解（タイムアウト）として記録**、staircase は up、無音
- タップで finalizeTrial を呼び正誤判定。正解時のみ短い効果音（C5→E5）+ 軽ハプティクス
- 不正解時は無音
- **試行中の正誤フィードバック表示はしない**（最終試行除く）

### 試行終了時の正解側ハイライト（最終 1 試行のみ）
- 60 秒経過または 30 試行到達時、最後の試行の正解側ボタン枠を `color.highlight.correct` で 1 秒表示
- ハイライト後、結果サマリ S1-04 へ遷移

### セッション全体の制限
- セッション 60 秒経過で強制終了（残り試行スキップ → 結果サマリ）
- 30 試行達成でも自動終了
- 中断 × ボタン → ConfirmDialog
  - 続ける（primary）
  - 中断する（secondary）→ S1-01 へ戻り、未完了記録

### a11y
- 提示中のガボール `accessibilityLabel`：「左の縞模様」「右の縞模様」（2 つの View にそれぞれ）
- 回答ボタン `aria-label`：「左の縞模様が時計回りに傾いている」「右の縞模様が時計回りに傾いている」
- 設問 `aria-live="polite"`：「どちらが時計回りに傾いていますか？」
- 残り時間：6 秒以上は `aria-live="off"`、5 秒以下は `aria-live="polite"`
- Web キーボード操作：← で左、→ で右を選択（answer フェーズ中のみ有効）

### モーション
- ガボール領域はサイズ・位置を維持。フェード切替なし（提示時間が正確に守られるよう）
- prefers-reduced-motion: reduce 環境では feedback 1 秒を 0ms（瞬時遷移）に縮める。提示時間自体は維持

## 8. ガボール表示領域の物理計算（暫定 40cm 固定）

- viewing distance = 40cm 固定
- pixelsPerDegree = `(40 * tan(1°)) / pixelSizeCm`、pixelSizeCm は dpr メタ + 端末タイプ既定 dpi から導出
- Game 2 のパッチ A/B: `cpd=3, sigmaDeg=0.6, contrast=0.3` を初期値（仕様 §6.1 中央値）
- 実装上の検証ポイント：iPhone 12（dpr=3）と PC（dpr=1）で同じ視野角サイズになること

## 9. テスト要件（Sprint 1 完了の最低条件 ≥ 5 件）

1. GaborPatch が cpd / contrast / sigmaDeg を反映してオフスクリーン Canvas を生成する
2. Game2Screen の状態機が `fixation(500ms) → presentation(5000ms) → answer → feedback(1000ms)` を順に通過する（amendment 後）
3. staircase 3-down/1-up：3 連続正解で param が 1 step 下がる、1 誤答で 1 step 上がる
4. staircase reversal カウントが正しくインクリメントされる
5. クランプ：min 1° / max 10° を超えない
6. 試行 3 秒以内未回答 → 不正解（タイムアウト）として記録、staircase up
7. 60 秒経過 / 30 試行到達で自動終了
8. buildTrialSpec の左右ペア生成（amendment 後）：correctSide ランダム、orientation 差が paramValue
