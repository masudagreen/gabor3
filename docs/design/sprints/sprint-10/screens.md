## 2. レスポンシブ全画面チェック（NF-21/22）

検証幅 **360 / 375 / 1280px**（必須）。各画面で：
- 本文 24px・ボタン/行/タブ 56pt（タブは48pt許容）・グラフ軸 18pt が維持される。
- ゲーム格子：`min(short_edge-32, 360)`（スマホ）/ 480px（PC 中央）。n=3/4/5 で破綻しない。
- タブ画面コンテンツ：PC は最大幅 720px 中央寄せ、タブバーは全幅。
- ダイアログ/カード：最大幅 360（スマホ）/ 480（PC）。
- スマホ縦持ち固定、PC/タブレット横（タブレットは PC 同等、NF-22）。
- 360px で 3 タブ各 48pt・主要 CTA・本文が崩れないこと。

## 3. セーフエリア全画面チェック（NF-29/30）

| 画面 | モード | 確認 |
|---|---|---|
| ゲームプレイ中（S4） | game（フルスクリーン許容） | 背景 #808080 はステータスバー/Home Indicator まで広がる。**X・残り秒・タブバーはセーフエリア内** |
| 距離リマインド/ホーム結果/履歴/設定/オンボ/全ダイアログ/データリセット通知 | default | 全コンテンツがセーフエリア内（top/bottom/left/right inset 適用） |
| ボトムタブバー | — | 常に bottom セーフエリア内、下端に inset パディング |

- iOS=SafeAreaInset、Android=StatusBar.currentHeight、Web=`env(safe-area-inset-*)` + viewport-fit=cover。
- `SA-1 SafeAreaWrapper` を非ゲーム画面で必須使用。

## 4. ガボール描画品質最終チェック（NF-26〜28b）
- 角度を最小単位（system §7.3）ずつ 0〜360° 動かし、四隅に背景色露出なし（NF-27/28）。
- クリッピング N=1.5（system §7.1）。回転中も隙間なし。
- 変化/静止の弁別性（NF-28b、system §9.4）：静止初期角度 12° 以上分散、静止は時間変化しない。

## 5. 全体結合の観点
- 起動 → リセット通知 →（初回オンボ）→ 距離リマインド → ゲーム → 結果 → もう一度 の一連が途切れず動く。
- タブ切替（プレイ中=中断ダイアログ / 非進行=自由）が全状態で正しい。
- 設定変更が次セッションに反映（n/m/r/a/b/採点方式/視聴距離/ダーク/音/振動/片眼）。
- 完了セッションのみ集計・バッジ判定・履歴反映（中断は除外）。
