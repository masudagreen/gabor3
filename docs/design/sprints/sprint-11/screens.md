## 2. レスポンシブ全画面チェック（NF-21/22）

検証幅 **360 / 375 / 1280px**（必須）。各画面で：
- 本文 24px・ボタン/行/タブ 56pt（タブは 48pt 許容）・グラフ軸 18pt・**レベル番号 18pt・個数案内 18pt** が維持される。
- ゲーム格子：`min(short_edge-32, 360)`（スマホ）/ 480px（PC）。gridSize 3/4 で破綻しない。
- タブ画面コンテンツ：PC は最大幅 720px 中央寄せ、タブバーは全幅。
- ダイアログ/カード：最大幅 360（スマホ）/ 480（PC）。
- 設定の範囲チップ（RG-1 回転速度 9 個）が 360px で折り返す。変化順（OR-1）が 1 カラム。
- スマホ縦持ち固定、PC/タブレット横（タブレットは PC 同等、NF-22）。
- 360px で 3 タブ各 48pt・主要 CTA・本文・レベル番号が崩れないこと。

## 4. ガボール描画品質最終チェック（NF-26〜28c）
- 角度を最小単位（system §7.4）ずつ 0〜360° 動かし、四隅に背景色露出なし（NF-27/28）。
- クリッピング N=1.5（system §7.1）。回転中・**振動の折り返し中**も隙間なし。
- 回転/静止の弁別性（NF-28b、system §7.2）：静止初期角度 12° 以上分散、静止は時間変化しない。各 rotationSpeed（6〜2）で弁別が成立。
- **振動の視認性（NF-28c）**：振動（往復回転）が一方向回転と区別でき、静止とも区別できる（system §4.1 モーション狙い）。実機で振幅・周期を体感確定。

## 5. 全体結合の観点
- 起動 → リセット通知（旧 v1〜v2 検出時）→（初回オンボ）→ 距離リマインド → ゲーム → 結果開示 → ホーム結果（クリア/失敗・レベル変化・現在レベル）→ もう一度 の一連が途切れず動く。
- レベル昇降が観察できる：クリアで +1（次ゲームのレベル表示変化 + LD-1 告知）、2 連続失敗で −1、クランプ（上限/L1）。
- タブ切替（プレイ中=中断ダイアログ / 非進行=自由）が全状態で正しい。中断はレベル・記録に影響しない。
- 設定変更が次ゲームに反映（範囲・変化順 → 梯子変化 + 現在レベルクランプ + 連続失敗リセット / 視聴距離 / ダーク / 音 / 振動 / 片眼）。
- 完了ゲームのみ集計・バッジ判定・履歴反映（中断は除外）。
- 日次到達レベル/最高到達レベル/連続日数/累計回数が正しく集計・表示。
