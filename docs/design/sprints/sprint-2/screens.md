## 4. S2-03：Game 1 プレイ画面

`GaborGrid` を中心に、3×3〜5×5 のグリッドが緩やかにモーフィング。

> **仕様改訂（強制 60 秒視聴）**：完了ボタンは**廃止**。本アプリの目的（ガボールアイを所定の時間見せる）に合致させるため、60 秒の試行は途中終了不可。残り時間 0 秒まで自由に選択を変更でき、0 秒到達と同時に自動採点へ進む。緊急脱出のみ画面左上の × で可能（中断モーダル → ホーム）。

### スマホ縦（375×667）

```
┌─────────────────────────────────────┐
│  ✕    残り 47 秒                    │ ← GameStatusBar（残り秒 30px Bold tabular-nums）
│                                     │
│  変化したパッチをタップ              │ ← guide text 24pt（先頭 3 秒のみ表示
│  60 秒のあいだ、何度でも選び直せます │   その後フェードアウト、副ガイド 20pt）
│                                     │
│  ┌─────┬─────┬─────┐                │
│  │ ▦  │ ▦  │ ▦  │                │ ← GaborGrid（4×4 例）
│  ├─────┼─────┼─────┤                │   各セル 70×70px（タップ領域 48pt+）
│  │ ▦  │[▦] │ ▦  │                │   選択中セルは黄色 4px 枠、再タップで解除
│  ├─────┼─────┼─────┤                │
│  │ ▦  │ ▦  │[▦] │                │
│  └─────┴─────┴─────┘                │
│                                     │
│  選択中: 2 個                       │ ← font.body 18pt（カウンタ）
│                                     │
│  （完了ボタンは廃止：60 秒経過で    │ ← Sprint 1 までの「完了」ボタン位置は
│   自動採点）                        │   空白に。残り時間カウントダウンが唯一の
│                                     │   進行表示
└─────────────────────────────────────┘
```

### PC 横
- グリッドは中央 480×480px
- 下部に完了ボタンは置かない（レイアウトは中央寄せのみ）

### グリッドサイズ別レイアウト

| difficulty | rows×cols | スマホ各セル | PC 各セル |
|---|---|---|---|
| 易（staircase up） | 3×3 | 100×100px | 140×140px |
| 中 | 4×4 | 76×76px | 108×108px |
| 難 | 5×5 | 60×60px ※タップ領域 48pt 確保 | 88×88px |

### 状態（時系列）
1. ロード時：固視点 500ms（中央に十字、グリッドはまだ表示せず）
2. プレイ開始：60 秒間、グリッド全体表示。1〜3 個のパッチが線形に角度モーフィング
3. プレイ中、ユーザーがセルをタップ → 選択状態切替（多重選択可、再タップで解除可）
4. 残り時間 0 秒到達 → **自動的に採点フェーズへ遷移**（早期終了不可）
5. 採点 → 正解パッチを 1.5 秒拡大ハイライト → S2-04 結果サマリ
6. 緊急中断のみ × で可能（ConfirmDialog → ホーム、staircase 非反映）

### タップ無しのまま 60 秒経過した場合の挙動（仕様 §7.1 対応）

ユーザーが 60 秒間 1 度もパッチをタップしないまま時間切れになった場合（完了ボタンは廃止済み）：

1. 60 秒到達と同時に**自動採点フェーズへ自動遷移**する
2. グリッド上に「変化していたパッチ」を **1.5 秒間拡大ハイライト**で提示する（通常採点後と同じ演出）
   - これにより「どのパッチが変化していたか」をユーザーに必ず見せる
3. ハイライト終了後、自動で S2-04 結果サマリへ遷移する
4. 結果サマリ S2-04 では `TrialRecord.isCorrect = null`（未挑戦）として記録され、表示は次の通り：
   - 中央の閾値表示は通常通り（最大角度差を表示）
   - 「正答 0 個 / 誤タップ 0 個」ではなく **「未挑戦」ラベル**を表示（観察可能な区別）
   - 補足文言：「タップせずに時間切れになりました。次は変化したパッチをタップしてみてください」（font.body 18pt、`color.fg.secondary`）
5. staircase は **up（易方向）に 1step 動く**（spec.md §7.1）。この旨はユーザーには明示しない（難易度の自動調整は裏側挙動）

> **未挑戦の判定基準**：1 試行を通じてタップ数 = 0 のまま 60 秒経過したとき。1 つでもタップしていれば「採点済み（正答 0／誤タップ N もありえる）」として通常扱い。

### a11y（未挑戦時の追加）
- 自動採点フェーズ移行時：`aria-live="polite"` で「時間切れです。変化していたパッチを表示します」
- 結果サマリ遷移時：「未挑戦として記録されました」と読み上げ

### モーフィング表現
- 角度差は staircase 値（初期 5°、難 3°、易 8°）
- 60 秒で線形に最大角度差まで変化（spec.md §7.1）
- prefers-reduced-motion 環境では 5 段階階段状（system.md §9.4 参照）

### 中断
- × ボタン → ConfirmDialog → ホーム

### a11y
- 試行中の SR：「変化したパッチをタップしてください。グリッドサイズ N 行 M 列。60 秒間は自由に選び直せます」（開始時 1 回のみ）
- 各セル `aria-label="N 行 M 列のパッチ"`、`aria-pressed={selected}`
- 矢印キーでセル間移動（Web）、Enter/Space で選択切替
- × ボタンは Tab でフォーカス可（緊急脱出経路）
- 試行中のスコア表示は出さない（仕様 §7.1）
- 残り時間カウントダウン（GameStatusBar 中央 30px Bold tabular-nums）：残り 5 秒以下で `aria-live="polite"`

### 採点後の正解ハイライト
- `highlightChangingIds` を 1.5 秒間設定
- 該当セルを scale(1 → 1.18 → 1)、枠 `color.highlight.correct`
- 点滅なし（OPT-9 / NF-11）
- ハイライト終了後、自動で S2-04 へ遷移（OPT-7 例外：採点表示は確実に見せるため）
