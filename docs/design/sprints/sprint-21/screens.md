# Sprint 21 — チュートリアルレベル Lv0（画面設計、v3.2）

> 対象：spec §8 S21 / §4.8・F-15（チュートリアル）・F-06（起動フローへの組込み）・`UserProfile.tutorialCompleted`・AS-31〜AS-34。
> **v3.2 新規**：オンボーディング完了後の**初回プレイ時のみ 1 度だけ**、本番 L1 に入る前に固定 3 ラウンド（R1=個数1 / R2=個数2 / R3=個数3、いずれも 30 秒・一方向・3x3・速度5）のチュートリアルを通す。各ラウンドは**個数を明示**（本番の「全て探せ」と異なる学習用表示）、**正誤に関わらず順に進む**。R3 完了で本番 L1 自動開始、`tutorialCompleted=true`。チュートリアルはレベル昇降・記録・統計・ストリークに**一切影響しない**（AS-33）。
> **設計方針**：チュートリアルは **S5 ゲーム画面（sprint-5）のレイアウト・コンポーネントを流用**し、チュートリアル固有の差分（練習表記・個数明示・3 ラウンド進行・橋渡し）のみを足す。新規大規模画面は作らない。

## 起動フローへの位置づけ（F-06・§4.8、v3.2）

```
起動
 └（旧データ検出時）F-11 データリセット通知（S3-2）
 └（初回のみ）オンボーディング（S7-1：免責→年齢→距離→概要）
 └ 距離リマインド（S7-2、自動進行）
 └【v3.2 新規・初回プレイのみ】チュートリアル Lv0（本書 S21）★
 │    R1（個数1）→ R2（個数2）→ R3（個数3）正誤に関わらず順送り
 │    R3 完了 → 橋渡し表示「練習はここまで。本番（レベル 1）を始めます」
 └ 本番 L1 自動開始（ホームタブ・セッション制、S5）
       ・2 回目以降：tutorialCompleted=true → チュートリアルをスキップして直接 L1
```
- **表示条件（AS-32）**：`UserProfile.onboardingCompleted=true` かつ `tutorialCompleted=false` の初回プレイ時のみ自動表示。完了で `tutorialCompleted=true` を永続化、2 回目以降は表示しない。全データ削除で `tutorialCompleted=false` に戻り再び表示（§7.9）。
- **距離リマインドとの順序**：オンボ完了後の初回は「距離リマインド → チュートリアル → L1」。チュートリアルもガボール注視のため、距離リマインドの後に置く（F-06 受け入れ基準「距離リマインド後にチュートリアル Lv0 が表示され、3 ラウンド完了後に本番 L1 が自動開始」）。
- **下限との関係（AS-34）**：本番で失敗を続けても Lv0 には戻らない。L1 が下限。Lv0 はクランプ範囲 [1, 総レベル数] 外の特別レベル。

### 進行（正誤に関わらず順送り、AS-31）
- 各ラウンドは締切（30 秒）またはクリアで締める（締め方は Generator 余地、§4.8）。推奨：本番 F-03 に準じて締切時に開示。
- **誤答でもやり直さない**：R1 → R2 → R3 と必ず順に進む（クリアでも失敗でも次へ）。TB-2 ドットが 1 つ進む。
- **進捗非干渉（AS-33/F-15）**：チュートリアルの結果（クリア/失敗）はレベル・連続失敗カウント・`SessionRecord`・ストリーク・統計のいずれにも影響しない。レベル昇降告知（LD-1）は**出さない**（レベル概念が動かないため）。

### a11y（F-15）
- 各ラウンド開始時に `aria-live="assertive"` で「**練習。{n} 個の回転を探してください**」（`game.a11y.start.tutorial`）。**「練習」コンテキストと個数を読み上げる**（F-15 受け入れ基準）。

- `GaborGrid` `role="group"` aria-label「回転しているパッチをすべて選んでください」。各 `GaborPatchCell` `role="checkbox"` + `aria-checked` + aria-label「パッチ {行}-{列}」（本番と同一）。
- X = aria-label「ゲームを中断」。カウントダウン aria-live（CD-1 既存）。
- 試行中は正誤フィードバックなし（締切まで音/振動なし、本番と同一）。

### 規範（F-15・§4.8、本番 F-03 との差分）
- **✅/❌ 開示は残す**（学習目的：「どれが回転だったか」を学ぶ）。✅実線=回転・正しく選択、✅薄め（透過50%）=回転・選び逃し、❌=静止・誤選択（本番 OV-2 と同一）。
- **総合結果（OV-3）はチュートリアルでは「失敗」を強調しない**（OPT-6 優しい言い回し）。クリア時は「できた！」等、誤りがあっても責めず「次へ」を主にする（Designer 判断で文言を学習導線向けに。総合「失敗」赤バッジは出さないか、控えめに「もう一度見てみましょう」相当でもよい）。**レベル昇降告知（LD-1）は出さない**（進捗非干渉）。
- **3 秒カウントダウン（CD-1 disclosure）はテンポ学習のため使ってよい**（本番と同じ流れを体験。§4.8 で 3 秒開示の有無は Designer 判断）。0 で次ラウンドへ自動遷移（R1→R2→R3）。簡素化する場合は「次へ」タップ導線でもよい（Generator 判断、ただし正誤に関わらず進む）。
- **内部数値（速度・角度）は出さない**（本番 F-03 と同一）。表示するのは練習・個数・結果・開示カウントダウンのみ。
- 開示中は格子タップ無効（GG-1 pointer-events: none）。
- **R3 開示後 → 橋渡し（S21-3）へ**。

### a11y
- 出現時 `aria-live="assertive"` で総合結果（「できた」等）を 1 度 → CD-1 disclosure が 3/2/1 を毎秒読み上げ（使う場合）。各 ResultMark の aria-label は本番と同一。

## チュートリアル中の中断（F-07 準用、§4.8）

- チュートリアル中の × / 他タブ選択時の扱いは Generator 判断（§4.8 は開示・カウントダウンを Designer 余地とする）。推奨：
  - **進行中（締切前）の × / 他タブ → 中断ダイアログ（DG-1）を出してよい**が、チュートリアルは進捗非干渉のため「記録されず」の文言は不要（「練習を中断しますか？」相当）。OK で中断 → 本番 L1 へ進む or ホーム待機（Generator 判断、ただしチュートリアルは進捗に影響しないため再表示条件＝`tutorialCompleted` 未了なら次回プレイで再びチュートリアル）。
  - 簡素化する場合：チュートリアルは中断ダイアログを省き、× で本番 L1 へスキップ（`tutorialCompleted=true`）でもよい（Generator 判断。ただし「初回のみ」の原則と整合させる）。
- いずれの実装でも、**チュートリアルの結果はレベル・記録・統計に影響しない**（AS-33）原則は不変。
