### 設定項目（F-13、データモデル §7.2、v3.1）
| 項目 | UI | 値域 | 既定 | 永続化（Settings/UserProfile） |

| 個数の範囲 | RG-1 チップ複数（6） | {1..6} の部分集合 | 既定 ON {1,2,3,4}（追加 5,6 OFF） | `variableRanges.count` |

> **v3.1 既定有効集合（`defaultVariableRanges`）は v3.0 と同一 → 総レベル数 720**。追加値は**既定 OFF（未選択チップスタイル）**で、ON にすると段数が増え総レベル数が増える（spec §4.1/AS-27、system §16.5）。`sessionMinutes` はレベル変数ではなく梯子に影響しない（AS-23）。

> **削除した v2 設定 UI（F-13 受け入れ基準・spec §11）**：n/m/r/a/b の手動スライダー、採点方式①②③、空間周波数変化 b。これらの UI は v3.0 に**一切存在しない**。レベル自動決定 5 変数の手動値固定 UI はない（範囲・変化順は「梯子の作り方」を変えるだけ）。

### 状態遷移（F-13、v3.1）
- 各コントロール変更 → 即座に `Settings`/`UserProfile` に保存（F-13「即座に保存」）。視聴距離は即保存。
- **`sessionMinutes`（v3.1、SR-1a）**：[−]/[+] で 1〜15 分を 1 分刻みで増減（1 で [−] disabled、15 で [+] disabled）。即座に `Settings.sessionMinutes` 保存、次セッションの長さに反映。**レベルの梯子（総レベル数・各レベルの中身）には影響しない**（クランプ・連続失敗リセットは発生しない、AS-23）。
- **拡張チップ（v3.1、RG-1）**：追加値（個数 5/6・時間 60/55/50/45/15/10・サイズ 5x5/6x6・速度 7/6.5/1.5/1）は**既定 OFF（未選択スタイル＝枠のみ）**。ON にすると総レベル数プレビューが即増加し、梯子が変わる → 現在レベルクランプ + 連続失敗 0 リセット（範囲変更と同じ波及、§4.5）。チップは flex-wrap 折返し（横スクロールなし、system §16.5）。
- **範囲（RG-1）/ 変化順（OR-1）の変更 → 梯子（総レベル数・各レベルの中身）が変わる**：
  - 総レベル数プレビューを即更新（「現在の設定：{N} レベル」）。
  - **現在レベルを新有効範囲にクランプ**（新上限超過なら新上限へ丸め）、**連続失敗カウントを 0 リセット**（spec §4.5、F-13 受け入れ基準）。
  - クランプが発生する変更時は予告/事後トーストで「レベルの梯子が変わりました。現在レベルは {clamped} に調整されました」を案内（`font.caption`）。
- **各変数 1 値以上必須**：最後の 1 チップを外そうとする操作はブロックし「少なくとも 1 つ選んでください」を案内（F-13 受け入れ基準）。
- 「免責事項を読む」→ `DC-1 DisclaimerPanel`（閲覧のみ、同意ボタンなし）をモーダル/別画面 → 閉じる（F-10）。
- 「全データ削除」→ タップ（1 段階目）→ `DG-1 ConfirmDialog`（削除確認、2 段階目）→「削除する」で全 `gaboreye:v3:*` を空に → `LevelState` を currentLevel=1/consecutiveFailures=0/highestLevel=0 に初期化 → 設定はデフォルトへ → 完了後ホームへ（F-13/§7.9）。「キャンセル」で何もしない。

### 状態：エンプティ/ロード/エラー
- 初回（設定未保存）：範囲はフル選択、変化順はデフォルト、その他は既定値。
- ロード：永続化読込中は短時間スケルトン or 既定値で即表示（NF-3 起動 3 秒以内）。
- 範囲外入力不可（チップは値集合内のみ）。エラー状態は実質発生しない。

### a11y
- タイトル `h1` を Skip link の着地点。
- RG-1：各チップ `role="checkbox"` + `aria-checked`、グループ `role="group"` aria-label「{変数名}の範囲」、制約は `aria-describedby`。
- OR-1：`role="list"`、各 `role="listitem"`、上下ボタン aria-label「{変数名}を 1 つ上へ/下へ」、順序変更を `aria-live` で読み上げ。
- Toggle=`role="switch"`、SegmentedControl=`role="radiogroup"`。
- 全データ削除ダイアログはフォーカストラップ、初期フォーカス「キャンセル」。

### レスポンシブ
- 360/375：1 カラム。RG-1 拡張チップ（時間 11・速度 13）は **flex-wrap で 3 段程度に自動折返し**（横スクロールなし、system §16.5）。OR-1 は 1 カラム。`sessionMinutes` ステッパー [−][+] は 56pt 角、ラベルと同行（収まらなければ値+ステッパーを 2 段に）。
- 1280：最大 720px 中央寄せ、1 カラム維持（可読性優先）。RG-1 チップは横に並びやすくなる。

## 画面 S3-2：起動時データリセット通知（F-11、v3）

`RZ-1 DataResetNotice`。起動時に旧名前空間（`gaboreye:v1:*` / `v1.1:*` / `v1.2:*` / **`v2:*`**）を検出・消去し `gaboreye:v3:*` 初期化（現在レベル L1・連続失敗 0）した**初回のみ 1 度**表示。

```
┌──────────────────────────┐ ← scrim
│  ┌────────────────────┐  │
│  │ 最新版へのアップデート│  │ ← font.h2 30px
│  │                      │  │
│  │ 最新版への           │  │ ← font.body 24px
│  │ アップデートのため、  │  │
│  │ 過去データを          │  │
│  │ リセットしました      │  │
│  │  ┌────────────────┐  │  │
│  │  │      OK         │  │  │ ← Button primary lg 64px（≥56pt）
│  │  └────────────────┘  │  │
│  └────────────────────┘  │
└──────────────────────────┘
```
- 表示は旧データ（v1〜v2）検出時のみ。2 回目以降の起動では出さない（F-11）。
- OK で閉じ、起動フロー（S7：オンボ or 距離リマインド）へ。
- a11y：`role="dialog"`、OK に初期フォーカス、OK は 56pt 以上。

> **v2.0 からの変更（F-11）**：消去対象に **`gaboreye:v2:*` を追加**（v2 はスコア管理スキーマで v3 レベル管理と非互換、AS-13）。レベルへの変換移行は行わない（全リセット）。表示タイミングはオンボーディング（S7）より前。S3 でデータ層・消去ロジック・通知コンポーネントを用意し、S7 で起動フローに組み込む。
