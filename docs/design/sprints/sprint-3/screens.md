### 状態（時系列）

| フェーズ | 持続時間 | 表示 |
|---|---|---|
| trialStart（固視待機） | 500ms | 固視点のみ、ボタン disabled |
| presentation | 300〜800ms（staircase） | 8 個のガボール、固視点維持 |
| mask | 200ms | 全 8 位置をランダム高コントラストガボール |
| answer | 最大 2000ms | ガボール消失、固視点維持、ボタン enabled |
| feedback | 800ms | 正解位置に矢印（後述） |
| nextTrial | 100ms | クールダウン |

### 試行終了時のフィードバック
- 最終 1 試行ではなく **各試行終了時に** 正解位置を矢印で 0.8 秒提示（仕様 §7.3）
- 矢印は固視点から正解パッチ位置へ向かう線分＋三角形、`color.highlight.correct`（黄）
- ユーザー回答の正誤は表示しない（連続性優先、Game 2 と同方針）

> 仕様 §7.3 では「試行終了時に正解位置を矢印で 0.8 秒」と明記。Game 2 とは異なり Game 3 は毎試行ハイライト。

### 中断（× ボタン）
- ConfirmDialog → ホーム

### a11y
- 固視点：装飾扱いだが「中心に固視点があります」を試行開始時に SR 通知（assertive ではなく polite、最初の 1 回のみ）
- ガボールは aria-hidden（提示時間が短すぎて読み上げ実用性がない）
- 8 ボタン `aria-label`：「時計の 12 時の方向」「時計の 1 時 30 分の方向」など
- キーボードショートカット：
  - 数字キー 1〜8：時計の 12, 1:30, 3, 4:30, 6, 7:30, 9, 10:30 の順に対応
  - 矢印キー：上 → 12 時、右上 → 1:30、右 → 3 時、…の 8 方向（Web 補助）
- 2 秒以内の回答制限：制限残り時間を SR で読み上げない（注意散漫を避ける）
- BGM 設定 ON 時のみリズム BGM 再生、OFF 時は完全無音
- ボタンラベルは **font.body 24px（=18pt、OPT-1 床）Medium**、tabular-nums。「10:30」のような長いラベルでも 24px を維持し、ボタン径 72px 内に収める

### Round 3 追記：odd one 位置を DOM 上で判別不能にする
- ガボールパッチ 8 個は **すべて同じ class（`patch`）／同じ data-attribute／同じ aria** で記述する
- odd one を識別する手がかりは「描画パラメータ（orientation の数値）」のみで、DOM ツリー上の class 名／属性名／order からは判別できないこと
  - `class="patch odd"` のような区別、`data-is-odd="true"` のような属性、`aria-label="odd one"` のような読み上げは**禁止**
  - JS から識別する場合は内部 state（描画ロジック側の変数）に保持し、DOM には漏らさない
- スクリーンリーダーは class を読まないため a11y 直接的影響はないが、JS Inspector・Devtools で正解を見破られないようにするのが目的
- 試行終了後の正解矢印フィードバック（0.8 秒、`color.highlight.correct`）のときに初めて DOM 上に正解位置がマークされる（`aria-label="正解の方向"` 付き）。それまでは DOM 上等価

### ゲーム間の遷移時間
- 仕様 F-05：「ゲーム間は 10 秒以内の結果サマリで自動進行」
  - **「次へ」を押せば即座に次へ**（OPT-7 を尊重）
  - 押されなくても 10 秒経過で自動進行（OPT-7 例外：コース完走を妨げない措置）
  - 自動進行までの残り秒数を 18pt 以上で表示（「あと 8 秒で次のゲームへ」）

### コース全体の所要時間
- 距離リマインド：5〜15 秒（ユーザー次第）
- Game 1：60 秒
- 結果 1：5〜10 秒
- Game 2：60 秒
- 結果 2：5〜10 秒
- Game 3：60 秒
- 結果 3：5〜10 秒
- 合計：約 200〜225 秒（仕様 180〜220 秒目安、+5 秒は許容）

## 6. S3-05：ストリーク加算ロジック（修正）

Sprint 2 で「2 ゲーム完了で +1」と暫定運用していたものを、仕様通り **3 ゲーム完了で +1** に修正。

### 仕様（spec.md F-13）
- おまかせコース完了で +1（同日 2 回目以降は加算しない）
- おまかせコース＝ Game 1 → Game 2 → Game 3 すべて完了

### 「完了」の判定
- 各ゲームのうち、`SessionRecord.gameNThreshold` が non-null（=ゲーム自体は最後まで進んだ）
- 中断（× ボタン → 「中断する」確認）は未完了扱い、ストリーク加算なし
- ゲーム途中の電話／アプリ切替（A-8）も未完了扱い

### 同日 2 回目
- ストリーク加算なし
- ただし `DailyStats.sessionCount` は +1
- ベスト閾値の更新は行う（複数セッションのうち最良）
