# Sprint 3 — ゲームコア：変化検出ロジック（画面・状態設計）

> 対象：spec §7 S3 / F-01（コア振る舞い）・F-02（採点 3 方式）・F-04（0〜100 集計）。
> S3 はロジック層中心（描画と ✅/❌ 開示 UI は S4）。本書は「ロジックが満たすべき観察可能な状態」と「採点 3 方式による画面差分」を定義し、S4 の描画実装が参照する状態モデルを固める。

---

## 1. ゲーム状態モデル（S4 描画が参照）

1 セッション = r ラウンド。各ラウンドの状態遷移：

```
[ラウンド開始]
   → 格子生成（n×n、変化パッチ k 個ランダム、静止パッチに初期角度/周波数固定）
   → カウントダウン m 秒開始（CountdownTimer）
   → ユーザーがパッチを直接タップ（トグル選択、何度でも）  ← 試行中：正誤フィードバックなし
   → [締切] 採点方式により分岐
   → 採点（TP/FP/FN 算出）
   → 結果開示（S4：✅/❌ オーバーレイ）   ← 試行中のみ無効化
   → 次ラウンド or セッション完了 → セッションスコア 0〜100（S6 で結果カード表示）
```

### 1.1 格子生成パラメータ（system §9.3/§9.4 採用）
- 変化パッチ個数 k：1〜floor(n²/3)（n=4→1〜5）、少なめ寄りの分布、**0 個は出さない**。
- 各変化パッチの種類：回転のみ 40% / 周波数のみ 40% / 両方 20%（独立）。
- 回転方向：時計/反時計 独立ランダム。周波数：増加/減少 独立ランダム（AS-13）。
- 静止パッチ：互いに 12° 以上離れた初期角度、初期周波数 2.0–4.0 cpd（時間変化なし）。
- 画面に個数・種類・内訳は**一切表示しない**（F-01）。

### 1.2 採点ロジック（全方式共通、F-02）
- 選択した変化パッチ = TP（+1）、選択した静止パッチ = FP（−1）、選ばなかった変化パッチ = FN（0）。
- ラウンド得点 roundScore = TP − FP（`RoundRecord` に tp/fp/fnCount, roundScore を記録）。
- 試行中は正誤を出さない（F-02 受け入れ基準）。

### 1.3 セッションスコア 0〜100（F-04）
観察可能な性質（具体式は Generator 判断）：
- 全ラウンドで変化パッチ全正答 & 誤選択ゼロ → **100**。
- 変化パッチを 1 つも選ばず誤選択もしない → 下限（0 寄り、満点と明確に区別）。
- FP はスコアを下げる方向。
- 中断セッションは集計・記録対象外（completedAt=null、F-07）。
- 完了時に整数スコア算出 → `SessionRecord.sessionScore` → 日次（S7）・履歴へ。
- 提案正規化（Designer メモ、Generator 採用可）：`session = clamp(0, 100, round(100 × ΣTP / Σ変化パッチ数 − 罰 × ΣFP / Σ静止パッチ数))`。罰係数で FP 減点。詳細は Generator。

## 2. 採点 3 方式の画面差分（F-02、S4 描画の前提）

格子の下（刺激領域直下）のアクション領域の差分。**3 方式で格子・上部カウントダウンは共通**。違いは下部のみ。

### 方式①：自動採点・確定なし（auto-no-confirm）
```
[上部] ✕   残り 14 秒
[中央] n×n ガボール格子（タップで選択トグル、m 秒内何度でも変更可）
[下部] （何も置かない。確定ボタンなし）
```
- m 秒経過で自動採点。確定ボタンは DOM に存在しない。

### 方式②：自動採点・確定あり（auto-confirm）
```
[上部] ✕   残り 14 秒
[中央] n×n ガボール格子
[下部] ┌──────────────┐
       │    確定        │ ← BN-1 ConfirmButton（primary lg 64px）
       └──────────────┘
```
- 「確定」押下で m 秒を待たず即採点。押さなければ m 秒経過で自動採点。

### 方式③：全問正解で自動遷移（all-correct-advance）
```
[上部] ✕   残り 14 秒
[中央] n×n ガボール格子
[下部] （確定ボタンなし。選択が全問正解になった瞬間に次ラウンドへ）
```
- 変化パッチを過不足なく選択した時点で m 秒を待たず次ラウンドへ自動遷移。
- 誤選択や選択漏れがある間は遷移しない。m 秒経過で強制採点。
- 全問正解即遷移時の開示は短い正解フィードバック（総合 ✅ を 0.6 秒、S4）。

> 共通：未選択のまま m 秒経過でも採点される（TP=0,FP=0）。採点時点の選択状態が回答（方式①②）。

## 3. 状態（エンプティ/ロード/エラー）
- ラウンド開始時のロード（格子生成）：体感ゼロ（生成は同期/即時）。万一遅延時は #808080 背景のまま m 秒カウント開始を待つ（NF-3 範囲内）。
- 未選択：許容（採点される）。エラー状態なし。
- パラメータが極端（n=5・m=10 等）でも崩れない設計（system §3.2 レスポンシブ）。

## 4. a11y（ロジック層に関わる部分）
- 各パッチ `role="checkbox"` + `aria-checked`（NF-15）。
- 採点後の読み上げ（「正解」「不正解」+ TP/FP/FN 要約）は S4 の ResultOverlayLayer が担当。
- 試行中はフィードバックなし（音/振動も採点まで出さない、F-14）。
