## 画面 S4-1：ゲームプレイ中（F-01/F-12）

**フルスクリーン許容（NF-29）**：`SafeAreaWrapper mode="game"`。ガボール背景 #808080 はステータスバー領域まで広がってよい。上部バーの文字・X・残り秒は top inset 内。

### レイアウト（スマホ縦 375、n=4 例）
```
┌─────────────────────────────────┐ ← 画面端（背景 #808080 がここまで）
│〔ステータスバー領域：背景灰〕    │

│  ✕            18           　    │ ← GB-1：左 X(48pt)、中央 残り秒 font.h2 30px 段階色
│                                  │
│   ┌──────┬──────┬──────┬──────┐ │
│   │ ▦▦   │ ▦▦   │ ▦▦   │ ▦▦   │ │ ← GG-1 GaborGrid（n×n）
│   ├──────┼──────┼──────┼──────┤ │   各セル GG-2 GaborPatchCell
```

### 規範・状態
- 格子全体辺 `min(viewport.short_edge - 32, 360)px`、隙間 6px（n≥4）/8px（n=3）。パッチ辺 = (全体辺−隙間)/n。
- ガボール描画は system §7：実サイズ 1.5 倍生成 → 矩形クリップ（NF-27/28、四隅に背景露出なし）。基準 3 cpd。
- **変化アニメ**：回転 a °/sec、周波数変化 b hz/sec（continuous、m 秒継続・一定速度、AS-13）。reduced-motion でも止めない（NF-13、課題上必須）。
- **選択**：パッチ直接タップでトグル。選択中 `selection.subtle` 2px 枠（縞の視認を阻害しない）。種類選択 UI なし、「現在の回答」テキストなし（F-01）。
- **カウントダウン（F-12/CD-1）**：上部バー中央、`font.h2` 30px tabular-nums。通常=白 / ≤5 秒=黄(warn) / ≤3 秒=赤(danger) + Black 太字。点滅なし。≤3 秒で毎秒ティック音（S9）。
- **ボトムタブバーはプレイ中も表示**（F-05：他タブ選択で中断ダイアログ、S5）。フルスクリーン背景でもタブバーは bottom セーフエリア内に重畳。
- 採点方式差分は S3 §2 参照（① 下部なし／② 確定ボタン／③ 全問正解即遷移）。

### a11y
- `GaborGrid` `role="group"` aria-label「変化しているパッチをすべて選んでください」。
- 各 `GaborPatchCell` `role="checkbox"` + `aria-checked` + aria-label「パッチ {行}-{列}」。Web Tab 到達 + Space トグル。
- X = aria-label「ゲームを中断」。カウントダウン aria-live（§CD-1）。
- 試行中は正誤フィードバックなし（音/振動も採点まで出さない）。

### レスポンシブ
- 360：全体辺 = 360−32 = 328px。375：343px（上限 360 未満）。1280：480px、中央寄せ。
- どの n（3/4/5）でも格子は正方形を維持、はみ出さない。

## 画面 S4-2：結果開示オーバーレイ（F-03）

採点後、**画面遷移せず**ガボール格子はそのまま残し、上に `OV-1 ResultOverlayLayer` を 200ms フェードイン（reduced-motion 0ms）。

### レイアウト
```
┌─────────────────────────────────┐ ← 背景 #808080（フルスクリーン継続）
│  ✕            0                  │ ← GameTopBar（残り 0、停止）
│   ┌──────┬──────┬──────┬──────┐ │
│   │ ▦✅  │ ▦❌  │ ▦▦   │ ▦▦   │ │ ← TP=実線✅ / FP=❌
│   ├──────┼──────┼──────┼──────┤ │   各パッチ中央に overlayBg 円 + アイコン
│   │ ▦▦   │ ▦✅  │ ▦▦   │ ▦◌  │ │   FN=薄✅（◌ 表現、透過50%・選び逃し）
│   ├──────┼──────┼──────┼──────┤ │   静止で選ばなかったパッチ=何も出さない
│   │ ▦▦   │ ▦▦   │ ▦▦   │ ▦▦   │ │
│   ├──────┼──────┼──────┼──────┤ │
│   │ ▦▦   │ ▦▦   │ ▦✅  │ ▦▦   │ │
│   └──────┴──────┴──────┴──────┘ │
│            ┌────────┐            │ ← OV-3 総合 ✅/❌ を刺激領域直下に 1 個
│            │   ✅   │            │   緑(aggregate.success) / 赤(danger)
│            └────────┘            │   数値テキストなし
└─────────────────────────────────┘
```

### 規範（F-03）
- 独立した結果画面に遷移しない（格子はそのまま）。
- **TP**：✅ 実線・不透明（`color.result.check.tp`）。**FN（選び逃し）**：✅ 薄め透過 50%（`color.result.check.fn`）で「取りこぼし」を区別。**FP**：❌（`color.result.cross.fp`）。正解で選ばなかった静止パッチは何も表示しない。
- 各アイコン下に `color.result.overlayBg` 半透明円（縞を完全には隠さない、直径 ≒ パッチ辺 55%）。
- **総合 ✅/❌**：刺激領域直下に `OV-3` 1 個（TP=変化全数 かつ FP=0 → ✅、それ以外 ❌）。**数値テキストは一切なし**（閾値/速度/前回比禁止）。
- 開示中は格子タップ無効（`GaborGrid` pointer-events: none）。
- 遷移：方式①② → 短いインターバル **1.5 秒**で次ラウンド（旧 10 秒固定廃止）。方式③で全問正解即遷移時 → 総合 ✅ を 0.6 秒だけ短く出して次へ。
- 最終ラウンド後はセッション結果（S6）へ。

### a11y（F-03/NF-10）
- 出現時 `aria-live="assertive"` で「正解」/「不正解」を 1 度読み上げ（OV-3）。各 ResultMark に aria-label「正解（選択済み）/正解（選び逃し）/不正解（誤選択）」。
- フォーカスは結果開示中は移動させない（自動遷移のため）。方式①②の 1.5 秒中はタップ無効。

### レスポンシブ
- アイコン/円はパッチ辺に比例（360〜1280 で破綻しない）。総合バッジは格子直下中央。
