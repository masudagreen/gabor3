# Sprint 4 — ゲームコア：回転変化生成・選択・クリア判定（画面なし・状態モデル設計）

> 対象：spec §8 S4 / F-01（ロジック）・F-03（判定）。
> **このスプリントは描画 UI を作らない**（描画と ✅/❌ 開示 UI は S5）。レベルの 5 変数から n×n 格子・回転パッチ・静止パッチを生成し、タップ選択・クリア/失敗判定が純ロジックとして動く。

## 1. ゲーム状態モデル（S5 描画が参照、1 ゲーム = 1 レベル挑戦）

```
[ゲーム開始]（currentLevel から 5 変数を導出：count/seconds/direction/gridSize/rotationSpeed）
   → 格子生成（gridSize×gridSize、回転パッチ count 個をランダム位置に配置、残りは静止）
       ・回転パッチ：direction（一方向/振動）・rotationSpeed（deg/sec）で回転
       ・静止パッチ：ランダム初期角度（互いに 12° 以上分散、時間変化なし、system §7.2）
   → カウントダウン seconds 秒開始（CountdownTimer、F-12）
   → ユーザーがパッチを直接タップ（トグル選択、何度でも） ← 試行中：正誤フィードバックなし
   → [締め切り] 次のいずれか：
       (a) 回転パッチを過不足なく全選択（誤選択0・選び逃し0）になった瞬間 → 即時クリア締め切り
       (b) seconds 秒経過（時間切れ）
   → クリア/失敗判定（§2）
   → 結果開示（S5：✅/❌ オーバーレイ + 総合クリア/失敗）  ← 開示中はタップ無効
   → レベル増減（S2 ロジック）→ ホーム結果表示（S7）
```

### 1.1 格子生成パラメータ（spec §4.1 から導出、Designer 確認）
- **格子サイズ**：現在レベルの `gridSize`（3x3 = 9 セル / 4x4 = 16 セル）。
- **回転パッチ個数**：現在レベルの `count`（1〜4）。ランダム位置に配置。**v2 の「個数ランダム・非表示」は廃止 → レベルで確定し画面に明示**（F-02）。
- **回転の挙動**：`direction`（一方向 or 振動）と `rotationSpeed`（6〜2 deg/sec）。回転パッチは全て同じ direction/speed（そのレベルの値）。
  - 一方向：時計回り or 反時計回りのどちらか（パッチごとにランダムでも全パッチ同方向でも可、Generator 判断。「一方向であること」が弁別できればよい）。
  - 振動：時計回り↔反時計回りを往復（system §4.1 モーション狙い、NF-28c）。
- **静止パッチ**：残りの (gridSize²−count) 個。互いに 12° 以上離れた初期角度で固定。時間変化なし（弁別の鍵、system §7.2）。
- **空間周波数は固定**（基準 3 cpd、時間変化なし）。v2 の周波数変化（b）は廃止。

### 1.2 クリア/失敗判定（spec §4.3、F-03）
- 締め切り時点のユーザー選択集合を、回転パッチ集合と比較：
  - **クリア**：選択 = 回転パッチ（誤選択 FP=0 かつ 選び逃し FN=0）。
  - **失敗**：誤選択あり（静止パッチを選んでいる）or 選び逃しあり（回転パッチを選んでいない）or 時間切れで上記が成立しない。
- **締め切りの契機**：全問正解になった瞬間に即時締め切り（v2 方式③に相当する 1 本）、または時間切れ。**確定ボタン・部分点なし**（AS-5、採点 3 方式廃止）。
- 開示用メモリ state：各パッチの「回転/静止」「ユーザー選択/未選択」を保持 → S5 で ✅（回転・選択）/ ✅薄（回転・未選択＝選び逃し）/ ❌（静止・選択＝誤選択）に振り分け。
- 記録：`GameRecord { result: 'clear'|'fail', levelParams: {count,seconds,direction,gridSize,rotationSpeed}, levelDelta }`。

## 2. 状態（エンプティ/ロード/エラー）
- ゲーム開始時のロード（格子生成）：体感ゼロ（同期/即時）。万一遅延時は #808080 背景のまま seconds カウント開始を待つ（NF-3 範囲内）。
- 未選択のまま時間切れ：失敗（FN>0）。エラー状態なし。
- 範囲設定で極端なレベル（count4・seconds20・4x4・速2 等）でも崩れない設計（system §3.2 レスポンシブ）。

## 3. a11y（ロジック層に関わる部分）
- 各パッチ `role="checkbox"` + `aria-checked`（NF-15）。
- ゲーム開始時の読み上げ「レベル {level}。{count} 個の回転を探してください」（system §6、`game.a11y.start`）は S5 が発火するが、必要な状態（level/count）はこの層で確定。
- 締め切り後の読み上げ（「クリア」「失敗」）は S5 の ResultOverlayLayer が担当。
- 試行中はフィードバックなし（音/振動も締め切りまで出さない、F-14）。
