## 画面 S5-1：ゲームプレイ中（F-01/F-02/F-12）

**フルスクリーン許容（NF-29）**：`SafeAreaWrapper mode="game"`。ガボール背景 #808080 はステータスバー領域まで広がってよい。上部バーの文字・X・残り秒・**レベル番号・個数案内**は top inset 内（セーフエリア内）。

### 規範・状態
- **GameTopBar（GB-1、v3.1 改訂、F-01/F-02）**：左 X + **セッション残り時間「あと {mm}:{ss}」**（`font.label` 24px Medium・`color.fg.secondary` 灰・段階色なし・控えめ、system §16.2）/ 中央 **ラウンド制限時間**カウントダウン（CD-1 inline、段階色）/ **右にレベル番号ピル `LB-1 inline`「レベル {currentLevel}」**。バーは不透明 `color.countdown.bg`（white/#15171C）で 7:1 を保証。レベル番号は `color.level.fg`（8.6:1）。**3 つの数値は サイズ・色・表記で差別化**（残り=灰/分:秒、制限時間=段階色/秒、レベル=ピル）。狭幅 360 で X と残り時間が重なる場合は残り時間を個数バナー同列右端へ逃がす（Generator 判断、制限時間タイマーとは別位置を維持）。
- **ラウンドごとのレベル変化（v3.1、F-04）**：レベルはラウンドごとに上下する。右ピルが**次ラウンドで変化後の値に切り替わる**ことで観察できる（system §16.3）。
- **個数案内（CB-1 新規、F-02）**：バー直下・格子の上に「{count} 個探せ！」を `font.body.lg` 26px Bold（18pt 以上）。不透明ピルで縞に被らない。**ゲーム開始時点で確定、ゲーム中変動なし**。
- 格子全体辺 `min(viewport.short_edge - 32, 360)px`、隙間 6px（n=4）/8px（n=3）。パッチ辺 = (全体辺−隙間)/n。
- ガボール描画は system §7：実サイズ 1.5 倍生成 → 矩形クリップ（NF-27/28、四隅露出なし）。基準 3 cpd。
- **回転アニメ（F-01）**：回転パッチは `rotationSpeed` deg/sec で `direction`（一方向＝連続回転 / 振動＝往復回転、system §4.1）。**振動は時計回り↔反時計回りの往復として観察できる**（NF-28c）。静止パッチは固定（時間変化なし）。reduced-motion でも止めない（NF-13、課題上必須）。
- **選択**：パッチ直接タップでトグル。選択中 `selection.subtle` 2px 枠（縞の視認を阻害しない）。**回転種類/速度を指定する UI はゲーム画面に存在しない**（F-01 受け入れ基準）。「現在の回答」テキストなし。
- **カウントダウン（F-12/CD-1）**：上部バー中央、`font.h2` 30px（22pt 以上、F-01 受け入れ基準）tabular-nums。残り秒 = 現在レベルの `seconds`。通常=白 / ≤5 秒=黄(warn) / ≤3 秒=赤(danger) + Black 太字。点滅なし。≤3 秒で毎秒ティック音（S10）。
- **ボトムタブバーはプレイ中も表示**（F-05：他タブ選択で中断ダイアログ、S6）。フルスクリーン背景でもタブバーは bottom セーフエリア内。

### a11y
- ゲーム開始時に `aria-live="assertive"` で「レベル {level}。{count} 個の回転を探してください」（system §6、`game.a11y.start`）。**レベル・個数の読み上げ**（spec a11y 要件）。
- `GaborGrid` `role="group"` aria-label「回転しているパッチをすべて選んでください」。
- 各 `GaborPatchCell` `role="checkbox"` + `aria-checked` + aria-label「パッチ {行}-{列}」。Web Tab 到達 + Space トグル。
- レベルピル aria-label「レベル {n}」。個数案内 aria-label「{n} 個の回転を探してください」。
- X = aria-label「ゲームを中断」。カウントダウン aria-live（§CD-1）。
- 試行中は正誤フィードバックなし（音/振動も締め切りまで出さない）。

### レスポンシブ
- 360：全体辺 = 360−32 = 328px。375：343px（上限 360 未満）。1280：480px、中央寄せ。
- gridSize 3/4 のどちらでも格子は正方形を維持、はみ出さない。レベルピル・個数案内は 360 でも折り返さず収まる（短文）。

## 画面 S5-2：結果開示オーバーレイ + 3 秒カウントダウン（F-03、v3.1 改訂）

締め切り後、**画面遷移せず**ガボール格子はそのまま残し、上に `OV-1 ResultOverlayLayer` を 200ms フェードイン（reduced-motion 0ms）。**v3.1：開示中に 3 秒固定の `CD-1 disclosure` カウントダウンを刺激領域直下（OV-3 近傍）に表示し、0 で自動的に次ラウンド or セッション要約へ遷移**（spec §4.6・AS-25・system §16.1）。

### レイアウト（失敗例：回転 3 個中 2 個選択、静止 1 個誤選択）
```
┌─────────────────────────────────┐ ← 背景 #808080（フルスクリーン継続）
│ ✕ あと4:02   0   レベル 23       │ ← GameTopBar（ラウンド残り 0 停止、セッション残り時間は継続表示、レベル維持）
│         ┌──────────────┐         │
│         │  3 個探せ！   │         │ ← CountBanner は残す（参照のため）
│         └──────────────┘         │
│   ┌──────┬──────┬──────┬──────┐ │
│   │ ▦✅  │ ▦❌  │ ▦▦   │ ▦▦   │ │ ← ✅実線=回転・正しく選択 / ❌=静止・誤選択
│   ├──────┼──────┼──────┼──────┤ │   各パッチ中央に overlayBg 円 + アイコン
│   │ ▦◌   │ ▦✅  │ ▦▦   │ ▦▦  │ │   ◌=回転・選び逃し（✅薄め透過50%）
│   ├──────┼──────┼──────┼──────┤ │   静止で選ばなかったパッチ=何も出さない
│   │ ▦▦   │ ▦▦   │ ▦▦   │ ▦▦   │ │
│   ├──────┼──────┼──────┼──────┤ │
│   │ ▦▦   │ ▦▦   │ ▦▦   │ ▦▦   │ │
│   └──────┴──────┴──────┴──────┘ │
│      ┌──────────┐   ┌────┐       │ ← OV-3 総合バッジ + CD-1 disclosure(3秒)
│      │  ✕ 失敗   │   │ 3  │       │   失敗=赤 / クリア=緑+「クリア」
│      └──────────┘   └────┘       │   3秒=赤(danger)+Black太字・不透明プレート
│   ▼ レベル 23 → 22（任意・LD-1）  │ ← ラウンド昇降告知（色+矢印+テキスト、NF-12）
└─────────────────────────────────┘
```
> 開示カウントダウンは OV-3 総合バッジの**右隣 or 直下**に置き、上部バー中央の制限時間タイマーと物理的に離す（混同回避、system §16.1）。

### 規範（F-03、v3.1）
- 独立した結果画面に遷移しない（格子はそのまま、F-03 受け入れ基準）。
- **✅ 実線（回転・正しく選択）**：`color.result.check.correct`、不透明。**◌＝✅ 薄め透過 50%（回転・選び逃し）**：`color.result.check.missed`。両者は透明度で区別。**❌（静止・誤選択）**：`color.result.cross.wrong`。正しく選ばなかった静止パッチは何も表示しない。
- 各アイコン下に `color.result.overlayBg` 半透明円（縞を完全には隠さない、直径 ≒ パッチ辺 55%。**5x5/6x6 では比例縮小**、system §16.6）。
- **総合クリア/失敗（OV-3）**：刺激領域直下に 1 個。クリア=緑 + ✅ + 「クリア」、失敗=赤 + 「失敗」。色+アイコン+テキストで NF-12。
- **3 秒開示カウントダウン（CD-1 disclosure、v3.1）**：OV-3 近傍に 3→2→1（`font.h2` 30px、`color.countdown.danger` 赤 + Black 太字・不透明プレート・点滅なし・数字のみ＝F-12）。**3 秒は確定値**。0 で自動遷移。3/2/1 で毎秒ティック音（S10）。
- **ラウンド昇降告知（LD-1、任意・v3.1）**：開示内に +1=▲緑 /±0=―灰 /−1=▼暗橙（責めない、§1.4）をコンパクト併置可（system §16.3）。これは v3.0 でホーム結果カードにあった LD-1 の移設。
- **内部数値（速度・角度）は一切表示しない**。表示するのはレベル番号・個数・結果・開示カウントダウン・レベル変化のみ（F-03 受け入れ基準）。
- 開示中は格子・開示カウントダウンともタップ無効（`GaborGrid` pointer-events: none、F-03）。
- **遷移（v3.1）**：**全問正解（クリア）でも即時終了せず**、必ず 3 秒開示してから遷移。3 秒後に累積経過時間が `sessionMinutes`×60 秒**未満なら更新後レベルで次ラウンド自動開始（S5-1 へ戻る）**、**到達/超過ならホームのセッション要約（S7-3）へ自動遷移**（spec §4.6・F-08）。v3.0 の「1.5 秒 → 必ずホーム結果へ」は廃止。
- クリッピング品質（NF-26〜28c）：回転中・振動中も四隅露出なし。

### a11y（F-03/NF-10）
- 出現時 `aria-live="assertive"` で「クリア」/「失敗」を 1 度読み上げ（OV-3）→ 続けて `CD-1 disclosure` が 3/2/1 を毎秒読み上げ。各 ResultMark に aria-label「回転（選択済み）/回転（選び逃し）/誤選択」。LD-1 併置時「レベル {from} から {to}」を `aria-live="polite"`。
- フォーカスは開示中は移動させない（自動遷移のため）。3 秒中はタップ無効。

### レスポンシブ
- アイコン/円はパッチ辺に比例（360〜1280・3x3〜6x6 で破綻しない、system §16.6）。総合バッジ + 開示カウントダウンは格子直下中央に横並び、360px で収まらなければ縦積み。

## v3.1 セッション動線（S14/S15：ラウンド反復・締切時判定）

> spec §4.3/§4.6・F-01/F-03/F-04。本画面（S5-1 プレイ ↔ S5-2 開示）がセッション中**連続反復**する。

### ループ構造
```
セッション開始（距離リマインド後、現在レベルから）
  └▶ ラウンド生成（現在レベルの n×n 格子・回転個数・「時間」秒）   ← S5-1
       └▶ ユーザータップ選択（締切前は正誤フィードバックなし）
            └▶ 「時間」秒経過＝締切 → クリア/失敗を判定（§4.3）
                 └▶ レベル昇降を即適用（§4.4：クリア+1 / 2連続失敗−1）
                      └▶ 開示（✅/❌・総合クリア/失敗・LD-1）+ 3秒カウントダウン  ← S5-2
                           └▶ 3秒後、累積経過時間を確認：
                                ├ < sessionMinutes×60 秒 → 更新後レベルで次ラウンド（S5-1 へ）
                                └ ≧ sessionMinutes×60 秒 → セッション要約（S7-3）へ
```
- **締切時判定に統一**：全問正解での即時終了は廃止（spec §11）。各ラウンドは必ず「時間」秒続く。
- **最後のラウンド**：途中で累積時間が指定時間を超えても打ち切らず完走（spec §4.6）。完走後に累積が指定時間以上ならセッション終了。
- **レベルはラウンドごとに上下**：次ラウンドの S5-1 で GB-1 レベルピルが変化後の値を表示（F-04 観察可能性）。
- **セッション残り時間**：GB-1 左の「あと {mm}:{ss}」が毎秒減る（控えめ・段階色なし、system §16.2）。
- **中断（F-07、S15/S6）**：プレイ中の × / 他タブで中断ダイアログ。OK で未完ラウンド破棄（完了済みラウンドのレベル変化は保持）。開示中（3 秒）の中断扱いは Generator 整合（進行中扱いでダイアログ or 開示中は自由遷移、いずれか一貫させる）。
