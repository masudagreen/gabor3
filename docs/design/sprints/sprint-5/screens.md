## 画面 S5-1：ボトムタブナビ（F-05）

`NV-1 BottomTabBar` を全画面の最下部に常時表示。3 タブ：ホーム / 履歴 / 設定。

### レイアウト
```
├─────────────────────────────────┤
│  ┌─────────┬─────────┬─────────┐│
│  │ ●(塗)   │ ▢(線)   │ ⚙(線)   ││ ← アイコン（塗/線で形も区別）
│  │ ホーム  │ 履歴    │ 設定    ││ ← ラベル font.label 24px
│  │ ▔▔▔(青) │         │         ││ ← アクティブ上辺インジケータ 3px
│  └─────────┴─────────┴─────────┘│ ← 各 64px 高 + bottom セーフエリア
└─────────────────────────────────┘
```

### 規範（F-05/NF-12）
- 常時 3 タブ表示。タップで対応画面に切替。
- アクティブ表示は**色＋形＋太字**の 3 重で（色のみ非依存）：上辺 3px インジケータ、前景 `tab.active.fg`、ラベル Bold、アイコン塗り。非アクティブはアイコン線・ラベル Medium・前景 `tab.inactive.fg`。
- 各タブのタップ領域 48pt 以上（推奨 64px）。
- bottom セーフエリア（iOS Home Indicator / Android nav bar）の内側に置き、タブバー下端に inset 分パディング。

### プレイ中の中断連動（F-05/F-07）
- **プレイ中（ラウンド進行中）に他タブをタップ → 中断確認ダイアログ（S5-2）を出す**。即切替しない。
- **完了後・待機中（ゲーム非進行中：セッション結果表示中・履歴・設定）はダイアログなしで自由切替**。

### a11y
- `role="tablist"`、各タブ `role="tab"` + `aria-selected` + aria-label（「ホームタブ」等）。
- Web：Tab で各タブにフォーカス、Enter/Space で起動、←/→ でタブ間移動（任意）。focus ring 3px。

### レスポンシブ
- 360：3 タブ各約 120px 幅、48pt 確保。375/1280：全幅 3 等分（PC もタブバーは全幅、コンテンツのみ中央寄せ）。

## 画面 S5-2：中断確認ダイアログ（F-07）

`DG-1 ConfirmDialog`（中断用途）。プレイ中のみ、**X ボタンと他タブ選択の両方で同一ダイアログ**。

### レイアウト
```
┌──────────────────────────┐ ← scrim（背後ゲーム見える）
│  ┌────────────────────┐  │
│  │ プレイを中断しますか？│  │ ← font.h2 30px
│  │                      │  │
│  │ 中断するとこの        │  │ ← font.body 24px
│  │ セッションは          │  │
│  │ 記録されません。      │  │
│  │  ┌────────────────┐  │  │
│  │  │   中断する       │  │  │ ← primary、56pt 以上
│  │  └────────────────┘  │  │
│  │  ┌────────────────┐  │  │
│  │  │   続ける         │  │  │ ← secondary、56pt 以上
│  │  └────────────────┘  │  │
│  └────────────────────┘  │
└──────────────────────────┘
```

### 状態遷移（F-07）
- **トリガー**：プレイ中の X タップ、またはプレイ中の他タブタップ。両方で同一ダイアログ。
- **「中断する」（OK）**：セッション中断、進行中セッションは記録に残さない（`SessionRecord.completedAt=null`／保存しない）。
  - X 起点 → ゲーム終了（ホームの待機/初期状態へ、または距離リマインドへ戻す扱いは Generator 判断だが「中断後の着地」はホームタブの非進行状態）。
  - タブ起点 → 当該タブ（履歴 or 設定）へ遷移。
- **「続ける」（キャンセル）**：ダイアログを閉じ、ゲーム継続（残り時間・選択状態を保持）。
- ゲーム完了後・待機中（非進行中）は X・タブ移動のいずれでもダイアログを出さず自由遷移。

### a11y
- `role="dialog"` `aria-modal="true"` `aria-labelledby`。初期フォーカスは安全側「続ける」。
- Esc = キャンセル（続ける）。フォーカストラップ。背後操作不可。
- ボタン各 56pt 以上（spec F-07）。

### レスポンシブ
- ダイアログ最大幅 360px（スマホ）/ 480px（PC）。中央配置、セーフエリア内（NF-30）。
