## 2. F-11 vs F-05 の整合性（OPT-7 の解釈）

仕様書には：
- F-05：「ゲーム間は 10 秒以内の結果サマリで自動進行する」
- F-11：「サマリ画面は『次へ』を押すまで自動で進まない（OPT-7）」

→ **解釈**：
- ユーザーが「次へ」を押せば即進行（推奨パス）
- 押さない場合は最大 10 秒で自動進行（コース完走を妨げない措置）
- 自動進行までの残り秒数を 18pt 以上で明示（OPT-7 の「タイムアウトはカウントダウン明示」を満たす）
- **単体プレイモードでは自動進行しない**（コース連続性が無いため）

### 未挑戦（スキップ／全試行未回答）時のレイアウト（仕様 F-11 / spec.md L195 対応）

**「未挑戦」と判定されるケース**：
- Game 1：60 秒間 1 度もパッチをタップせず時間切れ（S2-03 のタップ無しタイムアップ）
- Game 2：60 秒間で全試行が未回答（パッチ B 消失後 3 秒以内に回答なしが連続）
- Game 3：60 秒間で全試行が未回答
- 共通：ユーザーが手動でゲームをスキップした場合（中断確認 → 「スキップ」を選択した場合。本仕様では中断時は「ホームへ戻る」のみのため、現時点では明示的なスキップ操作は無し。将来追加時はこのパターンに該当）

**仕様準拠の方針**：
仕様 F-11「スキップ操作は『未挑戦』として記録され、サマリには表示されない」を、「サマリ画面そのものは表示するが、閾値・正答率・前回比は表示せず『未挑戦』ラベルで置換する」と解釈する（ユーザーに「何が起きたか」を説明する画面は必要。完全省略するとユーザーが状況を理解できない）。

## 3.1. S5-01：単体プレイモード時のレイアウト（仕様 F-05 / F-11 対応）

### 単体プレイモードでは自動進行カウントダウンを表示しない

§2 の「単体プレイモードでは自動進行しない」をレイアウトレベルで明示する。

**コースモード版（§3 メインレイアウト）との差分**：

| 要素 | コースモード | 単体プレイモード |
|---|---|---|
| 「次へ」ボタン | 表示（OPT-7 準拠） | 表示（OPT-7 準拠） |
| 自動進行カウントダウン文言 | 「あと 8 秒で次のゲームへ」表示（10 → 0 で減算、0 で次画面に自動進行） | **非表示**（自動進行しない） |
| カウントダウン aria-live | polite | （要素自体が無い） |
| 「次へ」押下時の遷移先 | 次のゲーム（Game 2 結果なら Game 3 距離リマインド／Game 3 結果ならクールダウン S5-02） | S2-08 単体プレイ後選択画面 |

| 戻るボタン（左上 ←） | 中断確認ダイアログ | そのままホームへ（記録は保存済み） |

## 10. テスト要件（追加分）
