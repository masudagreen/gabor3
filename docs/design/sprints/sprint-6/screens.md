### 動作

### 進捗ヒント（B-02〜B-08 で可視化粒度を揃える）

| バッジ | ヒント文言の型 | 例 |
|---|---|---|
| B-02 三日坊主突破 | 「あと N 日」 | 「あと 1 日」 |
| B-03 一週間の習慣 | 「あと N 日」 | 「あと 4 日」 |
| B-04 一ヶ月の継続 | 「あと N 日」 | 「あと 23 日」 |
| B-05 100 試行 | 「累計 N 試行 / 100 試行」 | 「累計 87 試行 / 100 試行」 |
| B-06 視野ハンター | 「Game 3 ベスト N 点 / あと M 点」 | 「Game 3 ベスト 64 点 / あと 16 点」 |
| B-07 弁別の達人 | 「Game 2 ベスト N 点 / あと M 点」 | 「Game 2 ベスト 72 点 / あと 8 点」 |
| B-08 全方位改善 | **「3 ゲーム中 N ゲームが先週比で改善中」** | 「3 ゲーム中 2 ゲームが先週比で改善中」 |

### B-08（全方位改善）のヒント方針

**仕様 F-14（spec.md L225）「未獲得バッジには獲得条件のヒントが表示される」への対応**：

B-08 は「3 ゲームすべて前週比上昇」が獲得条件なので、その判定の途中経過（=どれだけ揃ったか）を **N ゲーム / 3 ゲーム** の形で可視化する。これにより B-02〜B-07 の進捗可視化と粒度が揃う。

**算出ロジック**：
- 各ゲームについて、過去 7 日間の平均閾値（または日次ベスト平均）と、その前 7 日間の平均閾値を比較
- 「改善」= 閾値が小さくなった（角度差が縮まった = 識別力が上がった）
- 過去 7 日間または前 7 日間にデータがないゲームは「データ不足」扱いで判定不能（カウントから除外）

**表示パターン**（S6-03 詳細モーダルでの文言）：

| 状況 | 表示 |
|---|---|
| 3 ゲーム中 0 が改善中 | 「現在改善中のゲームはまだありません」 |
| 3 ゲーム中 1 が改善中 | 「3 ゲーム中 1 ゲームが先週比で改善中」 |
| 3 ゲーム中 2 が改善中 | 「3 ゲーム中 2 ゲームが先週比で改善中」 |
| 3 ゲーム中 3 が改善中（= バッジ獲得条件達成） | 獲得演出に切り替わる（ヒント文言は不要） |
| 一部ゲームでデータ不足（過去 14 日に十分な記録なし） | 「データがもう少し必要です（残り N 日分）」 |

**S6-03 詳細モーダル内での補足表示**（B-08 のときのみ）：

```
  現在の状態：
  Game 1（変化察知）   ↓ 改善中
  Game 2（二重表裏）   ↑ 横ばい／悪化
  Game 3（周辺視野）   ↓ 改善中
  
  3 ゲーム中 2 ゲームが先週比で改善中
```

- 各ゲームに `↓ 改善中` / `→ 横ばい` / `↑ 悪化` / `─ データ不足` のラベルを表示
- これにより、ユーザーは「どのゲームを次に強化すべきか」が観察できる（CVR/継続性向上）

**a11y（B-08 詳細モーダル）**：
- SR：「全方位改善バッジ。未獲得。獲得条件は 3 ゲームすべて前週比上昇。現在 Game 1 改善中、Game 2 横ばい、Game 3 改善中、3 ゲーム中 2 ゲームが先週比で改善中」

## 6. S6-05：バッジ獲得演出（完成版）

Sprint 5 の暫定実装を完成形に。

### 演出シーケンス（1.5 秒以内）

| 時刻 | 状態 |
|---|---|
| 0ms | 結果サマリ表示完了、バッジ獲得検知 |
| 100ms | バッジカード出現開始（scale 0.6） |
| 100〜600ms | scale(0.6 → 1.05) （`motion.easing.decelerate`） |
| 600〜800ms | scale(1.05 → 1.0)（弾性下げ、overshoot 5%） |
| 800〜1500ms | 静止表示、下に「獲得！」テキスト表示 |
| 1500ms 以降 | 通常状態、ユーザー次へ可 |

- 効果音：設定 ON 時のみ短いチャイム（1 音、0.5 秒以内）
- ハプティクス：設定 ON 時のみ軽い振動（短い 1 回）
- **点滅・フラッシュ・色振動なし**（NF-11 / OPT-9）

### 複数バッジ同時獲得時
- 順次表示（並列ではなく、1.5 秒ずつ）
- 最大 3 個までなら順次、4 個以上なら「+N 個獲得」とまとめ表示（極めて稀ケース）

### a11y
- 獲得時 `aria-live="assertive"` で「100 試行バッジを獲得しました」
- 演出中も「次へ」ボタンは操作可（演出を遮断したいユーザーへの配慮）

## 7. S6-06：日次ベスト永続化と表示

### 仕様（spec.md §9.3）
- 各ゲームの「閾値スコア」のうち、その日の最良値（最も低い角度差 = 最も難しい識別ができた）を保持
- 単体プレイのスコアもベスト判定に含む

### データモデル（spec.md §12.1）
- `DailyStats` テーブル：`game1BestThreshold` / `game2BestThreshold` / `game3BestThreshold`
- 1 日 1 レコード（端末ローカル日付ベース）
- セッション完了時に該当日の DailyStats を更新

### ベストの判定ロジック
- ゲーム別閾値（角度差）が**現在より小さい**場合に更新
- 例：今日の Game 2 ベストが 4.2° → セッション後 4.0° → 更新（4.0°）
- 大きいベストでは更新しない

### 表示更新
- ホーム画面では暫定で表示しない（コンパクト保持）
- 進捗グラフの「日次ベスト」タブ S5-05 で表示
- セッション完了画面では「今日のベスト更新」と表示（追加機能、任意）

## 8. ストリーク 0:00 跨ぎ判定の詳細

### 仕様
- 0:00（端末ローカル時刻）跨ぎで前日コース未完了 → 0 にリセット
- 当日完了済みの状態で 0:00 を跨ぐ → ストリーク維持

### 判定タイミング
- アプリ起動時 / ホーム表示時に判定
- バックグラウンド復帰時にも判定（forground イベント）

### 判定ロジック（疑似コード）
```
const today = localDate(now());
const lastCompleted = Streak.lastCompletedDate;

if (lastCompleted == null) {
  // ストリーク 0、変化なし
} else if (lastCompleted == today) {
  // 今日完了済み、ストリーク維持
} else if (lastCompleted == yesterday(today)) {
  // 昨日完了で今日未完了。22 時前なら維持、22 時以降なら警告
  if (now().hour >= 22) {
    showResetWarning = true;
  }
} else {
  // 2 日以上前 → リセット
  Streak.currentStreak = 0;
  Streak.lastCompletedDate = null;
}
```

### タイムゾーン考慮
- 端末タイムゾーンを基準（仕様 A-6）
- 旅行などで時刻変更があった場合：基本的にタイムゾーン変更は尊重するが、悪用防止として「明らかな時刻巻き戻し」は無視（端末時刻 < lastUpdate なら無視）

## 10. テスト要件（追加分）

1. ストリーク 0:00 跨ぎロジック：前日完了→当日完了で +1
2. 前日未完了→翌日 0:00 跨ぎでリセット
3. 当日完了済み状態で 0:00 跨ぎ → 維持
4. B-01〜B-08 各バッジが条件通り発火
5. 同じバッジは 2 回獲得しない（既獲得は再発火しない）
6. バッジ獲得演出が 1.5 秒で完了
7. 22 時以降未完了でリセット警告表示
8. 日次ベスト：今日の Game 2 が 4.0° → DailyStats.game2BestThreshold = 4.0
9. 100 試行達成で B-05 獲得
10. 全 3 ゲーム前週比上昇で B-08 獲得
11. B-08 詳細モーダル：3 ゲーム中 2 ゲーム改善時に「3 ゲーム中 2 ゲームが先週比で改善中」が観察可能（DOM テキスト一致）
12. B-08 詳細モーダル：データ不足ゲームを含む場合「データがもう少し必要です」表示
