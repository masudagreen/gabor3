## バッジ定義（spec §5、3 軸 11 個）

高難度の閾値は system §9.2 を採用（a ≤ 3 °/sec が「遅い」、b ≤ 0.10 hz/sec が「小さい」、両方最難 = a≤3 & b≤0.10、クリア=FP0 & FN0 の満点セッション）。

| ID | 名称 | 軸 | 獲得条件（観察可能） | 未獲得ヒント |
|---|---|---|---|---|
| B-01 | はじめの一歩 | 継続 | 初回セッション完了 | 初めてのセッションを完了すると獲得 |
| B-02 | 三日坊主突破 | 継続 | 3 日連続プレイ | 3 日連続でプレイすると獲得 |
| B-03 | 一週間の習慣 | 継続 | 7 日連続プレイ | 7 日連続でプレイすると獲得 |
| B-04 | 二週間マスター | 継続 | 14 日連続プレイ | 14 日連続でプレイすると獲得 |
| B-05 | 一ヶ月の継続 | 継続 | 30 日連続プレイ | 30 日連続でプレイすると獲得 |
| B-06 | スロー回転ハンター | 高難度 | 回転速度 a が遅い域（≤3°/秒）で変化パッチを全問正答したセッション | 回転速度を遅く設定して全問正解すると獲得 |
| B-07 | 微差を見抜く目 | 高難度 | 周波数変化 b が小さい域（≤0.10）で全問正答 | 周波数変化を小さく設定して全問正解すると獲得 |
| B-08 | 達人の眼 | 高難度 | a も b も最難域（a≤3 かつ b≤0.10）で高スコア | 最も難しい設定で高スコアを出すと獲得 |
| B-09 | 好調のしるし | 高スコア | セッションスコア 80 以上 | スコア 80 以上を達成すると獲得 |
| B-10 | パーフェクト | 高スコア | セッションスコア 100 | スコア 100（全ラウンド満点）を達成すると獲得 |
| B-11 | 安定の高得点 | 高スコア | スコア 80 以上を累計 5 セッション | スコア 80 以上をあと {n} 回で獲得 |

> 高難度バッジの判定は `SessionRecord.paramsSnapshot`（n/m/r/a/b/scoringMode）+ そのセッションの満点性で評価（§6.4）。旧ゲーム依存バッジは一切存在しない（spec §5.4）。

## 画面 S8-1：バッジ一覧（履歴タブ下部、F-09 バッジ部）

履歴タブ（S7-1）の下に続くセクション。グリッド配置（スマホ 2 列 / PC 3〜4 列）。

### レイアウト
```
バッジ                              ← font.h3 26px
┌──────────────┬──────────────┐
│  🏅          │  🏅          │ ← BG-1 BadgeCell（獲得＝フルカラー）
│ はじめの一歩  │ 三日坊主突破  │   名称 font.body.bold 24px
│ 2026-05-24   │ 2026-05-26   │   獲得日 caption 20px
├──────────────┼──────────────┤
│  🔒(灰)      │  🔒(灰)      │ ← 未獲得＝グレースケール + 🔒（形で区別）
│ 一週間の習慣  │ 達人の眼      │
│ 7 日連続で    │ 最も難しい    │   ヒント font.body 24px
│ プレイすると  │ 設定で高得点  │
│ 獲得          │ で獲得        │
└──────────────┴──────────────┘
```

### 規範（§5.4/F-09）
- 全 11 バッジを一覧表示。獲得＝フルカラー + 獲得日、未獲得＝グレースケール + 🔒 + 条件ヒント（NF-12：色＋鍵アイコンで区別）。
- セルタップで詳細（条件全文）をモーダル/展開。
- 点滅なし（NF-11）。
- B-11 のヒントは動的「あと {n} 回」（残り回数）。

### 状態
- 全未獲得（初期）：全セル 🔒 + ヒント。
- 一部獲得：混在。

### a11y
- 各 `BadgeCell` に aria-label「{名称}、{獲得済み（{日付}） / 未獲得：{ヒント}}」。
- グリッドは `role="list"`、各セル `role="listitem"`。

## 演出 S8-2：バッジ獲得（§5.4、`BG-2 BadgeAwardToast`）

セッション結果カード（S6-3）上層で**獲得時に 1 度だけ**。

### 規範
- `motion.duration.badge` 1500ms、拡大 + フェードのみ（**点滅なし**、reduced-motion 時は静的に 1500ms 表示）。
- バッジアイコン 🏅 + 名称 `font.h2` 30px。
- 音（badge.mp3）+ ハプティクス（heavy+medium、F-14/S9、設定で個別 OFF 可）。
- 「もう一度」ボタンと干渉しない位置（中央上）。
- 複数同時獲得時：順番に 1 個ずつ、または 1 つにまとめて表示（Generator 判断、合計時間が長くなりすぎない）。

### a11y
- 獲得を `aria-live="polite"` で読み上げ（「バッジ獲得：はじめの一歩」）。
