## バッジ定義（spec §6、3 軸 11 個）

閾値は system §9.3 を採用（高難度＝振動/遅い回転速度 rotationSpeed≤3/最難寄り、高レベル＝最高到達レベルの絶対値・割合）。デフォルト 720 レベル前提の Generator 仮置き可。範囲設定で総レベル数が変わる場合は割合ベース判定（Generator 判断）。

| ID | 名称 | 軸 | 獲得条件（観察可能、spec §6） | 未獲得ヒント |
|---|---|---|---|---|
| B-01 | はじめの一歩 | 継続 | 初回ゲーム完了（クリア/失敗いずれか） | 初めてゲームを完了すると獲得 |
| B-02 | 三日坊主突破 | 継続 | 3 日連続プレイ | 3 日連続でプレイすると獲得 |
| B-03 | 一週間の習慣 | 継続 | 7 日連続プレイ | 7 日連続でプレイすると獲得 |
| B-04 | 二週間マスター | 継続 | 14 日連続プレイ | 14 日連続でプレイすると獲得 |
| B-05 | 一ヶ月の継続 | 継続 | 30 日連続プレイ | 30 日連続でプレイすると獲得 |
| B-06 | 振動を見抜く目 | 高難度到達 | 回転方向が「振動」のレベルをクリア | 振動するパッチのレベルをクリアすると獲得 |
| B-07 | スロー回転ハンター | 高難度到達 | 回転速度が「遅い」域（rotationSpeed≤3）のレベルをクリア | 回転がゆっくりなレベルをクリアすると獲得 |
| B-08 | 達人の眼 | 高難度到達 | 最難域（個数4・4x4・振動・速度≤2.5 等が揃う高難度）のレベルをクリア | 最も難しいレベルをクリアすると獲得 |
| B-09 | 二桁の壁 | 高レベル到達 | 最高到達レベル ≥ 10 | 最高到達レベルが 10 に達すると獲得 |
| B-10 | 熟達者 | 高レベル到達 | 最高到達レベルが「中盤以上」（提案：総レベル数の 50%、デフォルト 720 なら ≥360） | 最高到達レベルが中盤に達すると獲得 |
| B-11 | 頂を目指して | 高レベル到達 | 最高到達レベルが「終盤」（提案：総レベル数の 85%、デフォルト 720 なら ≥612） | 最高到達レベルが終盤に達すると獲得 |

> **判定の入力**：高難度（B-06/07/08）は `GameRecord.levelParams`（クリアしたレベルの direction/rotationSpeed/count/gridSize）+ `result='clear'`。高レベル（B-09/10/11）は `LevelState.highestLevel`。継続（B-01〜05）は `Streak.currentStreak` / `PlayStats.totalGames`。**旧スコア依存バッジ（0〜100）・b 依存バッジは一切存在しない**（spec §6.4）。

## 画面 S9-1：バッジ一覧（履歴タブ下部、F-09 バッジ部）

履歴タブ（S8-1）の下に続くセクション。グリッド配置（スマホ 2 列 / PC 3〜4 列）。

### レイアウト
```
バッジ                              ← font.h3 26px
┌──────────────┬──────────────┐
│  🏅          │  🏅          │ ← BG-1 BadgeCell（獲得＝フルカラー）
│ はじめの一歩  │ 三日坊主突破  │   名称 font.body.bold 24px

│  🔒(灰)      │  🔒(灰)      │ ← 未獲得＝グレースケール + 🔒（形で区別）
│ 二桁の壁      │ 達人の眼      │
│ 最高到達      │ 最も難しい    │   ヒント font.body 24px

│ 達すると獲得  │ クリアで獲得  │
└──────────────┴──────────────┘
```

### 規範（§6.4/F-09）
- 全 11 バッジを一覧表示。獲得＝フルカラー + 獲得日、未獲得＝グレースケール + 🔒 + 条件ヒント（NF-12：色＋鍵アイコンで区別）。
- セルタップで詳細（条件全文）をモーダル/展開。
- 点滅なし（NF-11）。
- 高レベル軸のヒントは到達レベル基準の文言（「最高到達レベルが {threshold} に達すると獲得」）。割合判定の場合も観察可能な「あといくつ」表現が望ましい（Generator 判断）。

### 状態
- 全未獲得（初期）：全セル 🔒 + ヒント。
- 一部獲得：混在。

### a11y
- 各 `BadgeCell` に aria-label「{名称}、{獲得済み（{日付}） / 未獲得：{ヒント}}」。
- グリッドは `role="list"`、各セル `role="listitem"`。

## 演出 S9-2：バッジ獲得（§6.4、`BG-2 BadgeAwardToast`）

ホーム結果カード（S7-3）上層で**獲得時に 1 度だけ**。

### 規範
- `motion.duration.badge` 1500ms、拡大 + フェードのみ（**点滅なし**、reduced-motion 時は静的に 1500ms 表示）。
- バッジアイコン 🏅 + 名称 `font.h2` 30px。
- 音（badge.mp3）+ ハプティクス（heavy+medium、F-14/S10、設定で個別 OFF 可）。
- 「もう一度」ボタンと干渉しない位置（中央上）。
- レベルアップ告知（LD-1）・レベルアップ音とタイミングが重なる場合は短い間隔で連続（Generator 判断、合計時間が長くなりすぎない）。
- 複数同時獲得時：順番に 1 個ずつ、または 1 つにまとめて表示（Generator 判断）。

### a11y
- 獲得を `aria-live="polite"` で読み上げ（「バッジ獲得：はじめの一歩」）。
