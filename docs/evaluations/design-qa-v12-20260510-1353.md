## 発見した問題（重要度順）

### [Critical 1] system.md §1.11.1 / §1.19 規範違反：カウントダウン danger 色のコントラスト不足
- **計測値**：`color.countdown.danger = #FF4D4D` を `color.bg.surface = #1A1D24` に置いたとき、コントラスト比 = **5.16:1**
- **規範値**：system.md §1.11.1 で「`color.countdown.normal/.warn/.danger` は背景 #1A1D24 / surface 上で **7:1 以上**」と明記
- **影響箇所**：02-countdown-pattern.html、03〜06 ゲームプレイの GameStatusBarV12、08-result-overlay.html、01-launch-distance.html（最初から赤）
- **再現**：`getComputedStyle(.countdown-xl.danger).color → rgb(255,77,77)` を WCAG 計算
- **修正方向**：danger を `#E63232` 程度に下げる（暗くする）か、ダーク時のみ `#FF7878` バリアントを正規採用に。system.md の規範自体が「7:1 を担保」と明言しているのでトークン値を見直すか規範を「AA 4.5:1」に緩めるかの二択

### [Critical 2] components.md §28 規範違反：ResultBadge success の白文字コントラスト 3.07:1
- **計測値**：white on `#22A862` (success bg) = **3.07:1**
- **規範値**：components.md §28「`color.result.aggregate.success` … コントラスト **7:1 確保**」
- **WCAG**：AA テキスト基準 4.5:1 すら未達（AA Large 3:1 はギリギリ通過）
- **影響箇所**：08-result-overlay.html 試行全体総合 ✓、07-postsession.html の各ゲーム行の ✓ アイコン
- **修正方向**：success 背景色を暗緑寄り（例 `#0F7A45` で 7:1+）に、もしくはテキストアイコン色を黒に

### [Critical 3] system.md §1.12 規範違反：ガボールクリッピングが円形マスク採用
- **規範**：system.md §1.12.1 / §1.12.3「**矩形枠でクリッピング**」「**円形マスクではなく矩形マスクを採用**」「実サイズ N=1.5 倍で生成」
- **実装**：03-g01-rotation.html / 04-g04-3x3-grid.html / 05-g07-4x4-grid.html / 08-result-overlay.html で `<circle cx="50" cy="50" r="50">` の**円形クリッピング** + N=1.0
- **影響**：規範のリファレンスとしての mockup が「実装が真似してはならない方式」を示している。Generator が mockup を見て実装すると NF-27 / NF-28 違反になる
- **目視影響**：cell 背景が #808080 で circle 内ガボール背景も #808080 のため**実害は出ていない**が、規範のリファレンスとしては誤り
- **修正方向**：SVG `<rect>` ベースに切り替え、`<g>` 内で 1.5 倍サイズ（viewBox 0 0 100 100 のまま実描画 150% でクリップ）にする

### [Major 1] レスポンシブ破綻：全 mockup で 360px viewport overflowX = 17px
- **計測**：mockup の `.device { width: 375px + border 1px*2 = 377px }` 固定 → 360px viewport で 17px はみ出し（横スクロール）
- **規範**：CLAUDE.md「360 / 375 / 1280 px の 3 ブレイクポイント」「`scrollWidth - innerWidth = 0`」
- **影響**：360px Android 端末（小さめ）で横スクロールが必須化、UX 破綻
- **修正方向**：`.device` を `width: min(375px, 100vw)` または `max-width: 375px; width: 100%` に

### [Major 2] レスポンシブ破綻：07-postsession.html 375px viewport で overflowX = 34px
- **計測**：375 viewport で `.device width = 409px`（375 + padding 16*2 + border 1*2）
- **影響**：標準 iPhone SE / iPhone 13 mini 幅 375 で横スクロール
- **修正方向**：`.device` から padding を除去して内側に padding を持たせる

### [Major 3] a11y 実体不在：03-g01-rotation.html / 04-g04-3x3-grid.html / 05-g07-4x4-grid.html の cell に role / aria-label / data-target-id 不整合
- **screens.md §3.5 / §5.5 / §6.4** で「ImageChoiceCell に `role="checkbox"`, `aria-checked`」「各セルに `data-target-id`」「`aria-label`：ゲーム別に動的生成」と明記
- **実装**：
  - 03-g01-rotation.html cell：data-target-id ✓ あり、role ❌ なし、aria-label ❌ なし、aria-checked ❌ なし
  - 04-g04-3x3-grid.html cell：data-target-id ❌ なし、role ❌ なし、aria-label ❌ なし
  - 05-g07-4x4-grid.html cell：data-target-id ❌ なし、role ❌ なし、aria-label ❌ なし
- **影響**：SR ユーザーに「グリッド内のどのセルか」が伝達不能。Generator がコピーするとアプリも同様に SR 不適合
- **修正方向**：各 cell に `role="checkbox" aria-checked="false" aria-label="グリッド X 行目 Y 列目のガボール" data-target-id="g04-cell-X-Y"` を完備

### [Major 4] a11y 実体不在：07-postsession.html の 7 行中 5 行で aria-label が null
- **screens.md §8.3** で「各ゲーム行は `role="listitem"`、`aria-label="G-01 回転検出 正解 閾値 3.5 度毎秒"`」必須
- **実装**：G-01 / G-03 行のみ aria-label 完備、G-04 / G-05 / G-06 / G-07 / G-13 の 5 行は **aria-label = null**
- **影響**：SR で「G-04 コントラスト 0.18」しか読まれず、✓/✕ が badge の `aria-hidden="true"` のため読み上げ皆無
- **修正方向**：5 行すべてに同フォーマットの aria-label を付与

### [Major 5] トークン整合性違反：action.primary が #2C7CFF（system.md は #13449D）
- **system.md §1.2 / components.md §9** で `color.action.primary = #13449D`（v1 から継承）
- **mockup**：01-launch-distance.html / 07-postsession.html / 08-result-overlay.html で `--action-primary: #2C7CFF`（明るめの青、別色）
- **影響**：Generator がどちらを採用すべきか不明。トークン規範のソース・オブ・トゥルースが揺れる
- **修正方向**：mockup の `--action-primary` を `#13449D` に揃えるか、system.md に v1.2 で青系を明るくする旨を明記してトークン値を更新

### [Major 6] mockup 実体不在：26 定義画面中 18 画面が mockup 化されていない
- **存在**：S22-LAUNCH(01)・S22-DISTANCE(01)・S22-G01-PLAY(03)・S22-G04/05/06-PLAY(04)・S22-G07-PLAY(05)・S22-G13-PLAY(06)・S22-POSTSESSION(07)・S22-COUNTDOWN-PATTERN(02)・S22-RESULT-OVERLAY-PATTERN(08)
- **不在**：S22-G03-PLAY、S22-G03-RESULT、S22-G13-RESULT、S22-COOLDOWN、S22-PROGRESS、S22-BADGES、S22-SETTINGS、S22-DATA-RESET、S22-APPICON、S22-OB-DIFF、各 Result（G-04/G-05/G-06/G-07/G-13 個別）
- **影響**：とくに S22-G03-PLAY（中央線方式廃止が改訂の核）と S22-DATA-RESET（v1.2 文言改訂）は実機検証不能。S22-COOLDOWN（カウントダウン UI 統一の重要適用箇所）も不在
- **判断**：Designer 判断で「差分のみ画面は省略」「テンプレートで代替」は許容するが、screens.md §0 でテーブルに「新規」「改訂」と書かれた画面は mockup を要する。少なくとも **S22-G03-PLAY / S22-DATA-RESET / S22-COOLDOWN の 3 画面**は追加してほしい
- **修正方向**：S22-G03-PLAY（中央線無し版）+ S22-COOLDOWN（CountdownTimer xl 適用）+ S22-DATA-RESET（v1.2 文言）の 3 mockup を追加

### [Major 7] 02-countdown-pattern.html が 360px viewport で overflowX = 419px（3 列固定）
- **計測**：`.grid { grid-template-columns: repeat(3, 1fr); max-width: 1100px }` を 360px に放り込むと 1 列に折りたたまれず横スクロール
- **影響**：規範ドキュメントとして「3 段階比較」を見せる用途だが、モバイルでアクセス不能
- **修正方向**：`@media (max-width: 600px) { .grid { grid-template-columns: 1fr; } }` を追加

### [Warn 1] G-13 の 70/30 比率が実測 67/33（±3% 誤差）
- **計測**：stim 568.4px / keypad 275.6px → 67.1% / 32.9%
- **規範**：screens.md §7.5「上 **70%** が刺激領域、下 **30%** が keypad」
- **誤差要因**：keypad が flex の上に padding 16px を持つため肥大。flex 設定 70/30 でも実描画は 67/33
- **修正方向**：keypad の padding を含めて 30% に収めるか、`flex-basis` を 70%/30% から 72%/28% 程度に微調整

### [Warn 2] 07-postsession.html PC 横で「2 列グリッド展開」が実装されていない
- **規範**：screens.md §8.2 / system.md §1.10.3「PC 横 1280：7 ゲーム結果は 2 列グリッド（4 / 3 行）に展開」
- **実装**：1 列固定（`.device max-width 375` 相当）
- **影響**：PC 横で縦長スクロール。デスクトップでスペース効率が悪い
- **修正方向**：`@media (min-width: 720px) { .device { max-width: 720px; } .game-row { …2 列グリッド… } }`

### [Warn 3] 02-countdown-pattern.html の超大型カウントダウン（144px）はサイズプリセット xl=72px と乖離
- **規範**：system.md §1.11 サイズプリセット xl = 72px
- **実装**：`.countdown-xl { font-size: 144px }`（カードに合わせた強調表示用）
- **影響**：規範を学ぶ初心者が「xl=144px か」と誤読する可能性
- **修正方向**：cards のフォントサイズコメントに「これは規範値 72px ではなくサンプル強調用 144px」と注釈、もしくはサンプル値も 72px に統一

### [Minor 1] 06-g13-70-30-layout.html の keypad に aria-label 不在
- **screens.md §7.6** で「`aria-label`：「数字 1」「数字 2」...」と明記
- **実装**：button の text content が "1"〜"0" なので SR では読めるが、aria-label 規範未充足
- **修正方向**：各 button に `aria-label="数字 1"` 等を付与
