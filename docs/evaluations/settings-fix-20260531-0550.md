## パラメータ刻み所見
- 回転速度 a：6 → 6.1 → 6.2 → 6.3（＋連打）、6.3 → … 減算も追従。**0.1 刻み**で正確。
  表示は「6.1°/秒」等きれいな小数で、`6.299999…` のような浮動小数ゴミは出ない

## 発見した問題（重要度順）

### [Minor] 周波数変化速度(b) の小数表示桁が可変（0.155 と 0.16 が混在）
- 再現手順: 設定 → 周波数変化速度 ＋ を 2 回（0.15→0.155→0.16）
- 実際の表示: 1 回目「0.155hz/秒」、2 回目「0.16hz/秒」（末尾 0 が落ちて桁数が揺れる）
- 期待（望ましい）: 「0.160hz/秒」のように小数 3 桁固定表示で桁を揃えると、刻み（0.005）が
  視覚的に一貫して読み取りやすい。
- 補足: 値自体は数学的に正確（内部 0.16）で、float ゴミではない。刻み機能・保存・採点には影響なし。
  あくまで表示の体裁。閾値（崩れ・誤刻み）には**抵触しないため合否は合格**。
- 該当箇所のヒント: `src/screens/v2/SettingsScreen.tsx` の sfChangeSpeed Slider の
  `format={(v) => `${v}${t('settings.unit_hz_per_sec')}`}`（同様に rotationSpeed も `${v}` 直書き）。
- 修正の方向性（任意）: format で step に応じた固定桁にする（例 `v.toFixed(3)` / a は `toFixed(1)`）。
  Slider 側の `roundStep` が既に小数桁を保持しているため、format だけの軽微変更で揃う。

## 良かった点
- content の `width:'100%' + maxWidth:720` 併用が、PC では中央 720px キャップ、モバイルでは全幅、と
  両狙いを正しく満たしている。狭幅（360）でも一切はみ出さない堅牢な実装。
- Slider の `roundStep`（toFixed + min 基準の格子スナップ）が浮動小数の累積誤差を抑え、
  0.1 / 0.005 の細刻みでも「6.3」「0.155」とクリーンな値を保っている。
- F-13 の即時保存が効いており、リロード後も UI/ストレージ両方で設定が保持される。
