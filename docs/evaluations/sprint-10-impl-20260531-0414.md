## 発見した問題（重要度順）

### [Major] Skip link が Enter キーで起動しない（NF-14 / NF-9 違反）
- 再現手順:
  1. オンボーディング完了後の AppRoot（任意タブ）でページをリロード
  2. 最初に Tab を 1 回押す → 「メインコンテンツへスキップ」が左上(8,8)に可視化され、focus が当たる（ここまでは正常）
  3. Enter を押す
- 期待される振る舞い: NF-14 のとおりメインコンテンツ（`#ge-main-content`）へ focus が移り、タブバー等の前段を読み飛ばせる（NF-9「Enter/Space で起動」）
- 実際の振る舞い: focus が skip link に留まったまま動かない。`#ge-main-content` に `tabindex` も付与されず（`focusMain()` が発火していない）、focus 移動が起きない。
  - 切り分け実測：同要素に `click` を dispatch すると `focusMain()` が正しく動作し `#ge-main-content` に `tabindex=-1` 付与＋focus 移動が成立する（＝ハンドラ自体は正常）。**Enter キーでハンドラが呼ばれないことが原因**。
- 該当箇所のヒント:
  - `src/components/v2/SkipLink.tsx` — `Pressable` に `accessibilityRole="link"` / `role: 'link'` を付与。RNW は `<div role="link">` を生成し、button 以外の role に対して Enter/Space のキーボード活性化を合成しないため `onPress` が Enter で発火しない
- 修正の方向性（いずれか）:
  - (a) `role="button"`（accessibilityRole="button"）に変更する。RNW は button role に Enter/Space 活性化を合成する。視覚はリンク風のままで可
  - (b) Web で `onKeyDown` を追加し、`Enter`/`Space` で `focusMain()` を呼ぶ（`e.preventDefault()` 付き）
  - (c) 実際の `<a href="#ge-main-content">` 相当にして native anchor のキーボード挙動に委ねる
  - 修正後は Playwright で「Tab→Enter で `document.activeElement===#ge-main-content`（または内部の最初の focusable）」を必ず再確認すること
