## 発見した問題

### [Minor] SideBySideStimulus（GE-02 流用）の aria-label 文面が G-02 文脈のまま（既知・self-review §6.2 と一致）

**現象**：
G-04 プレイ画面の左右ガボールパッチに付与された aria-label が `「左の縞模様（タップで「左が時計回り」と回答）」` / `「右の縞模様（タップで「右が時計回り」と回答）」` のまま。SideBySideStimulus（GE-02）を構造的型互換で再利用しているため、内部の固定 aria-label 文面が G-02 由来。同じ理由で stimulus radiogroup の aria-label も `「左右の縞模様（時計回りに傾いている方を選んでください）」` で G-04 文脈と乖離。

**評価**：
- spec-v11.md §7.4 / screens.md S12-* の受け入れ基準に aria-label 文面は含まれない
- 主要回答経路（AnswerChoiceGroup の「左が濃い」「右が濃い」ボタン）の radiogroup aria-label は `「どちらのコントラストが高いか」` で正しく screens.md S12-02 §4 と一致
- 状態管理（aria-checked 動的更新、左右切替、複数選択不可）は完全に正しく機能
- self-review §6.2 で本問題を明記し、Sprint 13 で `SideBySideStimulus` に `leftAriaLabel` / `rightAriaLabel` optional prop を追加して G-02 / G-04 / G-05 すべてが文脈に応じた文面を渡せるようにする**統一改修プランが既に提示されている**
- design-qa の「aria-label の正解漏洩」（Critical）には該当しない（正解情報は漏れていない、文面が単に不正確なだけ）

**再現手順**：
1. ホーム → 単体プレイ → G-04 → はじめる → 距離リマインド経由でプレイ画面
2. DevTools で `document.querySelector('[data-testid="g02-stimulus-left"]').getAttribute('aria-label')` を実行
3. **期待値**：G-04 文脈の文面（例：「左の縞模様（タップで『左が濃い』と回答）」）
4. **実測値**：「左の縞模様（タップで『左が時計回り』と回答）」

**修正の方向性**：
self-review §6.2 / §7「Sprint 13 申し送り」に従い、Sprint 13 着手時に `SideBySideStimulus` に optional prop `leftAriaLabel` / `rightAriaLabel` / `groupAriaLabel` を追加し、G-02 / G-04 / G-05 の各呼び出しでゲーム文脈に合わせた文面を渡す（破壊的変更なし、デフォルトは GE-02 現行文面維持）。本スプリントでの修正は不要、Sprint 13 で同時改修推奨。
