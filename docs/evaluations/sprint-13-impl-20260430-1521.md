## 発見した問題

### [Minor] SideBySideStimulus 内部の `testID` が `g02-stimulus-${side}` でハードコード

**該当箇所**：`src/components/v11/games/SideBySideStimulus.tsx` L198, L208

呼び出し側で `testId="g05-stimulus"` を渡しても、内部の Animated.View / ImageChoiceCell の testID は `g02-stimulus-left` / `g02-stimulus-right` になっている（GE-02 由来の名残）。

**ユーザー影響**：なし。aria-label / aria-checked / 描画はすべて正常で、ユーザー操作に影響しない。Playwright や CI から「G-02 のテスト ID で G-05 を捕まえる」混乱があり得る。

**修正の方向性**：`testID={\`${testId ?? 'side-by-side-stimulus'}-${side}-anim\`}` のように propで受け取った `testId` を prefix にする。既存の `tests/v11/components/games/SideBySideStimulus.test.tsx` などが `g02-stimulus-${side}` セレクタに依存している場合、テスト側の修正も必要なため、今すぐ直すか Sprint 14 で `g02-stimulus-` 命名を一括リネームするかは Generator 判断でよい。**合否に影響しないため Minor**。
