## 発見した問題

### [Minor 1] ワイドスコアグラフの Y 軸 0 ラベルと X 軸 4/3 ラベルが視覚的に重なる
- 再現手順: 進捗グラフ画面 → ワイドスコアタブ → グラフ左下を観察
- 期待される振る舞い: Y 軸数値「0」と X 軸日付「4/3」が判別できる十分な余白（4px 以上）
- 実際の振る舞い: 「0」と「4/3」が垂直に近接しすぎ、「4/3」の上端と「0」の下端が重なって見える（screenshot: sprint18-progress-wide-seeded.png 参照）
- 該当箇所のヒント: `src/components/v11/charts/LineChart.tsx`（軸ラベル padding / X 軸ラベル `marginTop`）
- 修正の方向性: X 軸ラベル領域の `marginTop` を 4px 以上、または Y 軸ラベル末尾 `paddingBottom` を増やす

### [Minor 2] ゲーム別タブの Y 軸ラベルが幅 56px で「…」切り捨てられる
- 再現手順: 進捗グラフ画面 → ゲーム別タブ → G-01 → Y 軸ラベル「7.7…」「5.5…」「3.2…」
- 期待される振る舞い: 数値全体「7.75°」「5.50°」「3.25°」が読める
- 実際の振る舞い: コンテナ幅 56px / fontSize 24px のため、4 文字「7.75°」が `…` 省略されて「7.7…」と表示される。実体は正しく計算されているが視認できない
- 該当箇所のヒント: `src/components/v11/charts/LineChart.tsx` の Y 軸ラベル width 計算（おそらく定数 56px、ワイドスコア 0〜100 想定で固定値）
- 修正の方向性: Y 軸ラベルの最大文字数に応じた dynamic width、または `numberOfLines={1}` + `adjustsFontSizeToFit` 併用、または fontSize を 22px に微調整して 5 文字収納
- 注: 仕様 §F-11「軸ラベルは 18pt 以上」は満たしている（24px = 18pt）。可読性問題のみ
