## 発見した問題

### [Minor] 設定画面のトグル行に `aria-checked` 属性が web 出力されていない

- 再現手順: 設定画面で「効果音」「振動」のトグル要素 (`role="switch"`) の DOM 属性を確認
- 期待される振る舞い: `role="switch"` に対し WAI-ARIA 仕様で `aria-checked="true"|"false"` が必須
- 実際の振る舞い: `accessibilityState={{ checked }}` を設定しているが React Native Web 0.x の Pressable は `accessibilityState.checked` を `aria-checked` に変換しない（既知のフレームワーク挙動）。`role="switch"` / `aria-label` / `tabindex="0"` だけ出力されている
- 該当箇所のヒント: `src/components/v11/SettingsRow.tsx:131-132`（accessibilityRole="switch", accessibilityState={{ checked }}）
- 影響範囲の限定: 視覚的には「ON / OFF」テキスト + 青い track / thumb の色変化で並列して状態を伝えており（screens.md §11.9「色のみで状態を伝えない」遵守）、スクリーンリーダーも `aria-label="効果音"` を読み上げる。ただし読み上げに ON/OFF 状態が含まれない可能性あり
- 修正の方向性（v1.2 推奨）: SettingsRow のトグル部に `aria-checked` を直接付与する Web 専用 props（Platform.OS === 'web' 時に `accessibilityValue` / `aria-checked` を補う）。または `accessibilityRole='switch'` を維持しつつ `aria-label` に「効果音、ON」のように状態を含める
- 現スプリント不合格に至らない理由: 自己評価レポート §1.6 a11y 11.8 でも「accessibilityState.checked」を実装と明示しており、本問題は RN Web 側の制限。仕様書 §F-14「ON/OFF が一目で判別」は視覚で完全に達成。a11y 規約上の Critical/Major は付かず Minor 扱いとした
