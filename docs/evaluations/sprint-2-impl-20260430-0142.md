## 発見した問題

### [Minor] M-1: Game 3 disabled card に aria-disabled 属性が出力されない
- 観察：DOM 上 `<button role="button" tabindex="-1" aria-label="Game 3 周辺視野ハント。Sprint 3 で実装予定、現在は準備中">` だが `aria-disabled="true"` 属性は付与されていない
- 期待：screens.md S2-01 §「Game 3 disabled」に「aria-disabled=true, tabindex=-1」と書かれている
- 実態：React Native Web は `accessibilityState={{disabled: true}}` を `<View>` から `<button>` 化する際 `aria-disabled` に変換していない（実装は `disabled` prop を渡していないため）
- 影響：tabindex=-1 でキーボード操作除外は実現、aria-label の文言で「準備中」もスクリーンリーダーに伝わるため、機能影響は限定的。視覚 opacity 0.65 + 「準備中」チップでも明示
- 修正の方向性：`HomeScreen.DisabledGameCard` に `aria-disabled="true"` を Web 版で出力する（`Platform.OS === 'web' && {'aria-disabled': true}` で React Native Web に prop 透過させるか、`disabled={true}` を Pressable 互換 prop に変更）。1 行修正で済む。Sprint 7 a11y 仕上げで対応してよい

### [Minor] M-2: Web ダークモード反映が確認できなかった
- 観察：Playwright で `emulateMedia({ colorScheme: 'dark' })` 後にホームを再描画したが、配色がライトモードと同じスクリーンショット
- 期待：spec.md OPT-8「ダークモード必須」、Sprint 7 で本実装予定だが、`useColorScheme()` 連動の地は Sprint 1 から入っている想定
- 実態：`useColorScheme()` が React Native Web で `prefers-color-scheme` メディアクエリを直接 watch していない、もしくはリロード後の取得タイミングの問題。Sprint 7「ダークモード」の実装範囲外の可能性も
- 影響：Sprint 2 のスコープ外（Sprint 7 で本格対応）。Sprint 7 で確認する申し送り
- 修正の方向性：Sprint 7 で `react-native-web` の `useColorScheme` 動作確認 + 必要なら `Appearance.getColorScheme()` リスナー or 設定 UI 経由のスキーム強制切替を実装
