## 発見した問題（重要度順）

### [Minor] 全データ削除 ConfirmDialog の初期フォーカスが危険側（削除する）になっている
- 再現手順：設定 → 全データ削除 → ダイアログ表示直後に `document.activeElement` を確認
- 期待される振る舞い：components.md DG-1「開いたら安全側ボタン（キャンセル）にフォーカス」。破壊的操作のため初期フォーカスは「キャンセル」
- 実際の振る舞い：初期フォーカスが「削除する」（danger）に乗る。Enter 即押下で誤削除のリスク
- 該当箇所：`src/components/v2/ConfirmDialog.tsx`（cancel Pressable に `{...({autoFocus:true} as any)}` を付与しているが RN-web の Pressable では効かず、DOM 先頭の confirm にフォーカスが乗る）
- 修正の方向性：confirm より前に cancel を DOM/フォーカス順で優先するか、`ref` + `useEffect` で `visible` 時に cancel ボタンへ `.focus()` を明示。RZ-1 は単一ボタンのため影響なし

### [Minor] 採点方式ラジオが role=button のままで aria-checked を持たない／Toggle の aria-checked が出力されない
- 再現手順：採点方式 3 行のアクセシビリティツリー、効果音スイッチの aria-checked を確認
- 期待される振る舞い：screens.md S2-1 a11y「採点方式は radiogroup、各行ラジオ」。NF-15「選択状態は aria-checked / aria-pressed で明示」
- 実際の振る舞い：radiogroup コンテナは role=radiogroup だが、各行は role=button（aria-label に「。説明」を連結）で選択は ●/○ グリフのみ。Toggle は role=switch だが aria-checked 属性が DOM に出ていない（accessibilityState.checked が RN-web で aria-checked へマップされず）。選択中行は aria-label に状態語を含まない
- 補足：いずれも視覚（●/○・ノブ位置）と aria-label の ON/OFF 併記で状態は伝わるため実害は小。スクリーンリーダーの厳密な状態通知としては不足
- 該当箇所：`src/screens/v2/SettingsScreen.tsx`（採点ラジオを SettingRow onPress=button で実装）、`src/components/v2/Toggle.tsx`
- 修正の方向性：採点行を各 `accessibilityRole="radio"` + `accessibilityState={{selected}}` に。Toggle は現状の role=switch を維持しつつ aria-checked が出ない RN-web 制約を S10 a11y 仕上げで再確認（Web 限定で aria-checked を明示注入も可）
