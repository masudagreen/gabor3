## 発見した問題

### [Warn-1] F-21 PostSessionScreen の実機到達は未実施
- 現状: 7 ゲーム連続プレイの実走 (各 60 秒、合計 ≈ 7 分 + クールダウン) が時間的に困難なため、評価では中断ルート (PostSessionPlaceholder) と PostSessionScreen の自動テスト 19 アサーション (構造・閾値フォーマット・a11y・3 ボタン動作) でカバレッジを担保
- 期待: 実機で完走 → PostSessionScreen 表示 → ワイドスコア / ストリーク / 7 ゲーム結果リスト / 3 ボタンの目視確認
- 実際: テスト構造は完全だが、目視レビュー未実施
- 該当箇所のヒント: `src/screens/v11/PostSessionScreen.tsx`、`tests/v11/screens/PostSessionScreen.test.tsx`
- 修正の方向性: ユーザーが実機で 1 回フル完走を行い、視覚確認を完了させる (オーケストレーター完了報告で人間レビュー対象として明示推奨)

### [Warn-2] Native (iOS / Android) 音再生未対応 (Generator 申告通り)
- 現状: Web のみ OscillatorNode で音合成、Native では `_soundEnabled && Platform.OS === 'web'` 条件で no-op
- 期待: spec F-19 受け入れ「正解音 / 不正解音 / tick3-1 / gameEnd / badge の各イベントで音再生」は Native でも満たすべき
- 実際: Native はハプティクスのみ動作、音は鳴らない
- 該当箇所のヒント: `src/platform/audio.ts:327` `Platform.OS === 'web'` 条件
- 修正の方向性: 別スプリント (v1.2.x) で expo-audio または expo-av を導入。実機での音再生検証込みで分離されている (リスク管理として妥当)

### [Warn-3] アイコン図柄プロデザイナー差し替え未実施 / 中心配置やや上寄り
- 現状: Pillow で合成したガボールパッチ (45° 角度・暗背景 #1A1D24・コントラスト 0.78・ガウス窓ぼかし) は仕様通りだが、目視で中心がやや上寄りに見える
- 期待: spec F-20「ガボールパッチ 45° (左下→右上、時計回り) に固定」は満たすが、リリース品質には Designer のレビュー / 微調整推奨
- 実際: 仕様準拠だが美的調整の余地あり
- 該当箇所のヒント: `scripts/generate-icons.py`、`assets/icons/app/icon-source.svg`、`assets/icon.png` 他
- 修正の方向性: ユーザー / Designer がアイコンを目視確認 → 必要なら SVG マスターを差し替え → スクリプト再実行。**リリース前に Designer レビュー推奨**

## 良かった点

- `finalizeCourseSession` が「SessionRecord 保存 → DailyStats 更新 → Streak 更新 → バッジ評価 → 新規獲得バッジ返却」を 1 関数で完結させ、AppRouter 側のロジックがシンプル
- 永続化失敗時の段階的フォールバック (storage 失敗 → loadStreak/loadDailyStats で再構築 → さらに失敗 → 最小データ生成) で「最低でも postsession 画面に進める」設計が堅牢
- expo-haptics の動的 require + キャッシュで Native / Web / jest 環境を矛盾なく扱い、`_resetPlatformAudioForTest()` でテスト容易性を確保
- PostSessionScreen の `formatThresholdForPostSession` がゲーム別単位 (°/s, °, ×, plain 2 桁) を Spec の表示例に正確に合わせて分岐
- アイコン生成を Python スクリプトで再現可能にしてあり、Designer 版差し替え時に同じ派生サイズを 1 コマンドで生成可能
- G-13 paramRange 切替に伴う既存テスト 4 ファイルの値更新が漏れなく反映され、テスト 1570 件すべて PASS
- 中断 → PostSessionPlaceholder ルートの確認ダイアログが「ゲーム中断 → セッション中断」の 2 段階で誤操作防止 (UX 配慮)
- 設定の独立トグルが即座に AsyncStorage に永続化され、UI 表示も即時反映 (副作用ループなし)
