## 発見した問題

### [Minor] ライトモードで ClockAnswerButtons の枠線コントラストがやや薄い
- 1280px ライトモードのスクリーンショットで、ボタン背景 (actionSecondary) と canvas (bgCanvas) がほぼ同色のため、ボタン枠 (borderDefault 1px) があっても**輪郭の視認性が低い**。本文 24px Medium は判読可能だが、ボタンの「押せる領域」境界がぼけ気味
- 仕様/デザインに反するわけではない（components.md §19 の border 1px 要件は満たす）が、ユーザー体験の観点で枠を 1.5px or borderDefault のコントラストを少し強める検討余地あり
- Sprint 7 の a11y 仕上げ時に WCAG AA 1.4.11（非テキストコントラスト 3:1）チェックを推奨
- 修正方向：`getColors().actionSecondary` の light 値か `borderDefault` の light 値を再点検

### [Minor] feedback 矢印が transformOrigin を使うが web のみ対応
- `PeripheralLayout.tsx` CorrectArrow の `transformOrigin: '0% 50%'` は React Native Web では効くが、native（iOS/Android）では transform 配列での座標補正に依存
- 当面 web 専用なので問題なし（spec §6.1 でデバイスは Web ブラウザ前提）。native ターゲット時は再検証
- 修正方向：将来的に react-native-svg 導入時に SVG 化（自己レビュー §5.1 と整合）
