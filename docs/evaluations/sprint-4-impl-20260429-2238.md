### Minor

#### [Minor] AgeGroupScreen のラジオ選択状態が常に false で固定
場所: `src/screens/Onboarding/AgeGroupScreen.tsx` L93
```tsx
accessibilityState={{ selected: false, checked: false }}
```
カードはタップ即遷移する設計のため画面上では選択中状態を保持する必要がない。しかし `accessibilityState` を常に false で渡すと、SR 読み上げ時に「選択されていません」と毎回読み上げられる可能性がある。タップ前後で値を保持しないなら `accessibilityState` 自体を渡さず `accessibilityRole="button"` のままにするか、押下時の momentary highlight を入れる方が SR 体験は自然。観察可能挙動には影響しないので **Minor**。

修正の方向性：
- 即遷移なら `accessibilityRole="button"` に変更（screens.md は radio を要求しているが momentary なため button でも仕様矛盾は小さい）
- もしくは押下中の `pressed` 状態を `accessibilityState.selected` に流す

#### [Minor] DisclaimerScreen の `bypassScrollGate` がテスト用フックとして残存
場所: `src/components/DisclaimerSheet.tsx` L75-91
- 「テスト用：本文スクロールを強制バイパス」の prop が export 型に含まれている
- 実機では使われないが API 表面に残ると将来的に「同意ゲートを skip するバイパス」と誤読される可能性
- リリースビルドでは設定すべきでないことを TS doc に明記する or `__test_bypassScrollGate` 等の名前にする
- セキュリティ要件 NF-21（ログ出力なし）に直接違反するわけではないが、API 衛生面で **Minor**
