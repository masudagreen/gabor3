## 発見した問題（重要度順）

### [Major-1] V1ScoreChart：Y 軸ラベル「100」が縦に折り返して表示される

- **再現手順**：375 / 360 / 1280 すべての viewport で進捗グラフを開く
- **期待される振る舞い**：Y 軸の `0 / 25 / 50 / 75 / 100` がすべて 1 行で読める
- **実際の振る舞い**：`100` だけ `10` と `0` に縦折り返し（fontSize=24px、ラベル width=40px の StyleSheet `yLabel` で 3 桁が収まらない）
- **該当箇所**：`src/components/V1ScoreChart.tsx` の `yLabel` スタイル（width=40 / textAlign='right'）。CHART_PADDING_LEFT=48 なので width=40 は最大文字数 2 桁前提
- **修正の方向性**：`yLabel` の width を 44〜48px に拡張、または CHART_PADDING_LEFT を 52〜56px に拡張、もしくは `numberOfLines={1}` を強制。スクリーンショット `/tmp/sprint5-B-progress-375.png`、`/tmp/sprint5-C-progress-1280.png`、`/tmp/sprint5-D-progress-360.png` で再現

### [Major-2] V1ScoreChart：X 軸日付ラベルが末尾で重なる（@360 / @375）

- **再現手順**：viewport 360 または 375 で進捗グラフを開き、X 軸最右側を見る
- **期待される振る舞い**：4 日おきラベル `4/27 / 4/30` 等が 1 文字ずつ識別できる
- **実際の振る舞い**：375px viewport で `4/14/14/14/14/24/24/3` のように複数日付が重なって読み取り不能。X ラベル幅 width=48 / 中央寄せ left=cx-24 で配置するため、4 日刻みでも幅 48px の窓が重なる（28 日 / (375 - 64 padding) ≒ 11px/日）
- **該当箇所**：`src/components/V1ScoreChart.tsx` の X ラベル描画ループ（291-303 行）
- **修正の方向性**：(a) スマホ幅では描画間隔を 7 日（週 1 件）に切り替える、(b) ラベルを傾ける、(c) ラベル width を縮め overflow:visible にせず、両端ラベルの位置をクリップ。screen.md S5-04「PC 横ではグラフ右に統計縦並び」と整合性を取るなら (a) が最も読みやすい。1280px では問題なし
- スクリーンショット `/tmp/sprint5-B-progress-375.png`、`/tmp/sprint5-D-progress-360.png` で再現

### [Minor-1] WeeklyGraph タブの aria-selected 値が DOM に出ない

- React Native Web は `accessibilityState.selected` を `aria-selected` に変換するはずだが、実機 DOM 走査では `aria-selected=null` だった。SR 利用者がアクティブタブを判別しにくい
- **該当箇所**：`src/components/WeeklyGraph.tsx` `<TabButton>` の Pressable に `accessibilityState={{ selected: active }}` 指定済み
- **修正の方向性**：Web では Pressable に追加で `aria-selected={active ? 'true' : 'false'}` を直接付与する（react-native-web の解釈ばらつき回避）

### [Minor-2] CooldownScreen の実機動作確認が AppRouter ルート未公開で困難

- AppRouter に直接 `cooldown` ルートがなく、コースを完走しないと到達できない
- 単体テストでは動作確認済み（10 秒タイマー、スキップ可、aria-live polite at <=5）。Sprint 6 でルート公開＋デバッグメニューに動線を追加すると評価しやすい
