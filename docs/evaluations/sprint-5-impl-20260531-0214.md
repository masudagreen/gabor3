## 発見した問題（重要度順）

### [Critical] Web の focus outline が常時表示（`:focus-visible` になっていない）— 全インタラクティブ要素
- 再現手順:
  1. `npm run build:web` → `npx serve -s dist -l 4599`、Playwright で `http://localhost:4599/`。
  2. ホーム「プレイを始める」→ ゲーム開始。n×n 格子の各セルに 3px の青枠が常時出る。
  3. 検証: `document.activeElement.blur()`（フォーカスを body に逃がす）後でも、各セル `getComputedStyle(cell).outline === "rgb(19, 68, 157) solid 3px"`。フォーカス中の要素も同一 outline で、**フォーカス位置が判別不能**。
- 期待される振る舞い: outline はキーボードフォーカス時（`:focus-visible`）のみ表示。非フォーカス時はガボール格子に枠が出ず、選択時のみ subtle 枠（NF-9 / F-01「ガボール視認を阻害しない」）。
- 実際の振る舞い: 全 16 セル＋全ボタン（プレイを始める／タブ／確定／中断ダイアログ 2 ボタン）に 3px 青 outline が**恒常表示**。ガボール刺激の上に青い箱が 16 個重なり、変化検出の核体験を阻害。フォーカスリングとしても機能しない（常時点灯でフォーカス位置不明）。
- 該当箇所のヒント: `src/theme/focusStyle.ts` の `buildFocusStyle()` が `{ outlineColor, outlineWidth:3, outlineStyle:'solid', outlineOffset:2 }` を**インラインスタイルで無条件付与**。react-native-web ではインライン outline は `:focus-visible` に紐づかず常時描画される。波及は 11 コンポーネント（`grep -rl useFocusStyle src/`：BottomTabBar / ConfirmButton / ConfirmDialog / DataResetNotice / GaborPatchCell / GameTopBar / SegmentedControl / SettingRow / Slider / Toggle / HomeWaitingScreen）。
- 修正の方向性: インライン outline 付与をやめ、(a) Web ではグローバル CSS で `*:focus-visible { outline: 3px solid <focusRing>; outline-offset:2px; }` を 1 度だけ注入する、または (b) `onFocus/onBlur` で `focusVisible` 状態を持ち、フォーカス時のみ outline スタイルを適用する。GaborPatchCell は特に「非フォーカス・非選択時は枠ゼロ」を満たすこと。修正後、`blur()` 状態で全セル outline=none・Tab 移動で当該要素のみ outline 表示、を Playwright で再確認。

### [Major] 中断ダイアログの初期フォーカスが破壊的「中断する」に当たっている
- 再現手順:
  1. ゲーム開始 → 左上 ✕（aria-label「ゲームを中断」）を押す。
  2. ダイアログ表示直後に `document.activeElement` を読む → `aria-label === "中断する"`（破壊的 OK ボタン）。スナップショットでも `button "中断する" [active]`。
- 期待される振る舞い: spec `screens.md` S5-2「初期フォーカスは安全側『続ける』」、自己評価も「autoFocus をキャンセル側に付与」と記載。安全側「続ける」にフォーカスし、Enter 即押しで誤って中断しないこと。
- 実際の振る舞い: フォーカスが「中断する」に当たる。キーボード/Enter 操作で**意図せずセッション中断**しうる（記録は元々残らないが、進行中セッションを失う UX リスク）。
- 該当箇所のヒント: `src/components/v2/ConfirmDialog.tsx` L88-94 で cancel Pressable に `{...({ autoFocus: true })}` を付与しているが、react-native-web の Pressable は `autoFocus` を DOM に伝播しない／モーダル初期フォーカスが先頭の confirm ボタンに当たるため意図が効いていない。
- 修正の方向性: マウント後に `ref` 経由で cancel ボタンを明示的に `.focus()`（`useEffect` + `Platform.OS==='web'`）。あるいは DOM の `tabIndex`/フォーカストラップ初期化で cancel を先頭にする。修正後、ダイアログ表示直後の `activeElement` が「続ける」であることを Playwright で確認。Esc=キャンセルは現状動作 OK。

### [Minor] F-12 カウントダウンの warn(≤5s) と danger(≤3s) が同一色（色 3 段階になっていない）
- 再現手順: ゲーム中、残り秒の色/太さを毎値サンプル（light モード）。
  - ≥6s: `rgb(14,15,18)` weight 700 / ≤5s(5,4): `rgb(161,28,22)` weight 700 / ≤3s(3,2,1): `rgb(161,28,22)` weight **900**。
- 期待される振る舞い: spec F-12「通常=白 / ≤5 秒=黄 / ≤3 秒=赤」。warn と danger は**異なる色**であること。
- 実際の振る舞い: warn と danger が同じ赤系色で、区別は太字（700→900）のみ。6→5 の遷移は色変化（黒→赤）するが、5→3 は色が変わらない。light モードで「黄」は白背景に対しコントラスト不足になるため暗赤に寄せた設計意図は理解できるが、3 段階の色差が出ていない。
- 該当箇所のヒント: `src/theme/tokens.ts` の `countdownV2[light].warn` と `.danger`。
- 修正の方向性: light モードでも warn（暗いアンバー/オリーブ系で 7:1 確保）と danger（赤）を**別色**にする。太字差は補強として維持。dark モードのトークンも確認（dark では白/黄/赤の素直な 3 色が可能）。

### [Minor] aria-checked が DOM に出力されない（選択パッチ）
- 現象: ガボールセルを選択しても `aria-checked` 属性が `null`（選択は内部 state と subtle 枠で表現）。SR が選択状態を読み上げられない（NF-15）。
- 補足: 自己評価で「S10 a11y 仕上げで対応予定」と既に申し送り済み。S5 不合格要因には数えないが、S10 までに必ず解消すること。

### [Minor] 中断ダイアログ表示中もゲームタイマーが進行する
- 現象: ✕／他タブでダイアログを出している間も背後の m 秒カウントが進む（dialog-360 で「4」→継続減少を観測）。自己評価は「`paused` で一時停止」と記載するが実機は一時停止していない。
- 影響: 「続ける」復帰時に残り時間が減っている。spec F-07「残り時間・選択状態が保持される」の選択保持は満たすが、時間保持の解釈次第。Designer 判断の余地はあるが、自己評価の記述と挙動が不一致のため要確認。
