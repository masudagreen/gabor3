## 発見した問題（重要度順）

### [Major-1] V1ScoreChart の X 軸ラベル「4/10」「4/17」など 4 文字日付がエリプシス省略される

- **再現手順**：375px 幅で `npm run web` → 進捗グラフタブ → 過去 28 日分の dailyStats を inject → 進捗グラフを表示
- **期待される振る舞い**：X 軸ラベルに「4/10」など完全な日付が表示される（screens.md §22 / 24px 床要件）
- **実際の振る舞い**：DOM の `textContent="4/10"` だが、ラベルコンテナが `width:48px` / `text-overflow:ellipsis` / `overflow:hidden` で `scrollWidth=51` のため 3px はみ出して **「4/...」と表示される**
- **検証データ**：
  - `xlabel-7`（4/10）：scrollWidth=51, clientWidth=48 → 視覚的に「4/...」
  - `xlabel-14`（4/17）：scrollWidth=51, clientWidth=48 → 視覚的に「4/...」
  - `xlabel-21`（4/24）：scrollWidth=51, clientWidth=48 → 視覚的に「4/...」
  - `xlabel-27`（4/30）：scrollWidth=51, clientWidth=48 → 視覚的に「4/...」
  - `xlabel-0`（4/3、3 文字）：scrollWidth=48, clientWidth=48 → 完全表示
- **該当箇所のヒント**：`src/components/V1ScoreChart.tsx` L391-397 `xLabel` スタイル（`width: 48`）
- **修正の方向性**：`xLabel.width` を 48 → 56〜60 に拡張する（or `minWidth: 48` + `width: 'auto'`）。Sprint 5 Major-1 で Y_LABEL_WIDTH を 48 にしたのと同様に、X ラベルも 4 文字 + tabular-nums を許容する幅が必要。Y ラベル（「100」3 文字）と X ラベル（「4/10」4 文字）で必要幅が異なる
- **Sprint 5 Major-2 の修正（7 日刻み切替）自体は機能している**：ラベルが重なる問題は解消、しかしラベル単体の見切れが残った
- **影響**：受け入れ基準 §13 Sprint 5「X 軸ラベル 24px 以上で読める」のうち「読める」が 4 月 10〜31 日の日付で部分的に成立しない。視覚的に補えるが、SR は完全な textContent を読み上げるため SR 利用者には影響なし。視覚利用者で月後半（10 日以降）の縦軸スケール感を見るユーザーに影響

### [Minor-1] B-04（30 日継続）の earned 判定とテストデータの不整合（仕様読みの差）

- **再現手順**：`currentStreak=30` を localStorage に inject → ホーム → バッジ一覧 → B-04 が「未獲得」「まもなく獲得！」と表示
- **観察**：`buildBadgeHint` の B-04 は `Math.max(0, 30 - currentStreak)` で `currentStreak=30` のとき remaining=0 → 「まもなく獲得！」と返す。判定 `currentStreak >= 30` は満たすので `evaluateBadges` を経由すれば earned=true になる。今回は inject で earned=false のまま見せたので「30 日達成済みなのに未獲得」という非整合的な見え方
- **修正の方向性**：実利用上は `applyCourseCompletion` 後の `evaluateBadges` 呼び出しで自動 earn されるので問題ないが、外部からのデータ流入時（移行・将来の同期）に備えて、`BadgeListScreen` の useEffect 初期化時にも `evaluateBadges` を 1 度走らせる「整合化」を追加してもよい。ただしこれは「別マシンから localStorage を流し込んだ」というレアケースなので Minor 扱い

### [Minor-2] B-08 ヒントテキストが `actionPrimary` カラーで描画されるが「3 ゲーム中 N」の数値も同色なので文字列単一強調

- **観察**：BadgeDetailModal の `hint` は `color: colors.actionPrimary` の太字 26px（`fontSize.bodyLg`）。screens.md S6-03 の例文「あと 16 点で獲得！」は `color.brand.accent` 指定。`actionPrimary` と `brand.accent` がトークン上で一致していれば問題なし、相違があれば色差
- **修正の方向性**：`tokens` の `actionPrimary` と `brand.accent` の関係を再確認（system.md §3）。同じ #1B59C8 系であれば実質問題なし、Minor
