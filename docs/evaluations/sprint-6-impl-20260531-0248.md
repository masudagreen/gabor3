## 発見した問題（重要度順）

### [Major] 中断後にホームが「スコア 0／100」のセッション結果カードを誤表示する（観点 5）
- 再現手順:
  1. クリーン起動 → オンボ完了 → 距離リマインド → ゲーム自動開始
  2. プレイ中に左上 ✕（aria-label「ゲームを中断」）→ ダイアログ「プレイを中断しますか？／中断するとこのセッションは記録されません。」→「中断する」
  3. ホームに着地した画面を確認
- 期待される振る舞い:
  中断＝記録しない（F-07 / AS-8）。完了セッションが存在しないので、ホームは「待機中（非進行）／再プレイ可能」状態（例：撤去された HomeWaitingScreen 相当の「プレイを始める」CTA、または距離リマインドに戻す）に着地し、**セッション結果として 0 点を提示しない**。spec は「結果表示中」と「待機中（非進行中）」を別概念として扱う（spec.md L175/L202、AS-8）
- 実際の振る舞い:
  `homePhase='result'` に遷移し SessionResultCard を表示。「今回のスコア 0／100」「🔥 今日からスタート」、region 読み上げも「セッション結果。スコア 0 点（100 点満点）。連続 0 日」。あたかも 0 点で完了したセッションがあるかのように見える（初回中断時。前回完了がある場合は前回スコアが残存表示され、これも誤誘導）。データ層は正しく未記録（sessions 0 件）なので、**UI 着地のみの不整合**
- 該当箇所のヒント:
  - `src/screens/v2/AppRoot.tsx` `confirmAbort`（L149-159）：`setHomePhase('result')` で結果カードへ着地
  - `src/screens/v2/AppRoot.tsx` `DistanceReminderScreen onAbort`（L207）：距離リマインドの ✕ も `setHomePhase('result')` で同じ誤着地（プレイ前なのに結果カード）
  - `homePhase` 型は `'distance' | 'playing' | 'result'`（L52）。`result` 用 lastScore 初期値 0（L107）
- 修正の方向性:
  1. `HomePhase` に `'idle'`（待機/再プレイ可能）を追加し、中断・距離リマインド中断はここへ着地させる。`idle` では SessionResultCard ではなく「プレイを始める」CTA（撤去した HomeWaitingScreen 相当、もしくは距離リマインドへ戻す導線）を表示する。
  2. もしくは「完了セッションが少なくとも 1 件あるか」のフラグ（例：`hasCompletedSession`）を持ち、結果カードは完了直後のみ表示。中断時は idle 着地（結果カードを出さない）。
  3. いずれにせよ「中断したのにスコア 0 の結果が出る」を解消すること。`recordCompletedSession` の戻り値が来ていない遷移では結果カードを描画しない、が判定基準。
