## 発見した問題（重要度順）

### [Major] Skip Link が Enter キーでは activate されない（Space では動作する）
- 再現手順:
  1. http://localhost:8081/ を開きオンボ Welcome 画面を表示
  2. Tab キー 1 回 → Skip Link 「メインコンテンツへ移動」が前面表示され focus される（focused:true / outline 3px）— ここまで OK
  3. Enter キーを押す → Skip Link 自身に focus が残り続ける（document.activeElement が Skip Link のまま、#main へ移らない）
  4. （別検証）Space キー押下 → これも main へは移らない（onPress 不発火）
  5. （別検証）プログラム的な `el.click()` 経由 → 正しく `#main` に focus が移る
- 期待される振る舞い: Enter キーで Skip Link を activate し、`mainRef.current.focus()` が走り `#main`（tabIndex=-1）にフォーカスが移る。WCAG 2.1.1（キーボード操作可能）の要件
- 実際の振る舞い: Enter / Space では onPress が発火せず、Click 合成のみ動作。Skip Link の役目（キーボードユーザーが Tab を何度も押さずに main へジャンプする）が機能不全
- 該当箇所のヒント:
  - 実装：`/Users/np_202212_11/projects/gabor3/src/components/SkipLink.tsx` の `<Pressable accessibilityRole="link" onPress={onActivate}>`
  - 原因：react-native-web の Pressable は内部で `<div>` レンダー + `role="link"`。HTMLAnchorElement と異なり、ブラウザは role="link" な div で Enter→click 合成をしない
- 修正の方向性:
  - Web 限定で `<Pressable onKeyDown={(e) => { if (e.key === 'Enter' || e.key === ' ') { e.preventDefault(); onActivate(); } }}>` を追加
  - もしくは Web では `<a>` タグ（実 anchor）を使う構造に切り替え（react-native-web の `accessibilityRole="link"` を `href="#main"` 相当の Web behavior に）
  - 自己レビュー (sprint-7c §4.1) で「Web 動作は単体テスト不可、Playwright 等の E2E で v2 確認推奨」と申し送りされており、本件は **その懸念の現実化**。次回反復で対応推奨

### [Minor] Skip Link が Tab 順で常に先頭に残るが、focus 喪失後は opacity:0 のまま視覚消失（仕様通り）
- screens.md 記載通りの動作なので問題ではないが、Tab 戻し（Shift+Tab）した際に再表示される動線がある点をリリースノートに残すと親切

### [Minor] AgeGroupScreen の accessibilityState.checked は反映タイミングが極短い
- 自己レビュー sprint-7c §4.1 で言及済み：「onSelect 即時呼び出しで親が state 更新する前の極短時間しか checked=true は見えない」
- 実害なし（SR 読み上げには十分）。v2 で onSelect 前に setTimeout を挟むと安心

### [Minor] DataDeletionConfirmModal の入力欄が初回 open 後に focus されない
- 段階 2 の「削除」入力欄に自動で focus が当たらない（実機で TextInput を click してから入力する必要あり）
- F-15 仕様には明記されていないため Minor 扱い。`useEffect` で `inputRef.current.focus()` を呼ぶと UX 改善

### [Minor] Snackbar の出現位置が削除モーダル背後に隠れる場合あり
- 「ダークモードを保存しました」が画面下部に出るが、その直後に「全データを削除」をタップして開いたモーダルの裏に Snackbar が一瞬見えてしまう（zIndex の重なり順）
- 設計上は 4 秒で自動消失するため Minor
