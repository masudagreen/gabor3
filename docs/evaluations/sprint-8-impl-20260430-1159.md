## 発見した問題

### [Minor] OB-02 チェックボックス aria-checked が null（未設定）

- **再現手順**：OB-02 免責画面でスクロール末尾まで送り、`[role="checkbox"]` の `aria-checked` を取得すると click 前後とも `null`
- **期待される振る舞い**：`aria-checked="false"` → click 後 `aria-checked="true"` で SR が状態を読み上げ
- **実際の振る舞い**：属性そのものが付かないため、SR で「チェックされていません / されています」状態判別不可
- **該当箇所のヒント**：v1 の `Checkbox` コンポーネント（`src/components/Checkbox.tsx` 推定）の React Native Web 互換実装で `accessibilityState={{checked: ...}}` を `aria-checked` 属性にマッピングする経路の問題と推察
- **修正の方向性**：v1 Checkbox 実装を確認し、`accessibilityState.checked` の RNW マッピング経路を検証。Sprint 8 のスコープ外で v1 由来の継承課題のため、Sprint 9 で v11 専用 Checkbox を作る場合は対応推奨
- **重要度の根拠**：Web 視覚的にはチェック ✓ が表示され押下で進めるためユーザー操作には影響なし、SR 利用者に限定的な影響にとどまる

### [Minor] OB-06 プレースホルダの表示順（オンボーディング完了バナーが結果サマリの「上」に出る）

- **再現手順**：OB-05 の「次へ」 → OB-06 でプレースホルダ画面が「✓ オンボーディング完了 / お疲れさまでした…」 → 「1 試行体験 / 実際のゲームは Sprint 9 以降…」 → 「ホームへ」の順で表示
- **期待される振る舞い**：onboarding.md §8 では「結果サマリ → 上に完了通知バナー」の順序が想定（バナーは結果の補足装飾）
- **実際の振る舞い**：完了通知バナーがメインコンテンツの先頭に来ているため、「結果サマリの装飾」というデザイン意図が伝わりにくい
- **該当箇所のヒント**：`src/screens/v11/Onboarding/OB06Experience.tsx`
- **修正の方向性**：Sprint 12 で G-04 本実装に差し替える際、`ResultSummaryV11` の `onboardingCompletionMode` フラグでバナーを結果カードの上または下に配置（onboarding.md §8 の図解準拠）
- **重要度の根拠**：Sprint 8 はプレースホルダ実装と Self-Review §7 で明記されており、Sprint 12 で本実装時に解消される。仕様外ではない
