## 発見した問題

### [Major] M-1 ImageChoiceCell の `aria-checked` 動的更新が SR に反映されない

- **再現手順**：
  1. G-01 プレイ画面に到達
  2. CDP `Accessibility.getFullAXTree` を取得 → セル `role: checkbox`, `checked: false` を確認
  3. cell 0 をタップ（黄色 4px 枠が視覚的に表示される）
  4. 再度 AX ツリーを取得 → `checked: false` のまま（**`true` に変わらない**）
  5. DOM 属性も `aria-checked` は `null`（未設定）のまま
- **期待される振る舞い**：選択中のセルは `aria-checked="true"` または AX ツリーの `checked: true` で SR ユーザーに「選択された」と通知される
- **実際の振る舞い**：視覚的には黄色 4px 枠が表示されるが、SR には選択状態が伝わらない（NF-12「装飾色だけに依存せず状態を伝える」と整合しない）
- **該当箇所のヒント**：`src/components/v11/ImageChoiceCell.tsx:54-66` `<Pressable accessibilityState={{ checked: isSelected, selected: isSelected, disabled: !!disabled }} />`
- **修正の方向性**：
  1. RN Web 0.19 の `accessibilityState.checked` → `aria-checked` 変換は `role="checkbox"` 設定時に効くが、Pressable 経由だと動的再レンダリング時に attribute が更新されないバグ仕様の可能性
  2. Web では `aria-checked` を明示的に props で渡す形に変える、もしくは `Platform.OS === 'web'` で `accessibilityChecked` 等の RN の新しい props を使う
  3. または `accessibilityValue={{ text: isSelected ? '選択中' : '未選択' }}` を併用
  4. 単体テスト `tests/v11/components/ImageChoiceCell.test.tsx` は `accessibilityState.checked` を見ているため通っているが、web 出力としては不十分
- **影響範囲**：G-01 全プレイ、Sprint 10 以降の他のグリッド型ゲーム（G-07 / G-10）にも継承される共通コンポーネントの問題

### [Minor] m-3 ResultSummaryV11 の「今回の閾値」MetricCard に「初回測定」が冗長表示される

- **実際の表示**：「今回の閾値 / 5.0° / 角度差 / 初回測定」（「初回測定」が表示）と「前回比 / — / 初回測定」の両方に「初回測定」が出る
- **期待**：threshold カードには diff 表示エリア自体が不要（前回比カードのみで開示）
- **該当箇所**：`src/components/v11/MetricCard.tsx:78-92` の `renderDiff` で `diff = undefined` 時に「初回測定」を出している。閾値カードでは diff prop が未指定だが renderDiff が呼ばれている
- **修正の方向性**：
  1. MetricCard に `showDiffArea?: boolean` を追加し、threshold カードでは false にする
  2. または renderDiff で `props.diff === undefined && hideWhenAbsent` の場合は何も返さない
