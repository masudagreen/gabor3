## must（仕様の根幹に関わる）

### 1. **連続プレイの構成（残ゲーム集合と進行）** — F-05 / 起動シーケンス
G-02・G-08 を完全削除した後、連続プレイで実行されるゲームは「G-01, G-03, G-04, G-05, G-06, G-07, G-09, G-10, G-11, G-12, G-13 の 11 ゲーム」で全件、という理解で合っていますか？それとも v1.1 の試作評価結果として、この場で他にも除外したいゲーム（例：G-09〜G-12 など UI 構造判断中のもの）があれば併せて指定してください。
また、(a) 順序は固定 / 日付シードでランダム / 毎回ランダム のどれか、(b) 1 周のみ / 完了後ループ可、(c) 途中で 1 ゲームをスキップする UI を残すか（現状は中断＝確認ダイアログで全体終了）、を確認させてください。

### 2. **G-01「ゆっくり回るアイテム」の挙動仕様** — §7.1
「いくつかコントラストの違う」ではなく「ゆっくり回るアイテムが**ある**」とのことですが、観察可能な挙動として以下を確定したいです：
- (a) 3×3 / 4×4 / 5×5 のうち今後採用するグリッドサイズ（現行 staircase は最大角度差で難化、グリッドサイズは固定？可変？）
- (b) 回転アイテムの個数：常に 1 個固定 / 1〜N 個ランダム（N は？）/ staircase で増減
- (c) 静止アイテムの角度：全て同一の角度に揃える / それぞれ異なるランダム角（=「角度が動いているか」だけが手がかり）
- (d) 回転速度：staircase の対象にする（速い=易→遅い=難）か固定値か。固定なら何度/秒目安
- (e) 回転方向：常に同方向（時計回り固定）/ アイテムごとにランダム / 各回でランダム
- (f) 採点方式：v1.1 の TP+1/FP-1/FN 0 を維持するか、回転アイテムが 1 個固定なら単一選択方式に変更するか
- (g) メッセージ「回転しているパッチをタップ」は出題方向（時計回り/反時計回り）情報を含めるか

### 3. **G-04 / G-05 / G-06 共通の 3×3 化と選択方式** — §7.4 / 7.5 / 7.6
3 ゲームとも 3×3 等の grid に変更とのことですが、観察可能な挙動を以下の粒度で確定したいです：
- (a) グリッドサイズは 3×3 固定 / staircase で 3×3 ⇄ 4×4 など可変
- (b) 「コントラスト/SF/窓サイズが違うもの」の混在数：9 個中 1 個違う / 複数個（何個？）違う / staircase で個数増減
- (c) 選択数：単一選択 / 複数選択（何個まで？）/「違うものを全て選んで」（個数不明のままユーザー判断）
- (d) 採点：完全一致のみ正解 / 部分点（TP+1/FP-1 の G-01 方式）
- (e) 「もっと違いを少なくして」の具体度：staircase 初期値を現行の何分の一にするか、または「Designer/Generator が初期値・step 幅・min/max を Web 体感で再調整して良い」と委ねるか
- (f) G-06 は「違いは今と同じで試させて」とありますが、これは「グリッド化＋選択方式は G-04/05 と統一だが、staircase の難度範囲（初期値・min/max・step）は現状の窓サイズ範囲を維持」という解釈で合っていますか

### 4. **G-13 数字探しの問題領域拡大の具体** — §7.13
「画面もっと広げて」を観察可能な数値で確定したいです：
- (a) 刺激領域はセーフエリア（ステータスバー下〜下端の Home Indicator 上）の幅・高さの何 % 以上 / 最大化（残りはキーパッドのみ）
- (b) keypad-10 の配置は維持（Designer 判断）として、刺激領域とボタン領域の比率の目安（例：画面縦の上 70% を刺激、下 30% をキーパッド、など）はあるか
- (c) 他のゲーム（G-01・G-03・G-04〜G-07 等）でも同水準まで刺激領域を広げるべきか、G-13 のみが対象か

### 5. **正解/不正解アイコン（✅/❌）の表示仕様** — F-10 改訂
「テキスト正解表示欄を撤去し、✅（緑）/❌（赤）だけをパッチ欄の下に表示」とのこと。以下を確定したいです：
- (a) 表示位置：「パッチ欄の下」とは刺激領域の直下に 1 個（試行全体の正誤）か、F-10 現行通り**各パッチ／ボタン上に重畳**したまま色とアイコンだけ ✅/❌ に置換か
- (b) 「不要にするテキスト正解表示欄」とは具体的に F-10 の「今回の閾値」「前回比」などの数値テキストのことか、それとも別の文字列（例：ゲーム下部の「正解：時計回り」のような表示）か
- (c) 表示時間：単体プレイは「次へ」まで永続表示、コースモードは 10 秒カウントダウン（現行 F-10）を維持で良いか
- (d) 複数選択ゲーム（G-01 / G-07 / G-09）では各パッチに ✅/❌ を個別重畳する現行ルールを維持するか、それとも刺激欄下に総合の ✅/❌ 1 個だけにするか
- (e) スクリーンリーダー用の代替テキスト（「正解」「不正解」）は読み上げで残すか

### 6. **カウントダウン表示の仕様** — F-07 受け入れ基準
「数字だけ・時計アイコン無し・5 秒以下で黄色」を確定したいです：
- (a) 通常時の文字色（白？グレー？）と 5 秒以下の黄色への切替閾値（≤5 秒で良いか）
- (b) さらに警告色（赤等）への 2 段階切替が 3 秒以下である / 無し のどちらか
- (c) 色変化のみで状態を伝えるのは NF-12（色だけで情報を伝えない原則）に抵触するため、補強表現（太字化／サイズ拡大／点滅／触覚フィードバック）を入れるか
- (d) この仕様変更は F-07 の「カウントダウン 22pt 以上常時表示」だけでなく、F-16 距離リマインドの 3 秒カウントダウン、F-15 クールダウン 10 秒、F-10 結果オーバーレイの 10 秒カウントダウンにも一律適用か（個別仕様か）

### 7. **音・振動の発火タイミング網羅** — 新規 NF / F-XX
「rapidreading2 を参考に Android で音・振動を出す」とのこと。観察可能な発火イベントを全て列挙したいので確認させてください：
- (a) 音：正解時 / 不正解時 / カウントダウン残り 3 秒以下の毎秒 / 60 秒ゲーム終了時 / バッジ獲得時 / セッション完了時 / 距離リマインドの自動進行時 — どれを鳴らすか
- (b) 振動：上記のうちどれをハプティクスで補強するか（特に正解/不正解は音のない環境で重要）
- (c) ユーザーが音/振動を設定でオフにできるか（v1.1 の Settings に項目があるか確認、無ければ追加）
- (d) サイレントモード時の挙動（音は鳴らさずハプティクスのみ等）

### 8. **アプリアイコン差し替えの範囲** — 新規
「アイコンはガボールパッチの 45 度の画像」とのこと。差し替え対象は以下全てで良いですか：
- (a) iOS app icon（1024×1024、各サイズ）
- (b) Android adaptive icon（foreground / background）
- (c) Android legacy icon
- (d) Web favicon / PWA manifest icon
- (e) Expo splash screen のアイコン部分
- (f) 「45 度」は時計回り 45° / 反時計回り 45°（左下→右上 / 左上→右下）どちらか、または Designer 判断で良いか

### 9. **ステータスバー回避の方針** — 新規 NF
「ゲーム画面以外で文字・アイコンがステータスバーに入り込まない」とのこと。確認したいのは：
- (a) ゲーム画面（刺激提示中）はフルスクリーン（ステータスバー領域も使用）で良いか、それとも全画面でセーフエリア準拠か
- (b) iOS は Safe Area Inset、Android は StatusBar.currentHeight の自動取得で対応で良いか（実装詳細だが、ユーザーが期待する観察挙動として「ステータスバーは透過 / 不透過 / 非表示」のどれか）
- (c) 現行 v1.1.5 で「OS ステータスバー保護＋刺激領域マージン調整」を実施済みだが、今回のフィードバックは v1.1.5 適用前に観察された問題か、適用後も残存しているか

## nice-to-have（解像度を上げる）

### 10. **G-03 解答表示の統一** — §7.3
「中央からの線になっているのを他と同様にパッチの上に表示してほしい」とのこと。F-10 の ◯/✕ オーバーレイ方式（各パッチ上に直接重畳）に統一する、という解釈で良いか確認。中央からの線（指示線）は完全廃止で OK か、視覚補助として残す余地はあるか。

### 11. **G-07 のメッセージ文言** — §7.7
「向きが同じ 3 個を選んで」に確定で良いか。現行は何個選択か（3 個固定 / staircase で個数可変）が spec.md §7.7 に明記されていないので、ついでに「3 個固定」を仕様に書き込んで良いか確認。

### 12. **「ゲーム一覧メニュー完全廃止」に伴う影響範囲** — F-04 / F-06
ゲーム一覧画面廃止に伴い、以下の機能は廃止 / 維持 / 形を変えて維持のどれか：
- (a) F-06 単体ゲームモード自体を廃止（連続プレイのみが本番動線）
- (b) F-04 ホーム画面そのものも廃止（起動 → 即秒読み）か、進捗グラフ・バッジ・設定への入口を持つホームは残すか（残す場合、それは「連続プレイ完了後に戻る画面」として存在する）
- (c) v1.1 既存ユーザーの「単体プレイ履歴」データは v1.1.5 まで蓄積されているが、v1.2 起動時にリセット / 残置のどちらか（F-17 の挙動に追加）

### 13. **G-04/05/06 グリッドの「3×3 隙間」基準** — §7.1（4×4 隙間問題）
「3×3 くらいの隙間までパッチと表示領域を大きく」とのこと。これは Designer 判断の範囲ですが、観察可能な目安として「グリッド全体が刺激領域の何 % を占める」という数値の希望はあるか（無ければ Designer に任せる）。また、4×4 グリッドを採用する他ゲーム（G-01 の可能性、G-10 の 8×8 等）にも同方針で隙間を最小化するか。

### 14. **ガボールクリッピング方式の品質基準** — 共通刺激パラメータ §6
「大きめのパッチを四角枠でクリッピング」を実装側に委ねる方針ですが、観察可能な受け入れ基準として「パッチ枠と縞模様の間に背景色（グレー）が見えない」「角度を 1° ずつ変えても枠との隙間が出ない」を NF / 受け入れ基準に追加して良いか確認。

### 15. **角度自由化に伴う既存 staircase データの扱い** — F-09 / F-17
現行 staircase は「45° 単位」前提で動いている可能性があるため、角度自由化により既存 StaircaseState が無効化される。v1.2 起動時に staircase を全ゲーム自動リセットする（F-17 の起動時リセットに統合）か、リセットせずパラメータ値だけ引き継ぐか確認。

## 人間からの回答（v1.2 確定事項）

> オーケストレーター ↔ ユーザーで合意済み。Planner はこれを spec-v11.md (v1.2) に反映する。

### A0. 改訂方針 (事前合意)
- spec-v11.md を **v1.2 として上書き改訂** (バージョンを v1.2 に上げる)。design は `docs/design-v11/` を amendment で更新。
- G-02, G-08 を **コード・データ・履歴・ナビ・index 全てから完全削除** (試作期なので過去スコア消失 OK)。
- ゲーム一覧メニュー画面は **完全廃止**。アプリ起動 = 連続プレイの秒読み画面が直接出る。
- 角度自由の最小単位は spec には書かず **Designer/Generator が技術判断で決定**、私経由で承認。

### A1. 連続プレイ構成
- 連続プレイは現状の **G-01, G-02, G-03, G-04, G-05, G-06, G-07, G-08, G-13 の 9 ゲーム**から **G-02, G-08 を削除した 7 ゲーム** (G-01, G-03, G-04, G-05, G-06, G-07, G-13)。
- 順序: 固定 (上記の番号順)。
- 周回: 1 周のみ。完了後は事後画面 (進捗グラフ・バッジ・設定への入口) に遷移。
- スキップ: 不可。中断ボタンのみ。

### A2. G-01「ゆっくり回るアイテム」
- グリッド: **3×3 固定**。
- 回転アイテムの個数: **ランダム** (1〜N 個、N と分布は Designer/Generator 判断)。
- 静止アイテムの角度: **それぞれ異なるランダム角** (角度自由化に従う)。
- 回転速度: **staircase 対象** (易: 速い → 難: 遅い)。
- 回転方向: **時計回り / 反時計回り の両方向ランダム** (アイテムごと)。
- 選択方式: **複数選択** (タップで切替)。
- 採点: **TP+1 / FP-1 / FN 0 の部分点**。
- メッセージ: 「回転しているパッチをタップ」 (方向情報・個数情報は含めない)。

### A3. G-04 / G-05 / G-06 共通の 3×3 化
- グリッド: **3×3 固定**。
- 違うパッチの個数: **複数個ランダム** (個数と分布は Designer/Generator 判断)。
- 選択方式: **複数選択** (タップで切替)。
- 採点: **TP+1 / FP-1 / FN 0 の部分点** (G-01 と統一)。
- 「違いをもっと少なく」の具体値: **Designer/Generator が Web 体感で再調整**。staircase 初期値・min/max・step は実装側裁量。
- G-06 のみ: 違い (窓サイズ範囲) は **現状維持**。3×3 化と複数選択方式は他 2 ゲームと統一。

### A4. G-13 数字探しの領域拡大
- セーフエリア内で **上 70% 刺激／下 30% キーパッド** に最大化。
- 他ゲーム (G-01, 03, 04-07) の刺激領域は **現状維持** (今回は G-13 のみ)。

### A5. ✅/❌ アイコン表示
- 表示位置: パッチ欄の直下に **試行全体の正誤を 1 個** 表示。
- F-10 のテキスト「今回の閾値」「前回比」など数値テキストは **撤去**。
- 表示時間: 単体プレイは「次へ」まで永続、コースモード (連続プレイ) は現行 10 秒カウントダウン維持。
- 複数選択ゲーム (G-01, G-04, G-05, G-06, G-07): **各パッチ上に個別 ✅/❌ を重畳併用** + 試行全体の総合 ✅/❌ も下部表示。
- スクリーンリーダー用「正解」「不正解」読み上げは **残す**。

### A6. カウントダウン表示
- 数字のみ (時計アイコン廃止)。
- 色: 通常 **白** ／ ≤5 秒 **黄** ／ ≤3 秒 **赤** の 2 段階切替。
- 補強: **太字化** (点滅は NF-11 違反のため不可)。サイズ拡大は適宜 Designer 判断。
- 適用範囲: F-07 (ゲーム時間)・F-15 (クールダウン 10 秒)・F-16 (距離リマインド 3 秒)・F-10 (結果オーバーレイ 10 秒) **全てに一律適用**。

### A7. 音・振動
- 音: 正解 / 不正解 / カウントダウン残り 3・2・1 秒のティック / ゲーム終了 / バッジ獲得。
- ハプティクス (振動): 正解 / 不正解 / バッジ獲得。
- 設定: **音 / 振動を個別 ON/OFF** 可能 (v1.1 Settings に項目を追加)。
- サイレントモード時: **音は鳴らさずハプティクスのみ**。
- 実装: `../rapidreading2/src/platform/audio.ts` の抽象を流用。

### A8. アプリアイコン差し替え
- 対象: iOS app icon / Android adaptive (foreground+background) / Android legacy / Web favicon・PWA / Expo splash の **全部差し替え**。
- 図柄: **ガボールパッチの 45° (左下→右上、時計回り 45°)** で固定。

### A9. ステータスバー回避
- ゲーム画面 (刺激提示中): **フルスクリーン許容** (ステータスバー領域も使用可)。
- それ以外の画面 (秒読み・事後画面・設定など): **セーフエリア準拠**。
- 実装: iOS = SafeAreaInset、Android = StatusBar.currentHeight 自動取得。
- ※ v1.1.5 適用後も残存している問題、という前提で対処。

### A10. G-03 解答表示
- F-10 と同じく **パッチ上重畳** に統一。中央からの線は **廃止**。

### A11. G-07 メッセージ
- メッセージ: **常に伏字「向きが同じものを選んで」**。個数は問題文に出さない。
- 個数: ランダム (Designer/Generator 判断)。「向きが同じ」アイテムが複数存在する。
- 採点: **TP+1 / FP-1 / FN 0 の部分点** (他 3×3 系と統一)。

### A12. ホーム画面の扱い
- 起動 = 即秒読み (一覧・ホーム表示なし)。
- 連続プレイ完了後の **事後画面** (=結果＋進捗グラフ＋バッジ＋設定への入口) は維持。
- v1.1 既存ユーザーの単体プレイ履歴は v1.2 起動時に **リセット**。

### A13. グリッド隙間
- 数値希望なし、**Designer 判断**。4×4 を採用するゲームがあれば同方針 (隙間最小化) を適用。

### A14. クリッピング品質基準
- NF に以下を追加:
  - パッチ枠と縞模様の間に背景色 (グレー) が見えない。
  - 角度を最小単位ずつ変えても枠との隙間が出ない。

### A15. staircase 既存データ
- v1.2 起動時に **全ゲーム自動リセット** (角度刻み変更で過去パラメータが無効化されるため)。F-17 の起動時リセットに統合。
