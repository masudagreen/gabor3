# GaborEye v3.0 — レベル管理リブート 質疑（確定）

> v2.0（スコア管理）→ v3.0（レベル管理）の大規模リブートに先立つ確認事項と確定回答。
> オーケストレーターがユーザーに直接確認し、2026-06-09 に確定。Planner は本回答を入力に spec を作成する。

---

## ユーザー要望（原文趣旨）

履歴はスコアではなく **レベル** を管理する。レベル = その人が正解できる難易度。
レベルは以下 5 変数の組み合わせで決まる：

| 変数 | 取りうる値（易 → 難） |
|---|---|
| 回転速度 | 6, 5.5, 5, 4.5, 4, 3.5, 3, 2.5, 2（9 値。速い=易、遅い=難） |
| 個数 | 1, 2, 3, 4（4 値） |
| 時間 | 40, 35, 30, 25, 20 秒（5 値） |
| 回転方向 | 一方向, 振動（2 値） |
| サイズ | 3x3, 4x4（2 値） |

- ゲーム画面に現在のレベルを表示する。
- 回転しているパッチの個数も表示する（例「3 個探せ！」）。
- 現在のレベルをクリアすると次のゲームでレベルが上がる。**2 回連続で失敗するとレベルが下がる**。
- レベルが 1 上がると（最内側変数が）増える。最内側変数が最大値に達したら次の変数を 1 段難化し、最内側はリセット（mixed-radix オドメータ）。
- 変化順は設定画面で設定可能にする。
- 各変数の振れ幅もテスト用に設定画面で設定可能にする。
- 現在の設定値で不要なものは削除する。

## Q1. レベルの「例」と「ルール文」の矛盾 → デフォルトの変化順

**確定：ルール文に合わせる（個数が最内側）。**

- 変化順（最内側 LSB → 最外側 MSB、デフォルト）：**個数 → 時間 → 回転方向 → サイズ → 回転速度**
- 各変数の易 → 難方向：個数 1→4 / 時間 40→20 / 一方向→振動 / 3x3→4x4 / 回転速度 6→2
- 組合せ総数 = 4 × 5 × 2 × 2 × 9 = **720 レベル**
- ユーザー提示の「例」（L2=時間35 等）はこのルールと食い違うため **例は説明用** とし、実際のデフォルト梯子は個数最内側に従う。
  - 正しいデフォルト梯子：L1(個数1,時40,一方向,3x3,速6) → L2(個数2) → L3(個数3) → L4(個数4) → L5(個数1,時35) → … → L20(個数4,時20) → L21(個数1,時40,振動) → …

## Q2. レベルの「クリア / 失敗」判定基準

**確定：全問正解=クリア / 誤答(FP>0)・取りこぼし(FN>0)・時間切れ=失敗。**

- 制限時間内に回転パッチを過不足なく全て選んだら **クリア**。
- 誤選択・選び逃し・時間切れは **失敗**。
- 現在の採点 3 方式（scoringMode ①②③）は **廃止**。クリア判定 1 本に統一。
- TP/FP/FN 部分点・0〜100 セッションスコアは廃止（レベル管理へ全面移行）。

## Q3. 1 回の「ゲーム」の単位

**確定：1 ゲーム = 1 ラウンド = 1 レベル挑戦。**

- 複数ラウンド設定（roundCount = r）は **廃止**。
- 1 画面 = 1 レベル挑戦。クリア / 失敗が即レベル増減に反映。

## Q4. 設定画面の整理方針

**確定：**

- レベルで自動決定される 5 変数（回転速度 / 個数 / 時間 / 方向 / サイズ）の **手動スライダーは削除**。
- 代わりに **各変数の取りうる範囲（振れ幅）** をテスト用に設定可能化。
- **変化順** を設定画面で組み替え可能化。
- **残す既存設定**：ダークモード / 音 ON-OFF / 振動 ON-OFF / 片眼ガイダンス / 視聴距離 / 免責事項再閲覧 / 全データ削除。
- **削除する既存設定**：sfChangeSpeed(b、既に死にパラメータ) / scoringMode / roundCount(r) / gridSize(n) / roundSeconds(m) / rotationSpeed(a) の手動設定 UI。
