# 仕様策定のための確認事項

> ガボール図形を用いた視力回復アプリ（React Native + Web）の企画・仕様化にあたり、リサーチ結果と素案を踏まえて確認したい点を整理しました。
> オーケストレーターから人間に代理質問してください。各項目には「推奨デフォルト」を記載しているので、「任せる」と言われた場合はこれを採用します。

## A. Gabor リサーチ結果サマリ（要点）

- **Gabor パッチ** = ガウシアン窓で振幅変調された正弦波の縞模様。ヒトV1（一次視覚野）受容野と数学的に一致するため、視覚野ニューロンを直接駆動する刺激として最適とされる。
- **メカニズム**：眼球光学を変えるのではなく「脳側のコントラスト感度関数（CSF）」を perceptual learning で底上げする。近視・老眼・遠視のいずれにも理屈上効きうる（ただし重度近視は対象外、医療効果は断定しない表現が必要）。
- **典型パラメータ**：空間周波数 1.5 / 3 / 6 / 12 / 18 cycles/degree（cpd）、ガウス窓 SD ≈ 0.17–1.0°、コントラスト 0.1–0.5、向きは 0/45/90/135° などからランダム抽出、位相もランダム化が一般的。
- **推奨利用**：1 日 3〜10 分、画面距離はスマホ 30cm / PC 40cm 程度、できれば毎日継続。即効性はなく数週〜1ヶ月単位で評価する。
- **学習を促す課題設計**：閾値付近（正答率 ~79%）で刺激を提示し、3-down/1-up staircase で難易度自動調整するのが perceptual learning 文献での標準。Two-alternative forced choice（2AFC）/ orientation discrimination / contrast detection が定番タスク。
- **既存アプリ・書籍の遊び方**：平松類『ガボール・アイ』は「並んだガボールから同じ縞模様を探す」神経衰弱型が中心。同型は飽きやすいというユーザー指摘とも符合。

参考：[UC Berkeley Polat et al. 2012 (PMC)](https://pmc.ncbi.nlm.nih.gov/articles/PMC2258094/) ／ [Visionary: Gabor patch解説](https://getvisionary.app/gabor-patches-explained) ／ [RevitalVision科学的背景](https://doctors.revitalvision.com/about/vision-improvement-technology/) ／ [Perceptual learning of orientation (Vis. Res.)](https://www.sciencedirect.com/science/article/pii/S0042698905002191) ／ [ESSE: ガボール・アイ解説](https://esse-online.jp/articles/-/3487) ／ [めがね新聞](https://megane-shinbun.com/health/gabor-patch/)

## B. ゲーム 3 種の企画素案

> いずれもガボールパッチを「見る対象」にすることで、V1 が刺激される設計を共通の土台とする。違いは「答え方」と「飽きさせない仕掛け」。

### Game 1: 変化察知（Morphing Detection）— 必須・ユーザー指定
- **見るもの**：3×3〜5×5 グリッドにガボールパッチを敷き詰める。一定時間（例 20–60 秒）かけて、そのうち 1〜3 個の角度 or サイズが**ごく緩やかに**連続変化する（例：90° → 92° を 30 秒）。
- **答え方**：「変化したパッチ」をすべてタップ。完了ボタンで採点。
- **難易度の出し方**：（a）変化量（°やサイズ%）、（b）変化スピード、（c）同時変化数、（d）グリッドサイズ、（e）他パッチのコントラスト。staircase で自動調整、もしくはレベル選択。
- **視覚野訓練の根拠**：閾値近傍の orientation/size 弁別は perceptual learning の古典課題。長時間注視でじわじわ変わるため、コントラスト感度と位置弁別を同時に鍛える。
- **権利配慮**：「アハ体験」という語は使わず、「変化察知」「Spot the Change」等の表記に統一。

### Game 2: 二重表裏判別（Dual Orientation Match）— 提案
- **見るもの**：画面に**ガボールパッチが 1 個だけ**、約 1 秒だけ表示 → 視覚マスク（高コントラストのランダムガボール）を 200ms → 同じ位置にもう 1 つガボールが約 1 秒。
- **答え方**：2AFC で「2 番目は 1 番目より時計回り？反時計回り？」（または「コントラストが高かった？低かった？」）。
- **難易度の出し方**：角度差 ±1°〜±10°、コントラストを閾値近くまで下げる、マスクの強度を上げる。3-down/1-up staircase で正答率 ~79% を維持。
- **じーっと見ても分からない理由**：刺激時間が短く（1 秒）、マスクで残像も消えるため、注視の持続では解けない。**瞬間的な V1 応答を頼りに判断するしかなく**、知覚学習の本道。
- **視覚野訓練の根拠**：Polat 2012 / RevitalVision で採用されている古典的 orientation/contrast discrimination パラダイム。エビデンスが最も厚い。
- **飽きさせない工夫**：1 試行 3 秒程度。ストリーク・コンボ・staircase 到達閾値の可視化（自分の「視力スコア」）。

### Game 3: 周辺視野ハント（Peripheral Pop-out）— 提案
- **見るもの**：画面中央に固視点（小さい十字）。周囲に 6〜12 個のガボールが配置され、**そのうち 1 個だけ向き or 空間周波数が違う**。表示時間は 300–800ms とし、その後マスク。
- **答え方**：固視点を見続けたまま、「odd one」が**どの方向（時計の文字盤で何時方向）にあったか**を 8〜12 択でタップ。
- **難易度の出し方**：odd と他の差（角度差・cpd 差）、表示時間、ガボール個数、固視点からの離心角（中心視 vs 周辺視）。
- **じーっと見ても分からない理由**：固視点を見ながらの周辺視判断であり、表示時間も短いため、サッカード（眼球運動）で見にいけない。**周辺視野の解像度を強制的に使う**ため、注視 = 解けない。
- **視覚野訓練の根拠**：周辺視の orientation pop-out は V1 の lateral interaction 強化に効くとされ、Polat の研究系列でも採用。Useful Field of View (UFOV) 拡大にも資する。
- **飽きさせない工夫**：1 試行 1〜2 秒で高速進行。BGM とリズムをシンクさせるとリズムゲーム的快感が出せる。

## C. 質問リスト

### must（仕様の根幹に関わる）

1. **対象ユーザーの想定範囲** — 大人の視力回復ニーズ（老眼・軽度近視）を想定していますが、子どもや高齢者（70 代以上）は対象に含めますか？医療機関監修や年齢別チューニングが必要か。
   - 推奨デフォルト：**20〜60 代の自助セルフケア層**を主ターゲット。子ども・高齢者・重度視覚障害者は「対象外」と明記。免責文を初回起動時とアプリ内設定で必ず表示。

2. **1 日あたりの推奨利用時間と「1 セッション」の定義** — 文献どおり 1 日 3〜10 分でしょうか。1 セッション = 何分？3 ゲームを必ず全部やらせる？
   - 推奨デフォルト：**1 セッション = 約 3 分**。「3 ゲームを 1 分ずつ」のおまかせコースと「好きなゲームだけ」モードを両方提供。1 日 1 セッションをデイリー目標に。

3. **継続を支える仕組み（飽き防止）の優先度** — モーフィング以外の 2 ゲーム以外に、「ストリーク」「達成バッジ」「ランキング」「日替わりお題」「キャラクターのねぎらい」などのメタ要素はどこまで入れますか？
   - 推奨デフォルト：**ストリーク（連続日数）+ 日次ベスト記録 + 軽い達成バッジ**を入れる。ランキング（他人との比較）は v1 では入れない（プレッシャーで離脱の懸念）。SNS 連携も v1 では除外。

4. **進捗・スコアの見せ方** — 「視力が回復している感」を何で示す？正答率？反応時間？staircase 閾値の推移？「視力推定値（◯.◯）」のような数値？
   - 推奨デフォルト：**ゲームごとの「閾値スコア」（角度差・コントラスト・cpd）を週次グラフで表示**。「視力 0.3 → 0.5」のような医療数値は出さない（誤解と薬機法リスク）。「あなたの V1 スコア」のような独自指標。

5. **アカウント・データ保存** — ローカルだけで完結？クラウド同期して機種変後も継続？匿名利用？ログイン必須？
   - 推奨デフォルト：**v1 はローカルのみ（端末内保存）**。クラウド同期はスコープ外。複数端末で同じユーザーを扱う必要が出たら v2 で検討。

6. **ガボール刺激の物理キャリブレーション** — ユーザーごとに画面サイズ・視聴距離が違うため、cpd（cycles/degree）が再現できません。簡易キャリブレーション（クレジットカードで画面 dpi を測る、距離はスライダーで申告）を入れますか？
   - 推奨デフォルト：**v1 は簡易キャリブレーション（端末タイプ自動推定 + 視聴距離スライダー：30/40/50cm）**。本格的な計測キャリブレーションは v2。アプリ起動時とトレーニング開始前に「画面から 30cm 離れてください」リマインドを表示。

7. **オフライン動作** — 通学・通勤などスキマ時間想定だとオフライン必須？
   - 推奨デフォルト：**完全オフライン動作**。ネットワーク不要。

8. **対応プラットフォームと縦横レイアウト** — iOS / Android / Web の 3 つ全部？タブレットも？縦持ちのみ？横持ちも対応？
   - 推奨デフォルト：**iOS / Android / Web（PC ブラウザ）3 つ対応**。スマホは縦持ち固定、PC/タブレットは横レイアウト。最小幅 360px、最大は 1280px までを想定。

9. **アクセシビリティ・配慮事項** — 色覚多様性、まぶしさ、てんかん配慮（点滅）、片眼トレーニング機能はどこまで？
   - 推奨デフォルト：**ガボールはグレースケールが基本（色覚配慮）**。点滅は使わない（マスクは静的ノイズ）。ダークモード対応必須。片眼トレーニング（左目だけ・右目だけ）はガイダンス（「片目を手で覆ってください」）として提供、機械的なオクルージョンはしない。

10. **言語** — 日本語のみ？英語も？
    - 推奨デフォルト：**v1 は日本語のみ**。文言は仕様書に集約しておき、後から多言語化しやすい構造にする。

### nice-to-have（解像度を上げる）

11. **オンボーディング（初回チュートリアル）の長さ** — 初回起動から 1 ゲーム完了まで何分以内に到達させるか？
    - 推奨デフォルト：**90 秒以内**。免責同意 → 視聴距離設定 → ゲーム 1 のミニ説明 → 1 試行で完了。説明はゲーム別に都度表示し、最初に全部読ませない。

12. **トレーニング前後のウォームアップ／クールダウン** — 「20-20-20 ルール」（20 分ごとに 20 秒、20 フィート先を見る）的な目休めを組み込みますか？
    - 推奨デフォルト：**1 セッション終了時に「10 秒間遠くを見ましょう」のクールダウン画面**を 1 枚挟む。ウォームアップは v1 では省略。

13. **音とハプティクス** — 正解音・不正解音・BGM はあり？無音前提？スマホはバイブする？
    - 推奨デフォルト：**正解時のみ短い効果音 + 軽いハプティクス**。BGM はオフ（集中力優先）。設定で全 OFF にできる。Game 3（周辺視野ハント）のみ任意でリズム BGM。

14. **失敗・スキップの扱い** — Game 1 で「変化が分からなかった」とき、ヒント表示は？タイムアップ後に答え合わせ（変化箇所をハイライト）する？
    - 推奨デフォルト：**毎試行終了時に正解箇所をアニメで提示**（学習促進のため）。スキップ機能あり、スキップは記録上「未挑戦」扱い。

15. **将来拡張・スコープ外の明示** — マルチプレイヤー、医療機関連携、Apple Watch ／ Google Fit 連携、Eye Tracking、ARKit/ARCore でガボールを空間配置、などはやる？やらない？
    - 推奨デフォルト：**v1 では全部スコープ外**。spec.md の「スコープ外」セクションに明記。
