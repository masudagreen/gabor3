## 改訂履歴（v1.2 主要変更点）

v1.1.2 → v1.2（2026-05-10）の主要変更：

- **ゲーム削減（13 → 7）**：G-02（左右並び傾き判別）と G-08（残像方位弁別）を**コード・データ・履歴・ナビ・index 全てから完全削除**。後続ゲーム番号は再採番せず欠番扱い（G-01, G-03, G-04, G-05, G-06, G-07, G-13）。
- **連続プレイ専用化**：ゲーム一覧画面・ホーム画面・単体プレイモード（旧 F-06）を**廃止**。アプリ起動 = 連続プレイの秒読み画面が直接出る。連続プレイ完了後の事後画面（結果＋進捗グラフ＋バッジ＋設定への入口）を新設。
- **G-01 仕様刷新**：3×3 固定グリッド、回転アイテム個数ランダム、両方向回転、回転速度を staircase 対象、複数選択 TP+1/FP-1 部分点、メッセージ「回転しているパッチをタップ」。
- **G-04 / G-05 / G-06 共通の 3×3 化**：3×3 固定、違うパッチ複数個ランダム、複数選択、TP+1/FP-1 部分点。「違いの強さ」初期値・step は Designer/Generator が体感調整。G-06 の窓サイズ範囲は現状維持。
- **G-03 解答表示統一**：中央線方式を廃止し、F-10 と同じくパッチ上 ◯/✕ 重畳に統一。
- **G-07 文言統一**：メッセージ「向きが同じものを選んで」（個数伏字、ランダム）。TP+1/FP-1 部分点。
- **G-13 領域拡大**：刺激領域を安全領域内で上 70% / キーパッド 30% に最大化。他ゲームの刺激領域は現状維持。
- **角度自由化（G-01・G-04 等）**：旧「45° 単位」前提を撤廃。最小単位は Designer/Generator が表示サイズと cpd から導出（spec には具体値を書かない）。
- **カウントダウン UI 統一**：時計アイコン廃止、数字のみ。色は通常 白 → ≤5 秒 黄 → ≤3 秒 赤の 2 段階切替、補強は太字化（点滅禁止）。F-07 / F-15 / F-16 / F-10 全箇所に一律適用。
- **結果表示の数値テキスト撤去**：F-10 から「今回の閾値」「前回比」などの数値テキストを撤去。代わりにパッチ欄直下に試行全体の ✅（緑）/ ❌（赤）を 1 個。複数選択ゲームは各パッチ上にも個別 ✅/❌ を重畳併用。SR 用「正解」「不正解」読み上げは残置。
- **音・ハプティクス追加（F-19 新規）**：音＝正解 / 不正解 / カウントダウン残り 3-2-1 秒 / ゲーム終了 / バッジ獲得。ハプティクス＝正解 / 不正解 / バッジ獲得。設定で個別 ON/OFF。サイレント時はハプティクスのみ。実装は `../rapidreading2/src/platform/audio.ts` の抽象を流用。
- **アプリアイコン差し替え（F-20 新規）**：iOS / Android adaptive / Android legacy / Web favicon / PWA / Expo splash の全プラットフォーム差し替え。図柄＝ガボールパッチ 45°（左下→右上、時計回り 45°）固定。
- **ステータスバー回避方針**：ゲーム画面はフルスクリーン許容、それ以外はセーフエリア準拠（iOS=SafeAreaInset、Android=StatusBar.currentHeight 自動取得）を NF として明記。
- **ガボールクリッピング品質基準**：「パッチ枠と縞模様の間に背景色が見えない」「角度を最小単位ずつ変えても枠との隙間が出ない」を NF に追加。
- **角度刻み記述方針**：「Designer/Generator が表示サイズと cpd から導出した最小可視差を採用する」と NF に明記（具体値は実装側裁量）。
- **起動時マイグレーション拡張（F-17）**：v1.2 起動時に staircase を**全ゲーム自動リセット**（角度自由化等で過去パラメータが無効化されるため）。v1.1 単体プレイ履歴も**リセット**。
- **スプリント計画刷新**：Sprint 22 を新設し、v1.2 改訂で影響を受けるスプリント（Sprint 9 G-01 / Sprint 12-14 G-04-06 / Sprint 11 G-03 / Sprint 14 G-07 / Sprint 17 G-13 / Sprint 8 起動フロー / Sprint 18 連続プレイ / Sprint 19 設定など）の再実装範囲を明示。新規スプリントとして音/振動・アイコン差し替えを Sprint 22 に統合。

## 0. サマリ／ビジョン

**GaborEye v1.2** は、ガボールパッチを土台にした **7 種類の「じーっと注視するミニゲーム」** を 1 セッション約 7〜8 分間連続プレイして、老眼ユーザーの V1（一次視覚野）レベルの視覚機能（コントラスト感度・周辺視・方位弁別・側方相互作用・テクスチャ統合・コントラスト感度）を **「注視持続そのものを訓練単位とする」** 思想で鍛えるアプリです。

v1.1.x までは 13 ゲームを試作し、ユーザー実機テストを通じて「面白い／継続したい」と判断されたものに絞り込んできました。v1.2 ではその選定結果として 7 ゲーム（G-01, G-03, G-04, G-05, G-06, G-07, G-13）を採用し、連続プレイ専用に動線を簡素化します。ゲーム一覧・単体プレイ・ホーム画面は廃止し、アプリ起動 = 連続プレイ秒読み画面が直接出る最短動線を実現します。

v1.2 で v1.1.x から本質的に変わる点：

1. **連続プレイ専用化**：起動 → 距離リマインド → 7 ゲーム順次プレイ → 事後画面 という単一動線。一覧・単体プレイ・ホーム画面を排除し、ユーザーが迷う場面を最小化
2. **3×3 固定 + 複数個ランダム + TP/FP 部分点**：G-01・G-04・G-05・G-06・G-07 が共通の構造（3×3 グリッド + 違うパッチが複数ランダム + 複数選択 + TP+1/FP-1/FN 0 部分点）を採用し、ユーザーが「お決まりの操作様式」を覚えやすい設計
3. **角度自由化**：v1.1.x まで「45° 単位」前提だった出題角度を自由化。Designer/Generator が表示サイズと cpd から導出した最小可視差を採用し、より細かい staircase 調整を可能にする
4. **音・ハプティクスのフィードバック追加**：注視訓練の妨げにならない範囲で、正解 / 不正解 / カウントダウン残り 3-2-1 秒 / ゲーム終了 / バッジ獲得時に音と振動を発火（個別 ON/OFF 可能）
5. **クリッピング品質基準の明示**：ガボール描画の品質を NF として明記（パッチ枠と縞模様の間に背景色が見えない／角度を最小単位ずつ変えても枠との隙間が出ない）

医療効果は引き続き断定せず、「自助セルフケア」として位置づける。

## 1. 老眼最適化原則（v1 から継承、横断規範）

v1 / v1.1 から OPT-1〜OPT-12 を継承する。v1.2 で文言調整があるものは別途明記。

- **OPT-1 大きい文字**：本文 18pt 以上、見出し 22pt 以上、ボタン文字 18pt 以上
- **OPT-2 大きいタップ領域**：操作可能要素は最小 48×48pt 以上、推奨 56pt
- **OPT-3 短い操作シーケンス**：1 タスクあたり最大 3 タップを目標
- **OPT-4 読書距離プリセット**：30 / 40 / 50cm の 3 段階スライダー、デフォルト 40cm
- **OPT-5 高コントラスト UI**：UI テキスト／ボタン背景のコントラスト比 7:1 以上（WCAG AAA 相当）。ガボール刺激そのものは別ロジック
- **OPT-6 優しい言い回し**：医療・専門用語は最小限。専門語使用時は括弧書きで補足
- **OPT-7 急かさない**：説明画面・事後画面はユーザー操作で進む。タイムアウトのある画面ではカウントダウンを 18pt 以上で明示
- **OPT-8 ダークモード必須**：OS 連動／明／暗の 3 択
- **OPT-9 片眼トレーニング配慮**：「片目を手で覆ってください」のガイダンスは大きいイラストと音声無しの静止表示
- **OPT-10 開始前の距離リマインド**：各セッション開始前に「画面から ◯◯cm 離れてください」を 18pt 以上で表示。**v1.1 から 3 秒カウントダウンで自動進行**
- **OPT-11 強制最小視聴時間**：各ゲームの 60 秒は早期終了不可。中断は緊急脱出用 × ボタン（中断＝セッション終了扱い）に限る
- **OPT-12 全ゲーム共通の注視フォーマット**：全ゲームは以下を共通項として持つ
  - 1 試行 = 60 秒固定（画像切替なし、フェードなし、点滅なし、マスクなし）
  - 注視中、ユーザーは選択肢のタップ／再タップで自由に何度でも回答を変更できる
  - 確定ボタンは存在しない
  - 60 秒経過で自動採点。採点時点の選択状態が回答とみなされる
  - 採点後は刺激画面にとどまり、結果オーバーレイ（パッチ上 ◯/✕ + 刺激領域直下の総合 ✅/❌）を表示。連続プレイ中は 10 秒カウントダウン後に次ゲームへ自動進行
  - 未回答のまま 60 秒経過した場合は「不正解」として記録し、staircase は易方向に動く
  - 試行中の正誤フィードバックは出さない（注視を妨げない）

## 2. 想定（Assumptions）

v1.1.x からの引継ぎ前提と、v1.2 で確定した事項（A0〜A15、`docs/questions-v12.md`）を統合して明示する。

- **A-1**：本アプリは医療機器ではない。医療効果は断定せず、薬機法を意識した文言で運用する（v1 から継承）
- **A-2 既存データは廃棄前提**：v1.2 起動時に v1.1.x 由来の永続化データ（特に staircase / 単体プレイ履歴）を**全リセット**する。理由は (a) 角度自由化で staircase 値が無効化、(b) 単体プレイモード自体の廃止。SessionRecord（連続プレイ履歴）・Streak・BadgeStatus は維持できる範囲で引き継ぐが、不整合のあるレコード（G-02 / G-08 を含む等）は破棄
- **A-3 ターゲットプラットフォーム**：iOS / Android（Expo Go / Web）一級サポート。Web ブラウザ（最小幅 360px〜最大 1280px）はレスポンシブ・キーボード操作・スクリーンリーダーを担保する
- **A-4 連続プレイ専用**：v1.1 まで「全 13 ゲーム連続コース」と「単体プレイ」を併設していたが、v1.2 では**連続プレイ 1 動線のみ**。順序固定（G-01 → G-03 → G-04 → G-05 → G-06 → G-07 → G-13）、1 周のみ、スキップ不可
- **A-5 1 試行 60 秒固定**：v1.1 から継続。連続プレイ全体の所要時間は約 7〜8 分（60 秒 × 7 + 結果画面 10 秒 × 6 + 距離リマインド 3 秒 + クールダウン 10 秒 ≒ 約 8 分）
- **A-6 staircase**：セッション間 1 step。連続 3 セッション正解で難化、1 セッション不正解で易化。閾値は直近 5 セッション平均（v1.1 から継続）
- **A-7 ガボール描画**：端末ごとの dpi メタデータと申告距離の組合せで cpd を近似（v1 から継承）。さらに v1.2 では NF として「パッチ枠と縞模様の間に背景色が見えない」「角度を最小単位ずつ変えても枠との隙間が出ない」を品質基準として明記
- **A-8 日本語ロケールのみ**。日付・週は ISO 週（月曜開始）で扱う
- **A-9 中断扱い**：× 中断は「未完了セッション」として扱い、スコア計算には含めない
- **A-10 年齢申告**：40 代／50 代／60 代／70 代以上。任意。70 代以上選択時は警告と医師相談推奨
- **A-11 角度刻みは実装側裁量**：v1.1 まで spec に「45°」「±N°」など具体値を書いていたが、v1.2 では「Designer/Generator が表示サイズと cpd から導出した最小可視差を採用する」とする（NF-26）
- **A-12 v1.2 の永続化キー**：`gaboreye:v1.2:*` プレフィックス（v1.1 とは名前空間を分離）

## 3. 対象ユーザー／ペルソナ

v1.1 から不変。

### 3.1 メインターゲット
**老眼層（40 代以上）**：手元の文字が見えづらくなった、夕方になると新聞が読みづらい、PC 作業で目が疲れるといった主観的不満を持つが医療機関には行くほどでない自助セルフケア指向のユーザー。

特に「**じっと一枚絵を見つめる時間を確保したい人**」「**短時間より滞在時間を重視する人**」を意識する。所要約 8 分は手軽さと持続的注視の両立を狙う。

### 3.2 対象外ユーザー
子ども（小学生以下）／70 代以上（任意申告で警告）／重度視覚障害者／強度近視（-6D 以上）／目に違和感のあるユーザー。

## 5. 機能一覧（フィーチャーリスト）

MECE な単位で **18 機能（F-01〜F-20、欠番 F-04 ホーム画面・F-06 単体ゲームモードを廃止）** を定義する。

> 注：v1.1 から **F-04（ホーム画面）と F-06（単体ゲームモード）は廃止**。F-04 / F-06 は欠番として保持し、ID は再採番しない。新規追加は F-19（音・ハプティクス）と F-20（アプリアイコン）。

### F-01 オンボーディング（90 秒以内）
- **目的**：初回起動から最初のゲーム 1 試行完了まで 90 秒以内で到達させる
- **主要な振る舞い**：起動時データ消去通知（初回 v1.2 起動時のみ） → 免責同意 → 年齢層申告 → 視聴距離設定 → 連続プレイ概要説明 → 1 試行体験
- **受け入れ基準**：
  - [ ] 初回起動時のみ自動表示。2 回目以降は表示されない
  - [ ] 起動時に v1 / v1.1.x 由来の永続化データが残っていれば対象キーを消去し、消去完了をユーザーに通知してからオンボーディングを開始する
  - [ ] 各ステップに「次へ」ボタンが 56pt 以上で配置される
  - [ ] 免責同意未チェックでは次へ進めない
  - [ ] 完了までの操作タップ数が 6 回以下
  - [ ] スキップ機能はない（必須通過）

### F-02 免責事項表示
- **目的**：医療機器でない旨と対象外ユーザーを明示
- **主要な振る舞い**：オンボーディング初回 + 設定からいつでも再閲覧 + 70 代以上選択時の追加警告
- **受け入れ基準**：
  - [ ] 文言が 18pt 以上で表示
  - [ ] 「同意する」ボタンを押すまで先に進めない
  - [ ] 設定 →「免責事項を読む」からいつでも再閲覧できる
  - [ ] 同意日時を端末ローカルに保存

### F-03 視聴距離キャリブレーション
- **目的**：刺激の cpd を端末ごとに近似再現する
- **主要な振る舞い**：端末タイプ自動推定 → 視聴距離スライダー（30 / 40 / 50cm の 3 ノッチ、デフォルト 40cm）→ プレビュー

### 6.3 staircase アルゴリズム（v1.1 から継続）

- **方式**：セッション間 3-down/1-up（連続 3 セッション正解で 1 step 難化、1 セッション不正解で 1 step 易化）
- **step size**：初期 1 unit（パラメータ依存。各ゲーム個別、Designer/Generator 判断）
- **閾値推定**：直近 5 セッションのパラメータ平均値（5 セッション未満なら直近全件平均）
- **状態の永続化**：ゲームごとに最終パラメータ・直近結果リストを保存
- **リセット**：設定 →「staircase をリセット」で各ゲームの初期パラメータに戻す。**v1.2 初回起動時は全ゲーム自動リセット**
- **クランプ**：パラメータが定義された min/max を超える場合はクランプ
- **正誤判定（部分点ゲーム）**：G-01・G-04〜G-07 は「TP - FP > 0 なら正解、それ以外は不正解」とする二値判定で staircase を駆動

### 6.4 ガボールクリッピング品質基準（v1.2 新規、A14 確定）

- **NF-27 ガボール枠と縞の一致**：パッチの矩形枠（または円形マスク）と縞模様の間に背景色（グレー）が見えてはならない。実装は「大きめのパッチを四角枠でクリッピング」する方式が想定される
- **NF-28 角度変化時の枠隙間禁止**：角度を最小単位（NF-26 で定義）ずつ変化させても、パッチ枠と縞模様の間に隙間や欠けが生じてはならない

- **進行**：60 秒間ずっと同一 3×3 グリッドが表示。回転アイテムは時計回り／反時計回りどちらかをアイテムごとにランダムに選ばれて、staircase で決まる速度でゆっくり回転し続ける。静止アイテムはそれぞれ異なるランダム角度で固定（角度自由化、NF-26）
- **グリッドサイズ**：**3×3 固定**（A2 確定）
- **回転アイテム個数**：**1〜N 個ランダム**（N と分布は Designer/Generator 判断）。同セッション内では固定

- **回転方向**：時計回り / 反時計回り の **両方向ランダム**（アイテムごとに独立）
- **選択肢**：3×3 グリッド内の各パッチ 9 個。**複数選択**（タップで選択トグル）
- **メッセージ**：「**回転しているパッチをタップ**」（方向情報・個数情報は含めない、A2 確定）
- **採点**：**TP+1 / FP-1 / FN 0 の部分点**。staircase 駆動の正誤判定は「TP - FP > 0 なら正解」

- **採点（v1.2 改訂、A11 確定）**：**TP+1 / FP-1 / FN 0 の部分点**（他 3×3 系と統一）
- **staircase パラメータ**：「向き同士のズレ許容角」（易：ズレ許容大 → 難：ズレ許容小、具体値は Designer/Generator 判断）

### 7.13 G-13 数字探し（Embedded Numeral、v1.2 領域拡大）
- **コンセプト**：ガボールパッチの集合体やノイズ画像の中に、低コントラストで「数字」が埋め込まれている。60 秒間ずっと見つめて、何の数字かを当てる
- **注視ポイント**：注視を続けることで埋もれた形が浮かび上がる「ポップアウト効果」。Vision Therapy で使われる古典的な刺激で、コントラスト感度の訓練に直結する
- **進行**：ノイズ + 微小コントラストの数字（0〜9 のいずれか 1 つ）を 60 秒間表示 → 採点
- **領域配分（v1.2 改訂、A4 確定）**：**安全領域内で上 70% を刺激領域、下 30% を keypad 領域** に最大化する
- **選択肢**：keypad-10 ボタン（0〜9）を維持。配置は刺激画像直下で空間対応がつく形（テンキー風 3×4 等、Designer 判断）
- **採点**：選択された数字が埋め込まれた数字と一致なら正解。**単一選択**
- **staircase パラメータ**：数字のコントラスト（易：高コントラスト → 難：低コントラスト、具体値は Designer/Generator 判断）
- **結果指標**：正誤 / 今回の閾値
- **正解開示（F-10 統一）**：刺激画面のまま、正解の数字ボタンに ✅、ユーザーが誤選択した場合のみ誤選択ボタンに ❌、刺激領域直下に試行全体総合 ✅/❌ をオーバーレイ表示
- **受け入れ基準**：
  - [ ] 安全領域内で上 70% が刺激領域、下 30% が keypad 領域となる
  - [ ] keypad-10 ボタンの具体配置（3×4 / 1×10 等）は `docs/design-v11/sprints/sprint-22/screens.md` に Designer が明記する
  - [ ] 結果オーバーレイ ✅/❌ がユーザーの選択ボタンと正解ボタンの上に重畳される
  - [ ] 他のゲーム（G-01・G-03〜G-07）の刺激領域は v1.2 では現状維持（拡大対象は G-13 のみ、A4 確定）

> **§7 まとめ**：v1.2 では 7 ゲーム（G-01, G-03, G-04, G-05, G-06, G-07, G-13）に絞り、G-01・G-04・G-05・G-06・G-07 は「3×3 グリッド + 違うパッチ複数ランダム + 複数選択 + TP/FP 部分点」の共通フォーマットに統一（G-07 のみグリッドサイズは Designer 判断）。G-03 は単一選択、G-13 はキーパッド単一選択。G-02 / G-08 / G-09 / G-10 / G-11 / G-12 は廃止または対象外。

### 8.1 起動シーケンス（v1.2 改訂）

```
[アプリ起動]
```

### 12.8 角度自由化と描画品質（v1.2 新規）
- **NF-26 角度刻みは Designer/Generator 判断**：v1.2 では出題角度を自由化（離散 4 択を撤廃）。最小単位は **Designer/Generator が表示サイズ・cpd・端末性能から導出した最小可視差を採用する**。spec には具体値を記載しない。実装後 Designer のレビューと Planner のレビューを経て確定し、必要に応じて `docs/design-v11/system.md` に明記する
- **NF-27 ガボール枠と縞の一致**：パッチの矩形枠（または円形マスク）と縞模様の間に背景色（グレー）が見えてはならない（A14 確定）
- **NF-28 角度変化時の枠隙間禁止**：角度を最小単位（NF-26）ずつ変化させても、パッチ枠と縞模様の間に隙間や欠けが生じてはならない（A14 確定）

### 12.9 ステータスバー回避（v1.2 新規、A9 確定）
- **NF-29 ゲーム画面はフルスクリーン許容**：刺激提示中（G-01・G-03〜G-07・G-13）は OS ステータスバー領域も使用してよい（フルスクリーン化を許容）

### 12.10 音・ハプティクス品質（v1.2 新規、F-19 と整合）
- **NF-31 音再生レイテンシ**：ユーザー操作（タップ）から音再生開始まで 100ms 以内
- **NF-32 ハプティクス強度**：iOS は UIImpactFeedbackGenerator.style.medium 相当、Android は VIBRATOR_STRONG_FREQUENCY 相当をデフォルトとする。具体実装は Generator 判断、`../rapidreading2/src/platform/audio.ts` の抽象を流用
- **NF-33 サイレントモード尊重**：OS のサイレント設定が ON のとき音は鳴らさず、ハプティクス（振動 OFF でなければ）のみ発火する

### 13.2 Sprint 22 実装順序の推奨

依存順に並べると以下：

1. **22-A**（起動フロー変更とホーム廃止）— 他のサブタスクの前提となる動線変更
2. **22-B**（削除対象ゲームのコード除去）— gameRegistry を整理してから他ゲーム改修
3. **22-N**（マイグレーション拡張）— 既存ユーザーデータへの影響を早期確定
4. **22-H**（カウントダウン UI 統一）+ **22-I**（F-10 改訂）— UI 共通基盤
5. **22-M**（クリッピング品質）+ **22-J**（音・ハプティクス）— ゲーム横断の品質基盤
6. **22-C**（G-01）→ **22-D**（G-03）→ **22-E**（G-04/05/06）→ **22-F**（G-07）→ **22-G**（G-13）— ゲーム個別改修
7. **22-L**（ステータスバー回避）+ **22-K**（アイコン差し替え）— リリース前最終調整

### 13.3 スプリント間の原則
- Sprint 22 終了時に `npm test` のテスト数が増えていること（**目標：v1.1 完了時 +約 130 件**）
- Sprint 22 はサブタスク単位で PR を分割してよい（22-A〜22-N の 14 サブタスク）
- 設計影響が大きい 22-A / 22-B / 22-N は Designer のレビューを経てから着手
- Designer は Sprint 22 用に `docs/design-v11/sprints/sprint-22/screens.md` を作成し、(1) 起動フロー（即 F-16）、(2) F-21 事後画面、(3) F-07.1 統一カウントダウン、(4) F-10 改訂版、(5) G-01 / G-03 / G-04-06 / G-07 / G-13 各画面、(6) F-20 アイコン原画、(7) 設定画面の音/振動トグル、を一括定義する
