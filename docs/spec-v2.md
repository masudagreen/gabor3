## 0. v1.2 → v2.0 改訂履歴（本質的変更点）

v2.0 は v1.2 からの「追加改訂」ではなく **作り直し（リブート）** である。主要な転換点：

| 改訂対象 | 旧（v1.2） | 新（v2.0） | 理由 |
|---|---|---|---|
| **ゲーム構成** | 7 種類のミニゲーム（G-01, G-03〜G-07, G-13）を順序固定で連続プレイ | **1 種類に統合**：n×n のガボールパッチ格子で「ゆっくり変化（回転＋空間周波数）しているパッチをすべてタップ」する変化検出ゲーム | 多種ゲーム維持の複雑さを排し、最も注視時間と相関しやすい「回転検出」を核に単純化 |
| **変化の種類** | ゲームごとに弁別軸が異なる（回転/コントラスト/cpd/SD/向き/数字） | **回転（a deg/sec）と空間周波数（b hz/sec）の 2 種が 1 ラウンド内で同時混在** | 「動いているものを見つける」という単一の認知タスクに統合 |
| **ユーザーの回答** | ゲームごとに単一選択/複数選択/キーパッドが混在 | **変化しているパッチをすべてタップするのみ**（種類は区別しない） | 操作様式を 1 つに固定し学習負荷を最小化 |
| **1 単位の構成** | 7 ゲーム × 60 秒固定 = 1 セッション（約 8 分、1 日 1 周） | **1 ラウンド m 秒 × r ラウンド = 1 セッション。何度でも再プレイ可** | 気軽な反復プレイへ。所要時間をユーザーが調整可能に |
| **採点タイミング** | 60 秒固定・確定ボタンなし（OPT-11/12） | **3 方式を設定で切替**：①自動採点・確定なし ②自動採点・確定あり ③全問正解で即次へ | プレイ感の好みに応じて選べるようにする |
| **難易度調整** | セッション間 staircase（3-down/1-up） | **当面 staircase 廃止**。n/m/r/a/b を設定タブで手動固定。将来「漸進難化モード」を追加できる設計余地のみ残す | ユーザーが実機で好みのパラメータを確定する運用へ。自動調整は後日 |
| **画面構成** | 起動 = 即連続プレイ。完了後 F-21 事後画面が唯一のホーム的役割 | **下タブ 3 つ（ホーム / 履歴 / 設定）**。ゲームはホームタブでプレイ | 進捗・設定への常時アクセスを確保しつつ、起動即プレイは維持 |
| **中断** | ゲーム画面右上 × ボタンのみ | **× ボタンおよび他タブ選択の両方**で同一の確認ダイアログ（プレイ中のみ） | タブ構成に合わせた中断 UX |
| **進捗指標** | ワイドスコア（7 ゲーム平均）+ ゲーム別閾値タブ | **日次スコア（正答率ベース 0〜100）/ 連続日数ストリーク / プレイ回数累計** | 単一ゲームに即した指標へ |
| **バッジ** | B-01〜B-12（7 ゲーム前提、ゲーム依存条件あり） | **継続日数 / 高難度クリア / 高スコア の 3 軸で新規定義**。旧バッジ全廃 | ゲーム依存条件が無効化したため |
| **データ** | `gaboreye:v1.2:*` | **`gaboreye:v2:*` で新規初期化。起動時に旧データ全リセット** | スキーマ非互換のため |
| **継承する横断規範** | OPT-1〜12、F-19 音/ハプティクス、クリッピング品質、セーフエリア | **OPT-1〜12 を継承（OPT-11/12 は m 秒可変＋3 採点方式に置換、共通注視フォーマットを新ゲームに合わせ改訂）。F-19・NF 群は継承** | 老眼配慮と品質基準は引き続き必須 |
| **アイコン** | v1.2 で差し替え済 | **現状維持（差し替えはスコープ外）** | 今回は機能リブートに集中 |
| **オンボ/距離/クールダウン** | 90 秒オンボ・距離リマインド・クールダウンあり | **初回のみ簡素オンボ＋距離リマインドは継続。眼精疲労クールダウンは廃止** | 気軽な再プレイ方針に合わせ簡素化 |

## 1. 想定（Assumptions）

questions.md の Q&A（2026-05-30 確定）を反映する。must（Q1〜11）はユーザー確定回答、nice-to-have（Q12〜17）はユーザー明示回答がないため合理的仮定を採用し、ここに明示する。

### 1.1 確定事項由来の前提（Q1〜11）
- **AS-1**：変化は **回転と空間周波数が 1 ラウンド内で同時混在**する。回転速度 a（deg/sec）と空間周波数変化速度 b（hz/sec）を両方設定する（Q1）。
- **AS-2**：ユーザーは**変化しているパッチをすべてタップ**するだけ。変化の種類（回転か周波数か）は区別して答えない（Q2）。
- **AS-3**：採点は **3 方式を設定で切替**（Q3、詳細は F-02）。
- **AS-4**：スコアは**ラウンドごと TP+1 / FP-1 / FN 0 の部分点**で集計し、1 セッション（r ラウンド 1 周）を**正答率ベースの 0〜100 単一スコア**にまとめる（Q4）。
- **AS-5**：n / m / r / a / b は**当面すべて手動固定**（設定タブ）。staircase は当面オフ。**将来の漸進難化モードを実装できる設計余地のみ残す（v2.0 では実装しない）**（Q5）。
- **AS-6**：起動フローは「起動 →（初回のみ簡素オンボ）→ 距離リマインド → ホームタブでゲーム自動開始」（Q6）。
- **AS-7**：r ラウンド完了後はホームにセッション結果を表示し、「もう一度」で**何度でも再プレイ可**（1 日 1 周制限は撤廃）（Q7）。
- **AS-8**：中断ダイアログは**プレイ中（ラウンド進行中）のみ**、× ボタンおよび他タブ選択で表示。**OK = セッション中断（記録しない）/ キャンセル = 継続**の 2 択。完了後・待機中のタブ移動はダイアログなしで自由（Q8）。
- **AS-9**：履歴グラフは **日次スコア（正答率ベース）/ 連続日数ストリーク / プレイ回数累計**をプロット（Q9）。
- **AS-10**：バッジは**継続日数 / 高難度クリア / 高スコア**の 3 軸で新規定義。旧バッジは全廃（Q10、詳細は §5）。
- **AS-11**：起動時に旧データ（v1/v1.1/v1.2 由来）を**全リセット**し、新名前空間 `gaboreye:v2:*` で初期化（Q11）。

### 1.2 合理的仮定（Q12〜17、ユーザー明示回答なし）
- **AS-12**：変化しないパッチは**ランダムな初期角度・初期空間周波数で静止固定**。これにより「変化中か静止か」の弁別が課題となる（Q12）。
- **AS-13**：回転方向は時計回り/反時計回りを**パッチごとに独立してランダム**。空間周波数変化は**増加方向/減少方向どちらもあり得る**。変化は m 秒間（または採点まで）**一定速度で継続**する（Q13）。
- **AS-14**：設定タブには新パラメータ（n/m/r/a/b + 採点方式①②③ + 視聴距離）に加え、既存の **ダークモード / 音 ON-OFF / 振動 ON-OFF / 片眼ガイダンス / 免責事項再閲覧 / 全データ削除**を残す（Q14）。
- **AS-15**：オンボーディングは**初回のみ簡素版**を残す。距離リマインドは残す。**眼精疲労クールダウン（遠くを見る画面）は廃止**（Q15）。
- **AS-16**：結果開示は「変化パッチ上に ✅、誤選択に ❌、刺激領域直下に総合 ✅/❌」を踏襲。ラウンド間遷移は採点方式に依存（方式③＝全問正解で即次へ、方式①②＝採点後の短いインターバルで次へ。旧 10 秒固定は廃止し短縮、具体秒数は Designer 判断）（Q16）。
- **AS-17**：v2.0、アプリ名 GaborEye 維持、アプリアイコンは**現状維持**（差し替えはスコープ外）（Q17）。

### 1.3 横断的前提（v1.2 から継承）
- **AS-18**：本アプリは医療機器ではない。医療効果は断定せず、薬機法を意識した文言で運用する。
- **AS-19**：ターゲットプラットフォームは iOS / Android（Expo Go / Web）一級サポート。Web ブラウザは最小幅 360px〜最大 1280px でレスポンシブ・キーボード操作・スクリーンリーダーを担保。
- **AS-20**：日本語ロケールのみ。日付・週は ISO 週（月曜開始）。i18n キー基盤は維持。
- **AS-21**：完全オフライン動作。ネットワーク通信なし。
- **AS-22**：年齢申告は 40 代/50 代/60 代/70 代以上（任意）。70 代以上選択時は警告と医師相談推奨。
- **AS-23**：各パラメータ（n/m/r/a/b）の**可動範囲と初期値は spec で具体値を断定しない**。Designer/Generator が表示サイズ・cpd・体感から妥当な範囲と初期値を仮置きし、ユーザーが実機で試して確定する。

## 2. ビジョン / 課題 / ターゲット

### 2.1 ビジョン
**GaborEye v2.0** は、ガボールパッチを土台にした **単一の「変化検出ゲーム」** を、好きなだけ気軽に繰り返してプレイできるアプリ。老眼層が「ゆっくり動くものをじっと見て見つける」体験を通じて、V1（一次視覚野）レベルの視覚機能（運動方向検出・空間周波数感度・注視持続）を自助セルフケアとして鍛える。

医療効果は断定せず、「自助セルフケア」として位置づける。

### 2.2 課題
- v1.2 の 7 ゲーム連続プレイ（約 8 分・1 日 1 周）は、操作様式がゲームごとに異なり学習負荷が高く、また「1 日 1 周」の制約が気軽な反復を妨げていた。
- 多種ゲームの維持コストが高く、ユーザーが「どれが面白いか」を判断するための試作フェーズが長期化していた。

### 2.3 ターゲットユーザー
v1.2 から不変。

- **メインターゲット**：老眼層（40 代以上）。手元の文字が見えづらい、夕方に新聞が読みづらい等の主観的不満を持つが医療機関には行くほどでない自助セルフケア指向のユーザー。特に「じっと一枚絵を見つめる時間を確保したい人」を意識する。
- **対象外**：子ども（小学生以下）/ 70 代以上（任意申告で警告）/ 重度視覚障害者 / 強度近視（-6D 以上）/ 目に違和感のあるユーザー。

## 3. ユーザーストーリー / ユースケース

- **US-1（初回起動）**：初めて起動したユーザーとして、簡素なオンボーディングで免責に同意し視聴距離を設定したうえで、すぐに 1 ラウンドを体験したい。何のゲームかを 90 秒以内に理解できること。
- **US-2（日常プレイ）**：継続中のユーザーとして、アプリを開いたら距離リマインドの後すぐホームタブでゲームが始まり、変化しているパッチをタップしていきたい。
- **US-3（反復）**：1 セッション（r ラウンド）を終えたユーザーとして、結果を見てから「もう一度」で何度でも追加プレイしたい。
- **US-4（中断）**：プレイ中に用事ができたユーザーとして、× ボタンまたは他タブを押したときに確認ダイアログで中断でき、中断分は記録に残らないでほしい。
- **US-5（進捗確認）**：続けているユーザーとして、履歴タブで日次スコアの推移・連続日数・累計プレイ回数とバッジを見て達成感を得たい。
- **US-6（難易度調整）**：手応えを調整したいユーザーとして、設定タブで n/m/r/a/b と採点方式を変えて、自分に合った難しさを見つけたい。
- **US-7（個別配慮）**：片眼トレーニングをしたいユーザーとして、設定で片眼ガイダンスを有効にしたい。音・振動も個別に切り替えたい。

## 4. 機能一覧（F-01〜F-14）

> 各機能の受け入れ基準は**ユーザーから観察できる挙動**で記述する。閾値・速度・グリッドサイズ等の具体値は Designer/Generator 判断（AS-23）。

### F-01 変化検出ゲーム（コア）
- **目的**：n×n のガボールパッチ格子の中で、ゆっくり変化（回転＋空間周波数）しているパッチをすべて見つけてタップさせる。
- **主要な振る舞い**：
  - n×n のガボールパッチを格子状に表示する。
  - そのうち複数個（ランダム個数）が**ゆっくり変化**する。変化は **回転（速度 a deg/sec）と空間周波数（速度 b hz/sec）が同時混在**しうる（あるパッチは回転、別のパッチは周波数変化、両方が同一パッチで起こることもありうる。具体的な混在分布は Designer/Generator 判断）。
  - 変化しないパッチは**ランダムな初期角度・初期空間周波数で静止固定**（AS-12）。
  - 回転方向は時計回り/反時計回りをパッチごとに独立ランダム、周波数変化は増減どちらもあり（AS-13）。
  - ユーザーは変化していると思うパッチを**直接タップ**して選択（トグル）。種類は区別しない。
  - 1 ラウンドの制限時間は **m 秒**。これを **r ラウンド**繰り返して 1 セッション。
- **受け入れ基準**：
  - [ ] n×n のガボールパッチ格子が表示される（n は設定値）
  - [ ] 1 ラウンド内に、回転変化のパッチと空間周波数変化のパッチが同時に存在しうる
  - [ ] 変化パッチの個数はラウンドごとにランダムで、画面に個数は表示されない
  - [ ] 変化しないパッチはランダムな初期角度・初期周波数で静止している
  - [ ] ユーザーはパッチを直接タップして選択/選択解除でき、選択中のパッチが視覚的に明示される（控えめな枠など、ガボール本体の視認を阻害しない形）
  - [ ] 変化の種類（回転/周波数）をユーザーが指定する UI は存在しない
  - [ ] 「現在の回答：◯◯」のようなテキスト表示は行わない
  - [ ] 1 ラウンドの制限時間 m 秒のカウントダウンが画面上部に 22pt 以上で常時表示される（F-12 カウントダウン仕様に従う）
  - [ ] 変化は m 秒間（または採点まで）一定速度で継続する
  - [ ] r ラウンドを連続実行して 1 セッションが構成される

### F-02 採点方式（3 方式・設定切替）
- **目的**：ラウンドの締め方・採点タイミングをユーザーの好みに合わせて選べるようにする。
- **主要な振る舞い**：設定タブで以下 3 方式から 1 つを選ぶ（AS-3）。
  - **方式①（自動採点・確定なし）**：m 秒経過で自動採点。確定ボタンなし。選択は m 秒間何度でも変更可（旧 OPT-11/12 踏襲）。
  - **方式②（自動採点・確定あり）**：m 秒経過で自動採点。加えて**確定ボタン**があり、m 秒以内に確定すると早期に締め切れる。
  - **方式③（全問正解で自動遷移）**：選択が**全問正解（変化パッチを過不足なく選択）になった時点で次ラウンドへ自動遷移**。m 秒は上限タイムアウトとして機能し、m 秒経過時点で採点する。
- **採点ロジック（全方式共通）**：選択したパッチのうち、変化パッチ＝TP（True Positive、+1）、静止パッチ＝FP（False Positive、−1）、選ばなかった変化パッチ＝FN（False Negative、0）。ラウンド得点 = TP − FP（下限・正規化は F-04 参照）。
- **受け入れ基準**：
  - [ ] 設定で方式①②③のいずれかを選べ、選択がプレイに反映される
  - [ ] 方式①：確定ボタンが画面に存在せず、m 秒経過で自動採点される。m 秒内は選択を何度でも変更できる
  - [ ] 方式②：確定ボタンが存在し、押すと m 秒を待たず即採点される。押さなければ m 秒経過で自動採点される
  - [ ] 方式③：変化パッチを過不足なく選択した時点で、m 秒を待たずに次ラウンドへ自動遷移する。誤選択や選択漏れがある間は遷移しない。m 秒経過で強制採点される
  - [ ] 全方式で、採点時に TP+1 / FP−1 / FN 0 の部分点が算出される
  - [ ] 採点時点の選択状態が回答とみなされる（方式①②）
  - [ ] 未選択のまま m 秒経過した場合も採点される（TP=0、FP=0）
  - [ ] 試行中（採点前）は正誤フィードバックを出さない（注視を妨げない）

### F-03 結果開示（ラウンド単位 ✅/❌ オーバーレイ）
- **目的**：ラウンド採点直後に「変化していたパッチ」と「ユーザーの誤選択」を、刺激画面にとどまったまま開示する（AS-16）。
- **主要な振る舞い**：採点後、画面遷移せず、各パッチ上に ✅/❌ を、刺激領域直下に試行全体の総合 ✅/❌ を 1 個重畳表示する。
- **受け入れ基準**：
  - [ ] 採点後、独立した結果画面に遷移せず、ガボール格子はそのまま画面上に残る
  - [ ] 変化していたパッチ上に ✅、ユーザーが誤って選んだ静止パッチ上に ❌ をオーバーレイ表示する
  - [ ] 正しく選んだ変化パッチ（TP）と、正解だが選ばなかった変化パッチ（FN）の区別は視覚的に付く（区別方法は Designer 判断、例：TP は実線 ✅・FN は薄い ✅）
  - [ ] 刺激領域直下に、そのラウンド全体の総合 ✅（緑）/ ❌（赤）を 1 個表示する
  - [ ] 閾値・速度・前回比などの数値テキストは結果開示に一切表示しない
  - [ ] ✅/❌ アイコンはガボール本体の縞模様を完全には覆い隠さない透過度・サイズで配置される（具体値は Designer 判断）
  - [ ] スクリーンリーダー用に「正解」「不正解」が読み上げられる
  - [ ] 結果開示中は選択肢のタップ操作が無効化される（採点後の選択変更を防ぐ）
  - [ ] 方式③で全問正解により即遷移した場合の開示扱いは Designer 判断（即次へ遷移する場合は短い正解フィードバックで足りる）
  - [ ] 方式①②では採点後の短いインターバル（旧 10 秒固定は廃止、短縮、具体秒数は Designer 判断）の後に次ラウンドへ進む

### F-04 セッションスコア集計（0〜100 単一スコア）
- **目的**：r ラウンド分の部分点を、正答率ベースの 0〜100 単一スコアに集計する（AS-4）。
- **主要な振る舞い**：各ラウンドの TP/FP/FN を集計し、セッション全体の正答率を 0〜100 に正規化する。正規化方式（例：全変化パッチに対する TP 率と、誤タップによる減点の合算を 0〜100 にマップ）の具体式は Generator 判断とするが、観察可能な性質を以下に定める。
- **受け入れ基準**：
  - [ ] セッション完了時に 0〜100 の整数スコアが算出・表示される（医療数値「視力 0.X」は使わない）
  - [ ] 全ラウンドで変化パッチをすべて正しく選び誤選択ゼロのとき、スコアは満点（100）になる
  - [ ] 変化パッチを 1 つも選ばず誤選択もしないとき、スコアは下限（0）寄りになる（具体下限は Generator 判断だが、満点と明確に区別される）
  - [ ] 誤選択（FP）はスコアを下げる方向に作用する
  - [ ] 中断したセッションはスコア集計・記録の対象外（F-07）
  - [ ] セッションスコアは日次集計（F-09）・履歴グラフ（F-09）に渡される

### F-05 ボトムタブナビゲーション（ホーム / 履歴 / 設定）
- **目的**：3 つの主要画面を画面下部のタブで常時切り替えられるようにする。
- **主要な振る舞い**：画面下部に「ホーム」「履歴」「設定」の 3 タブを配置。ゲームはホームタブでプレイする。
- **受け入れ基準**：
  - [ ] 画面下部に「ホーム」「履歴」「設定」の 3 タブが常時表示される
  - [ ] 各タブをタップすると対応する画面に切り替わる
  - [ ] 現在選択中のタブが視覚的に明示される（色＋形/ラベルで、色のみに依存しない）
  - [ ] 各タブのタップ領域は 48pt 以上
  - [ ] **プレイ中（ラウンド進行中）に他タブをタップした場合は、中断確認ダイアログ（F-07）を出す**
  - [ ] **完了後・待機中（ゲーム非進行中）のタブ移動はダイアログなしで自由**
  - [ ] Web 版でキーボード（Tab/Enter）でタブを操作できる

### F-06 起動フロー（初回オンボ → 距離リマインド → 自動開始）
- **目的**：起動から最短でゲーム開始に到達させつつ、初回のみ必要な同意・設定を済ませる（AS-6, AS-15）。
- **主要な振る舞い**：
  - 起動 →（旧データ検出時は F-11 リセット）→（初回のみ）簡素オンボーディング → 距離リマインド → ホームタブでゲーム自動開始。
  - 簡素オンボーディング：免責同意（F-10）→ 年齢層申告 → 視聴距離設定 → ゲーム概要のごく短い説明。
  - 距離リマインド：「画面から ◯◯cm 離れてください」を表示し、短いカウントダウンで自動進行（OPT-10）。
- **受け入れ基準**：
  - [ ] 初回起動時のみオンボーディングが表示され、2 回目以降は表示されない
  - [ ] オンボーディング完了までの操作タップ数が 6 回以下
  - [ ] 免責同意未チェックでは先に進めない
  - [ ] オンボーディング完了後、および 2 回目以降の起動では、距離リマインドを経てホームタブでゲームが自動開始する
  - [ ] 距離リマインドは「画面から ◯◯cm 離れてください」を 18pt 以上で表示し（◯◯ は設定値）、カウントダウン（F-12 仕様）で自動進行する
  - [ ] 距離リマインドはユーザー操作（× 中断以外）なしで自動進行する
  - [ ] 起動からゲーム開始までに眼精疲労クールダウン画面は存在しない（廃止）

### F-07 中断ダイアログ（プレイ中のみ）
- **目的**：プレイ中の離脱時に、誤操作防止と「中断は記録しない」挙動を明示する（AS-8）。
- **主要な振る舞い**：プレイ中（ラウンド進行中）に、ホーム左上の × ボタンまたは他タブ選択をトリガーに、同一の確認ダイアログを表示する。
- **受け入れ基準**：
  - [ ] プレイ中に × ボタンを押すと確認ダイアログが出る
  - [ ] プレイ中に他タブをタップしても、× ボタンと同一の確認ダイアログが出る
  - [ ] ダイアログの選択肢は「OK（中断・記録しない）」「キャンセル（継続）」の 2 択
  - [ ] OK を選ぶとセッションが中断され、進行中のセッションは記録に残らない。× 起点ならゲーム終了、タブ起点なら当該タブへ遷移する
  - [ ] キャンセルを選ぶとダイアログが閉じ、ゲームが継続する（残り時間・選択状態が保持される）
  - [ ] ゲーム完了後・待機中（非進行中）は、× ボタン・タブ移動のいずれでもダイアログを出さず自由に遷移できる
  - [ ] ダイアログのボタンは 48pt 以上

### F-08 ホームタブ（ゲーム＋セッション結果＋再プレイ）
- **目的**：ホームタブをゲームのプレイ場所兼セッション結果の表示場所にする（AS-7）。
- **主要な振る舞い**：
  - 起動直後（距離リマインド後）はホームタブでゲームが自動開始する。
  - r ラウンド完了後、ホームタブにセッション結果（0〜100 スコア＋今日のストリーク等）を表示する。
  - 「もう一度」ボタンで何度でも再プレイできる。
- **受け入れ基準**：
  - [ ] ホームタブでゲームがプレイされる
  - [ ] r ラウンド完了後、ホームタブにそのセッションの 0〜100 スコアが表示される
  - [ ] 今日のストリーク（連続日数）が表示される
  - [ ] 「もう一度」ボタンが 56pt 以上で配置され、押すと（距離リマインドを経て）新しいセッションが始まる
  - [ ] 「もう一度」は 1 日の回数制限なく何度でも押せる
  - [ ] 結果表示中はゲーム非進行状態として扱われ、タブ移動が自由（F-05 / F-07）

### F-09 履歴タブ（進捗グラフ＋バッジ）
- **目的**：継続の成果を可視化して達成感を与える（AS-9）。
- **主要な振る舞い**：日次スコア・連続日数ストリーク・累計プレイ回数のグラフと、獲得/未獲得バッジ一覧を表示する。
- **受け入れ基準**：
  - [ ] **日次スコア（正答率ベース 0〜100）** の推移が折れ線等のグラフで表示される
  - [ ] 同日に複数セッションがある場合、その日の代表値（最良スコア＝max）が用いられる
  - [ ] **連続日数（ストリーク）** が現在値で表示される
  - [ ] **累計プレイ回数** が表示される
  - [ ] グラフの軸ラベルは 18pt 以上
  - [ ] 当日のスコアがグラフ上で強調される
  - [ ] データが少ない初期（目安 7 日未満）は「もう少しデータが集まると傾向が見えます」等の案内を表示する
  - [ ] バッジ一覧（獲得済み/未獲得）が確認でき、未獲得バッジには獲得条件のヒントが表示される
  - [ ] グラフ・バッジ表示に点滅エフェクトを使用しない

### F-10 免責事項表示
- **目的**：医療機器でない旨と対象外ユーザーを明示する。
- **主要な振る舞い**：オンボーディング初回 + 設定からいつでも再閲覧 + 70 代以上申告時の追加警告。
- **受け入れ基準**：
  - [ ] 文言が 18pt 以上で表示される
  - [ ] 「同意する」ボタンを押すまで先に進めない（オンボーディング時）
  - [ ] 設定 →「免責事項を読む」からいつでも再閲覧できる
  - [ ] 同意日時を端末ローカルに保存する
  - [ ] 70 代以上を申告した場合に追加警告と医師相談推奨を表示する

### F-11 起動時データリセット（v2 初期化）
- **目的**：v2.0 リブートに伴い、互換性のない旧データを完全消去してクリーン状態で開始する（AS-11）。
- **主要な振る舞い**：起動時に v1/v1.1/v1.2 由来の永続化キー（`gaboreye:v1:*` / `gaboreye:v1.1:*` / `gaboreye:v1.2:*`）を検出したら消去し、新名前空間 `gaboreye:v2:*` で初期化。消去完了をユーザーに 1 度だけ通知する。
- **受け入れ基準**：
  - [ ] 初回 v2.0 起動時に旧名前空間の永続化キーがすべて消去される
  - [ ] 消去後、`gaboreye:v2:*` のキーで初期化される
  - [ ] 消去完了後「最新版へのアップデートのため、過去データをリセットしました」を 1 度だけ表示する
  - [ ] 通知は OK ボタン（56pt 以上）で進められる
  - [ ] 2 回目以降の起動では本通知は表示されない
  - [ ] 削除は端末ローカルのみ（クラウドはない）

### F-12 カウントダウン UI 共通仕様
- **目的**：m 秒ゲームタイマー・距離リマインドのカウントダウンを統一フォーマットにする（v1.2 F-07.1 を継承）。
- **受け入れ基準**：
  - [ ] 数字のみ表示（時計アイコンは使わない）
  - [ ] 通常時の文字色は白、残り 5 秒以下で黄、残り 3 秒以下で赤に切り替わる
  - [ ] 色変化は太字化で補強する（色のみで情報を伝えない原則を満たす）
  - [ ] 点滅エフェクトは使用しない
  - [ ] m 秒ゲームタイマーと距離リマインドの両方に一律適用される

### F-13 設定タブ
- **目的**：ユーザー個別の調整ポイントを集約する（AS-5, AS-14）。
- **主要な振る舞い**：以下の項目を提供する。
  - **新パラメータ**：n（格子サイズ）/ m（1 ラウンド秒数）/ r（ラウンド数）/ a（回転速度 deg/sec）/ b（周波数変化速度 hz/sec）
  - **採点方式**：①②③の選択（F-02）
  - **視聴距離**：30/40/50cm
  - **継承項目**：ダークモード / 音 ON-OFF / 振動 ON-OFF / 片眼ガイダンス / 免責事項再閲覧 / 全データ削除
- **受け入れ基準**：
  - [ ] n / m / r / a / b を設定でき、変更がプレイに反映される
  - [ ] 各パラメータには可動範囲があり、範囲外の値は設定できない（範囲・初期値は Designer/Generator 仮置き、AS-23）
  - [ ] 採点方式①②③を選択できる
  - [ ] 視聴距離（30/40/50cm、デフォルト 40cm）を選択でき、即座に保存される
  - [ ] ダークモード（OS 連動/明/暗）を選択できる
  - [ ] 音 ON/OFF と振動 ON/OFF を個別のトグルで切り替えられ、ON/OFF が一目で判別できる
  - [ ] 片眼ガイダンス（off/左/右/交互）を設定できる
  - [ ] 「免責事項を読む」から免責を再閲覧できる
  - [ ] 「全データ削除」は 2 段階確認（タップ → 確認ダイアログ）で実行される
  - [ ] 各項目行は 56pt 以上の高さ
  - [ ] バージョン情報に `v2.0.x` と免責同意日時が表示される

### F-14 音・ハプティクスフィードバック（v1.2 F-19 継承・新ゲームに適合）
- **目的**：視覚以外のチャネル（聴覚・触覚）で正解/不正解/重要イベントを補強する。
- **主要な振る舞い**：
  - **音**：(1) ラウンド正解時、(2) ラウンド不正解時、(3) カウントダウン残り 3・2・1 秒の毎秒ティック、(4) セッション完了時、(5) バッジ獲得時。
  - **ハプティクス**：(1) 正解時、(2) 不正解時、(3) バッジ獲得時。
  - 設定で音 ON/OFF と振動 ON/OFF を個別に切替。サイレントモード時は音を鳴らさずハプティクスのみ。
  - 実装は `../rapidreading2/src/platform/audio.ts` の抽象を流用想定。
- **受け入れ基準**：
  - [ ] ラウンド採点時に正解/不正解に応じた音が再生される（音 OFF 時は無音）
  - [ ] カウントダウン残り 3/2/1 秒の各時点でティック音が再生される（音 OFF 時は無音）
  - [ ] セッション完了時に完了音が再生される（音 OFF 時は無音）
  - [ ] バッジ獲得時に獲得音が再生される（音 OFF 時は無音）
  - [ ] 正解/不正解/バッジ獲得時にハプティクスが発火する（振動 OFF 時は無振動）
  - [ ] サイレントモード（OS 設定）のとき音は鳴らさず、ハプティクスは発火する（振動 OFF でない限り）
  - [ ] 試行中（採点前の注視中）は採点フィードバック以外の音/振動を発火しない

## 5. バッジ仕様（v2.0 新規定義、3 軸）

旧バッジ（B-01〜B-12、7 ゲーム前提）は**全廃**。v2.0 は **継続日数（ストリーク）/ 高難度クリア / 高スコア（正答率）** の 3 軸で再定義する。各バッジの最終文言は Generator 実装時に確定するが、軸と条件の骨子を以下に定める。

### 5.1 継続日数（ストリーク）軸
| バッジ ID | 名称（仮） | 獲得条件（観察可能） |
|---|---|---|
| B-01 | はじめの一歩 | 初回セッション完了 |
| B-02 | 三日坊主突破 | 3 日連続でプレイした |
| B-03 | 一週間の習慣 | 7 日連続でプレイした |
| B-04 | 二週間マスター | 14 日連続でプレイした |
| B-05 | 一ヶ月の継続 | 30 日連続でプレイした |

### 5.2 高難度クリア軸
> 「高難度」の定義：回転速度 a が遅い／周波数変化速度 b が小さい設定でのクリアを指す。具体的な「遅い/小さい」閾値は Designer/Generator が可動範囲（AS-23）に基づき仮置きする。「クリア」＝そのセッションが高スコア（誤選択なく変化パッチを概ね正答）であることを指す。
| バッジ ID | 名称（仮） | 獲得条件（観察可能） |
|---|---|---|
| B-06 | スロー回転ハンター | 回転速度 a が「遅い」域の設定で、変化パッチをすべて正しく当てたセッションがある |
| B-07 | 微差を見抜く目 | 周波数変化速度 b が「小さい」域の設定で、変化パッチをすべて正しく当てたセッションがある |
| B-08 | 達人の眼 | a も b も最も難しい域の設定で高スコアを達成したセッションがある |

### 5.3 高スコア（正答率）軸
| バッジ ID | 名称（仮） | 獲得条件（観察可能） |
|---|---|---|
| B-09 | 好調のしるし | セッションスコア 80 以上を達成 |
| B-10 | パーフェクト | セッションスコア 100（全ラウンド満点）を達成 |
| B-11 | 安定の高得点 | スコア 80 以上を累計 5 セッション達成 |

### 5.4 バッジ共通の受け入れ基準
- [ ] 各バッジの獲得条件を満たすとバッジが付与される
- [ ] 獲得時に獲得演出（短時間、点滅なし）がセッション結果画面で 1 度だけ流れる
- [ ] 履歴タブのバッジ一覧で獲得済み/未獲得が確認でき、未獲得には条件ヒントが出る
- [ ] バッジ獲得時に音とハプティクスが発火する（設定で個別 OFF 可能）
- [ ] 旧バッジ（ゲーム依存条件）はアプリ内に一切存在しない

## 6. データモデル（v2.0、`gaboreye:v2:*` 名前空間）

v2.0 仕様承認時に本データモデルを凍結する。変更にはユーザー再承認が必要。永続化キーは `gaboreye:v2:*` プレフィックス。

### 6.1 `UserProfile`
| フィールド | 型 | 説明 |
|---|---|---|
| onboardingCompleted | boolean | オンボーディング完了フラグ |
| disclaimerAgreedAt | datetime | 免責同意日時 |
| ageGroup | enum('40s'/'50s'/'60s'/'70s+'/'unspecified') | 年齢層申告 |
| viewingDistanceCm | int(30/40/50) | 視聴距離 |
| deviceTypeEstimated | enum('iphone'/'android'/'tablet'/'pc') | 端末タイプ自動推定 |
| createdAt | datetime | 初回起動日時 |
| schemaVersion | string('2.0.0') | スキーマバージョン |

### 6.2 `Settings`
| フィールド | 型 | 説明 |
|---|---|---|
| gridSize（n） | int | 格子サイズ n×n（可動範囲・初期値は Generator 仮置き） |
| roundSeconds（m） | int | 1 ラウンドの制限時間（秒） |
| roundCount（r） | int | 1 セッションのラウンド数 |
| rotationSpeed（a） | float | 回転速度（deg/sec） |
| sfChangeSpeed（b） | float | 空間周波数変化速度（hz/sec） |
| scoringMode | enum('auto-no-confirm'/'auto-confirm'/'all-correct-advance') | 採点方式①②③ |
| darkMode | enum('system'/'light'/'dark') | ダークモード |
| soundEnabled | boolean | 効果音 ON/OFF |

| oneEyeGuidance | enum('off'/'left'/'right'/'alternate') | 片眼ガイダンス |

> 将来の「漸進難化モード」用に、難易度パラメータ（a/b/n 等）を動的に変化させるためのフィールド（例 `progressiveModeEnabled` / `progressionState`）を**追加できる構造**にしておく（v2.0 では実装しない、§10 スコープ外）。

### 6.3 `RoundRecord`（ラウンド 1 件 = 1 レコード）
| フィールド | 型 | 説明 |
|---|---|---|
| roundIndex | int | セッション内のラウンド番号（1..r） |
| changingPatchCount | int | 変化パッチの個数 |
| tpCount | int | 正しく選んだ変化パッチ数 |
| fpCount | int | 誤って選んだ静止パッチ数 |
| fnCount | int | 選ばなかった変化パッチ数 |
| roundScore | float | TP − FP のラウンド得点 |

### 6.4 `SessionRecord`（セッション 1 周 = 1 レコード）
| フィールド | 型 | 説明 |
|---|---|---|
| sessionId | uuid | セッション ID |

## 8. 非機能要件（v1.2 から継承・v2.0 向け調整）

### 8.1 パフォーマンス
- **NF-1** 刺激描画 60fps を目標。回転＋周波数変化アニメーションは 30fps 最低許容
- **NF-2** 画面遷移は 200ms 以内
- **NF-3** アプリ起動から距離リマインド表示まで 3 秒以内
- **NF-4** 中位スマホ（iPhone 12 / Pixel 5 相当）でフレームドロップを起こさない

### 8.2 オフライン
- **NF-5** ネットワーク通信を行わずに全機能が動作。**NF-6** 初回ダウンロード後オフラインで起動可能

### 8.3 アクセシビリティ
- **NF-7** 主要操作要素に aria-label / accessibility label を付与
- **NF-8** UI テキストとボタン背景のコントラスト比 7:1 以上（OPT-5）
- **NF-9** Web 版でキーボード操作：Tab で全インタラクティブ要素にフォーカス、Enter/Space で起動。focus outline 3px 以上
- **NF-10** スクリーンリーダーで主要要素が読み上げられる。結果開示で「正解」「不正解」が読み上げられる
- **NF-11** 点滅エフェクトは使用しない（てんかん配慮）。カウントダウンの色変化補強は太字化のみ
- **NF-12** 色のみで情報を伝えない（形・テキストでも判別可能）。タブ選択状態・カウントダウン色変化を補強する
- **NF-13** prefers-reduced-motion 対応：UI 装飾アニメーションを抑制（変化検出ゲーム本体は課題上アニメーションが必須のため、ゲーム刺激そのものは抑制対象外）
- **NF-14** Skip link（Web 版）
- **NF-15** 選択状態は aria-checked / aria-pressed で明示

### 8.4 老眼配慮（OPT-1〜12 継承）
- **NF-16** 本文最小フォント 18pt（OPT-1）。**NF-17** タップ領域 48pt 以上、推奨 56pt（OPT-2）。**NF-18** ボタン自動消失なし（OPT-7）
- **OPT-11 改訂**：旧「60 秒固定・早期終了不可」は廃止。**m 秒可変＋3 採点方式（F-02）に置換**。方式②の確定・方式③の全問正解遷移により早期締切が可能になる
- **OPT-12 改訂**：旧「全ゲーム共通の 60 秒注視フォーマット」は新ゲーム 1 種に合わせ改訂。共通項は「m 秒のラウンド・直接タップ選択・試行中は正誤フィードバックなし・採点後 ✅/❌ 開示」

### 8.5 プラットフォーム
- **NF-19** iOS 15 以上 / Android 8（API 26）以上。**NF-20** Web は最新 2 メジャーの Safari/Chrome/Edge/Firefox
- **NF-21** スマホ縦持ち固定、PC 横画面（最小 360px、最大 1280px）。**NF-22** タブレットは横レイアウトで PC と同等

### 8.6 セキュリティ
- **NF-23** ローカルストレージは端末標準の暗号化に従う。**NF-24** リリースビルドでログ出力しない

### 8.7 国際化
- **NF-25** 日本語のみ。i18n キー基盤は維持

### 8.8 ガボール描画品質（v1.2 継承）
- **NF-26** 角度・周波数の最小可動単位は Designer/Generator が表示サイズ・cpd・端末性能から導出（具体値は spec に書かない）
- **NF-27** パッチの枠（矩形/円形マスク）と縞模様の間に背景色が見えてはならない
- **NF-28** 角度・周波数を最小単位ずつ変化させても、パッチ枠と縞模様の間に隙間や欠けが生じてはならない
- **NF-28b 変化/静止の弁別性**：静止パッチの初期角度・初期周波数のばらつきと、変化パッチの単位時間あたり変化量を、ユーザーが「変化中」と「静止」を区別できるように調整する（最小可知差の具体値は Designer/Generator が体感調整）

### 8.9 ステータスバー回避
- **NF-29** ゲーム刺激提示中はフルスクリーン許容（OS ステータスバー領域も使用してよい）
- **NF-30** それ以外の画面（距離リマインド・ホーム結果・履歴・設定・ダイアログ）はセーフエリア準拠（iOS=SafeAreaInset、Android=StatusBar.currentHeight 自動取得）

### 8.10 音・ハプティクス品質
- **NF-31** 音再生レイテンシ 100ms 以内。**NF-32** ハプティクス強度は端末標準（iOS medium 相当 / Android strong 相当）をデフォルト。**NF-33** サイレントモード尊重（音は鳴らさずハプティクスのみ）
