# GaborEye V1 — 老眼向けガボール視覚トレーニング プロダクト仕様書

## 0. サマリ／ビジョン

**GaborEye** は、ガボールパッチ（V1 一次視覚野を直接駆動する縞模様刺激）を題材にした 3 つのミニゲームを 1 日 3 分プレイすることで、老眼ユーザーのコントラスト感度・周辺視・弁別能力を継続的に鍛える、完全オフラインのセルフケア向けスマホ／PC アプリです。「変化察知」「二重表裏判別」「周辺視野ハント」の 3 ゲームで飽きさせず、staircase 法による自動難易度調整と週次の独自スコア（V1 スコア）グラフで「効いている感」を可視化します。**40 代以上の老眼層に焦点を絞り**、大きい文字・短い操作・優しい所作・読書距離前提のキャリブレーションを徹底します。医療効果は断定せず、「自助セルフケア」として位置づけます。

## 1. 老眼最適化原則（全機能横断の規範）

本仕様の全機能は、以下の老眼最適化原則を満たさなければなりません。本書全体で繰り返し参照します。

- **OPT-1 大きい文字**：本文最小フォントサイズ 18pt（≒ 24px）以上、見出し 22pt 以上、ボタン文字 18pt 以上
- **OPT-2 大きいタップ領域**：操作可能要素は最小 48×48 pt 以上、推奨 56pt
- **OPT-3 短い操作シーケンス**：1 タスクあたりの操作ステップは最大 3 タップを目標
- **OPT-4 読書距離プリセット**：視聴距離は 30 / 40 / 50cm の 3 段階スライダー（デフォルト 40cm = 老眼読書距離中心）
- **OPT-5 高コントラスト UI**：UI テキスト／ボタンの背景コントラスト比は 7:1 以上（WCAG AAA 相当）。ガボール刺激そのものは別ロジック
- **OPT-6 優しい言い回し**：医療・専門用語は最小限。専門語を使うときは括弧書きで「（縞模様の刺激）」のように補足
- **OPT-7 急かさない**：ボタンの自動消失なし。説明画面はユーザー操作で進む。タイムアウトのある画面ではカウントダウンを 18pt 以上で明示
- **OPT-8 ダークモード必須**：白背景の眩しさ回避を選べる。設定で OS 連動／明示切り替えが可能
- **OPT-9 片眼トレーニング配慮**：「片目を手で覆ってください」のガイダンスは大きいイラストと音声無しの静止表示
- **OPT-10 開始前の距離リマインド**：各セッション／各ゲーム開始前に「画面から ◯◯cm 離れてください」を 18pt 以上で表示（OK タップで進行）
- **OPT-11 強制最小視聴時間**：ガボール刺激への最小曝露時間を確保するため、各ゲームの所定時間／所定試行数は早期終了できない。中断は緊急脱出用の × ボタン（中断=未完了試行扱い）に限る。これは「短時間で済ませる」ことよりも「所定の時間を必ず見る」ことが本アプリの訓練効果の根幹だからである

## 2. 想定（Assumptions）

質疑応答で確定しなかった暗黙の前提を以下に明示します。これらは Planner の合理的仮定であり、後続フェーズで疑義があれば再確認します。

- **A-1**：本アプリは「医療機器」ではない。医療効果は断定せず、薬機法を意識した文言で運用する
- **A-2**：ユーザーは画面に正対し、片手または両手で持つ／机上に置く。寝そべりは想定外
- **A-3**：ガボール刺激の物理 cpd 再現は端末ごとの dpi メタデータと申告距離の組合せで近似する。完全な物理計測キャリブレーションは V1 では行わない
- **A-4**：staircase は各ゲーム独立に走らせる（ゲーム間で閾値を共有しない）
- **A-5**：1 試行ごとの記録はゲーム単位で持ち、生のセッション履歴は端末ローカルに保存する
- **A-6**：日本語ロケールのみ。日付・週は ISO 週（月曜開始）で扱う
- **A-7**：「眼鏡・コンタクトを使用中ならそのままで」前提。本アプリは矯正の有無に関与しない
- **A-8**：ゲーム中の中断（電話着信／アプリ切替）は記録上「未完了試行」として扱い、スコア計算には含めない
- **A-9**：年齢申告（40 代／50 代／60 代／70 代以上）は任意。70 代以上選択時は警告と医師相談推奨を表示
- **A-10**：ガボール描画は 60fps を維持できるネイティブ／Web 描画手段（Canvas／WebGL／Skia 相当）が利用可能であることを前提とする

## 3. 対象ユーザー／ペルソナ

### 3.1 メインターゲット
**老眼層（40 代以上、骨太層ターゲット）**：手元の文字が見えづらくなった、夕方になると新聞が読みづらい、PC 作業で目が疲れる、といった主観的不満を持つが医療機関には行くほどでない自助セルフケア指向のユーザー。

### 3.2 ペルソナ例

| 属性 | ペルソナ A | ペルソナ B | ペルソナ C |
|---|---|---|---|
| 年齢 | 47 歳 | 55 歳 | 63 歳 |
| 状況 | 営業職、PC で疲れ目 | 主婦、新聞・スマホ | 退職前、読書好き |
| 矯正 | 老眼鏡デビュー直前 | 老眼鏡使用中 | 遠近両用使用中 |
| デバイス | iPhone + 自宅 PC | iPhone のみ | iPad 中心 |
| 利用シーン | 通勤中・寝る前 | 朝食後・夕方 | 読書前のウォームアップ |

### 3.3 対象外ユーザー（明示）

- 子ども（小学生以下）
- 70 代以上の高齢者（任意で年齢申告した場合は警告表示）
- 重度視覚障害者（白内障・緑内障・加齢黄斑変性などの診断歴がある場合は医師相談を推奨）
- 強度近視（-6D 以上）の単独利用は推奨しない（医師相談を推奨）

## 4. スコープ内 / スコープ外

### 4.1 スコープ内（V1）

- 3 種類のガボールゲーム（変化察知 / 二重表裏判別 / 周辺視野ハント）
- 3 ゲーム順次実行の「おまかせコース」と単体プレイモード
- **各ゲームの所定時間／所定試行数は早期終了不可（OPT-11）**。ガボール刺激への最小曝露時間を必ず確保する
- staircase（3-down/1-up）による自動難易度調整
- 簡易キャリブレーション（端末タイプ自動推定 + 視聴距離スライダー）
- 週次 V1 スコアグラフ／ストリーク／日次ベスト／達成バッジ
- 設定（音・振動・ダークモード・視聴距離・全データ削除）
- 90 秒以内オンボーディング、ゲーム別ミニチュートリアル
- 完全オフライン動作
- 日本語のみ
- iOS / Android スマホ縦持ち、Web ブラウザ（PC 横）対応、最小幅 360px〜最大 1280px

### 4.2 スコープ外（V1 では実装しない）

- マルチプレイヤー／対戦／フレンド機能
- ランキング・SNS 連携・シェア機能
- 医療機関連携／医師レビュー
- Apple Watch / Google Fit / HealthKit 連携
- Eye Tracking、ARKit/ARCore による空間配置
- クラウド同期・複数端末ログイン
- アカウント・認証・課金
- 多言語（英語含む）— ただし将来対応可能な文言構造にする
- 医療・薬機法的な「視力◯◯回復」表記
- BGM／キャラクター演出（Game 3 の任意リズム BGM 以外）
- 重度近視・小児向けの専用調整
- 物理的キャリブレーション（クレジットカードで dpi を測る等）
- Web 通知・プッシュ通知

## 5. 機能一覧（フィーチャーリスト）

MECE な単位で 16 機能を定義します。各機能 ID は以降のスプリント計画と相互参照されます。

### F-01 オンボーディング（90 秒以内）
- **目的**：初回起動から 1 試行完了まで 90 秒以内で到達させ、離脱を防ぐ
- **主要な振る舞い**：免責同意 → 年齢層申告 → 視聴距離設定 → ゲーム 1 ミニ説明 → 1 試行体験 の 5 ステップ
- **受け入れ基準**：
  - [ ] 初回起動時のみ自動表示。2 回目以降は表示されない
  - [ ] 各ステップに「次へ」ボタンが 56pt 以上で配置されている
  - [ ] 免責同意未チェックでは次へ進めない
  - [ ] 完了までの操作タップ数が 6 回以下
  - [ ] スキップ機能はない（必須通過）

### F-02 免責事項表示
- **目的**：医療機器でない旨と対象外ユーザーを明示し、誤期待による苦情・健康被害を防止
- **主要な振る舞い**：オンボーディング初回表示 + 設定からいつでも再閲覧 + 70 代以上選択時に追加警告
- **受け入れ基準**：
  - [ ] 文言が 18pt 以上で表示される
  - [ ] 「同意する」ボタンを押すまで先に進めない（初回のみ）
  - [ ] 設定 →「免責事項を読む」からいつでも全文を再閲覧できる
  - [ ] 同意日時を端末ローカルに保存する

### F-03 視聴距離キャリブレーション
- **目的**：ガボールの cpd を端末ごとに近似再現する
- **主要な振る舞い**：端末タイプ自動推定 → 視聴距離スライダー（30 / 40 / 50cm の 3 ノッチ、デフォルト 40cm）→ プレビュー
- **受け入れ基準**：
  - [ ] 端末タイプ（iPhone／Android／PC ブラウザ／タブレット）が自動推定される
  - [ ] スライダーは 3 段階のノッチ式で、現在値が 22pt 以上で表示される
  - [ ] プレビュー画面に「この大きさのパッチで訓練します」と例示が出る
  - [ ] 設定からいつでも変更でき、変更は即座に保存される
  - [ ] セッション開始前に「画面から 40cm 離れてください」のリマインドが表示される

### F-04 ホーム画面
- **目的**：今日の進捗とコース開始の唯一の入り口
- **主要な振る舞い**：本日の達成状態 / 連続日数 / 「3 分コースを始める」ボタン / 単体ゲーム選択 / 設定アクセス
- **受け入れ基準**：
  - [ ] 「3 分コースを始める」ボタンが画面中央付近に 56pt 以上で配置されている
  - [ ] 本日のセッション完了済みの場合は「今日のトレーニング完了」マークが表示される
  - [ ] 連続日数（ストリーク）が 22pt 以上で表示される
  - [ ] 単体ゲーム選択は 3 つのカード型 UI で並ぶ
  - [ ] 設定アクセスは右上または下部に常時配置されている

### F-05 おまかせ 3 分コース
- **目的**：1 セッション = 3 ゲーム 1 分ずつを連続実行
- **主要な振る舞い**：Game 1 → 結果サマリ（10 秒以内） → Game 2 → 結果サマリ → Game 3 → 結果サマリ → クールダウン（10 秒）→ セッション完了画面
- **受け入れ基準**：
  - [ ] 各ゲームは 60 秒（±5 秒）で自動終了する
  - [ ] ゲーム間は 10 秒以内の結果サマリで自動進行する
  - [ ] コース全体で約 3 分（180〜220 秒）以内
  - [ ] コース中断（戻るボタン等）時は確認ダイアログを出す
  - [ ] 完了するとセッション完了画面でストリーク／本日のスコアが表示される

### F-06 単体ゲームモード
- **目的**：好きなゲームだけを単独でプレイ
- **主要な振る舞い**：ホームから該当ゲームのカードをタップ → 距離リマインド → 60 秒プレイ → 結果サマリ → ホームに戻る
- **受け入れ基準**：
  - [ ] 3 ゲームそれぞれが単独で起動できる
  - [ ] 単体プレイの記録もセッション履歴に保存される（タイプは「単体」）
  - [ ] 単体プレイのみで本日のセッション完了とはみなされない（おまかせコース完了が条件）
  - [ ] 単体プレイ後はホームに戻る／同じゲームをもう一度／別のゲームへ の 3 択を提示

### F-07 Game 1：変化察知
- **目的**：長時間注視によるコントラスト感度と orientation/size 弁別の同時訓練
- **主要な振る舞い**：3×3〜5×5 グリッドのガボールパッチ → 60 秒間モーフィング表示（早期終了不可、画面上部に残り秒数表示）→ ユーザーは「変化したパッチ」をタップ／再タップで自由に選択・解除 → 60 秒経過で自動採点
- **受け入れ基準**：詳細は §7.1

### F-08 Game 2：左右並び傾き判別（Side-by-Side Tilt Discrimination）
- **目的**：左右に同時提示された 2 つのガボールを連続注視させ、orientation discrimination と注視持続を同時訓練
- **主要な振る舞い**：固視点 500ms → 画面左右に 2 つのガボールパッチを 5 秒間同時提示（点滅なし、マスクなし）→ 「左右どちらが時計回りに傾いているか」を 2AFC（左ボタン／右ボタン）で回答（回答制限 3 秒、無回答は不正解）→ フィードバック 1 秒 → 次試行
- **受け入れ基準**：詳細は §7.2

### F-09 Game 3：周辺視野ハント
- **目的**：固視点維持下での周辺視 orientation pop-out 訓練
- **主要な振る舞い**：固視点中央表示 → 周囲に複数ガボール（300〜800ms）→ マスク → 「odd one」の時計方向を 8〜12 択で回答
- **受け入れ基準**：詳細は §7.3

### F-10 staircase 自動難易度調整
- **目的**：各ゲームで正答率 ~79% を維持し、perceptual learning を最大化
- **主要な振る舞い**：3-down/1-up ルールで難易度パラメータを step 単位で増減。ゲーム独立に状態を持つ
- **受け入れ基準**：
  - [ ] 3 連続正解で次の試行は難しくなる（パラメータが閾値方向に 1step 動く）
  - [ ] 1 つでも誤答するとパラメータが 1step 易しい方向に動く
  - [ ] 難易度は最小値・最大値の範囲でクランプされる
  - [ ] reversal（方向転換）回数と最終 6 reversals の平均値を「閾値スコア」として保存
  - [ ] 各ゲームの staircase 状態はセッション間で引き継がれる（前回の閾値からスタート）
  - [ ] 設定から「staircase をリセット」ができる

### F-11 結果サマリ表示
- **目的**：1 試行／1 ゲーム終了直後に「正解」と「自分の閾値」を可視化し、学習効果を支援
- **主要な振る舞い**：1 試行終了時に正解箇所をアニメ提示（点滅でなく拡大ハイライト）、ゲーム終了時に閾値スコアと前回比を表示
- **受け入れ基準**：
  - [ ] 試行終了時に正解箇所が拡大ハイライトで 1.5 秒提示される（点滅なし）
  - [ ] ゲーム終了サマリには「今回の閾値」「前回からの変化」が 22pt 以上で表示される
  - [ ] サマリ画面は「次へ」を押すまで自動で進まない（OPT-7）
  - [ ] スキップ操作は「未挑戦」として記録され、サマリには表示されない

### F-12 V1 スコアと週次グラフ
- **目的**：「効いている感」を医療数値を出さずに可視化
- **主要な振る舞い**：3 ゲームの閾値を正規化合成した「V1 スコア」を日次集計し、過去 4 週間（28 日）を折れ線で表示
- **受け入れ基準**：
  - [ ] V1 スコアは 0〜100 の整数で表示される（医療数値「視力 0.X」は使わない）
  - [ ] 過去 4 週間（28 日）の日次値を折れ線グラフで表示する
  - [ ] グラフの軸ラベルは 18pt 以上
  - [ ] 当日のスコアはグラフ上で強調される
  - [ ] データが 7 日未満のときは「もう少しデータが集まると傾向が見えます」と表示される
  - [ ] スコア算出方法の詳細は §9.1

### F-13 ストリークと日次ベスト
- **目的**：継続インセンティブを医療数値ではなく行動指標で与える
- **主要な振る舞い**：おまかせコース完了で連続日数（ストリーク）が 1 増える。0 時切替で当日未完了なら翌日リセット。各ゲームの日次ベスト閾値を保持
- **受け入れ基準**：
  - [ ] ストリーク日数が現在値で表示される
  - [ ] おまかせコース完了でストリークが +1 される（同日 2 回目以降は加算しない）
  - [ ] 0:00（端末ローカル時刻）跨ぎで前日コース未完了ならストリークが 0 にリセットされる
  - [ ] 各ゲームの日次ベスト閾値が記録される
  - [ ] ストリーク表示には「今日終わるとリセットされます」の警告が前日 22 時以降に出る

### F-14 達成バッジ
- **目的**：軽い達成感の可視化
- **主要な振る舞い**：規定の達成条件を満たすとバッジが付与される。一覧画面で確認可能
- **受け入れ基準**：
  - [ ] 「初回コース完了」「3 日連続」「7 日連続」「30 日連続」「全ゲーム閾値改善」「100 試行達成」など 8 種類のバッジが定義される
  - [ ] 獲得時に獲得演出（1.5 秒）が結果サマリ画面で 1 度だけ流れる
  - [ ] 設定 →「バッジ一覧」から獲得済み／未獲得を確認できる
  - [ ] 未獲得バッジには獲得条件のヒントが表示される
  - [ ] 点滅エフェクトは使用しない（OPT-9 / a11y）

### F-15 設定画面
- **目的**：ユーザー個別の調整ポイントを集約
- **主要な振る舞い**：音・振動・ダークモード・視聴距離・片眼ガイダンス・staircase リセット・全データ削除・免責事項再閲覧・バージョン情報・バッジ一覧
- **受け入れ基準**：
  - [ ] 全項目がリスト型で並び、各行は 56pt 以上の高さ
  - [ ] トグル系（音／振動／ダークモード／片眼ガイダンス）は ON/OFF が一目で判別できる
  - [ ] 「全データ削除」は 2 段階確認（タップ → 確認ダイアログ → 「削除」入力）で実行される
  - [ ] 「staircase をリセット」は確認ダイアログ後に実行される
  - [ ] バージョン情報には V1.x.y と免責同意日時が表示される

### F-16 セッションクールダウン
- **目的**：眼精疲労軽減、20-20-20 ルールの簡易版
- **主要な振る舞い**：おまかせコース終了直後に「10 秒間遠くを見ましょう」画面を 1 枚挟む
- **受け入れ基準**：
  - [ ] 10 秒のカウントダウンが 22pt 以上で表示される
  - [ ] スキップボタンが 56pt 以上で配置されている
  - [ ] イラストは静止（点滅・アニメなし）

## 6. ガボール刺激のパラメータ仕様

### 6.1 老眼向けに調整したパラメータ範囲

| パラメータ | 範囲 | 老眼向け調整理由 |
|---|---|---|
| 空間周波数（cpd） | 1.5 / 3 / 6 / 9（cpd） | 老眼層は中高 cpd の感度低下顕著。12/18cpd は外し、9cpd を上限 |
| ガウス窓 SD | 0.3°〜1.0° | 小さすぎるパッチは見つけづらい。下限を 0.3° に上げる |
| コントラスト | 0.15〜0.6 | 下限を文献標準 0.1 から 0.15 に上げ、見えなさのストレスを軽減 |
| 向き | 0° / 45° / 90° / 135° のいずれかからランダム抽出（基準向き） | 標準 |
| 位相 | 0〜2π ランダム | 標準 |
| 表示色 | グレースケール（背景輝度 = 50%、刺激は背景輝度を中心に変調） | 色覚配慮（OPT・Q9） |
| 左右ペア角度差（Game 2 専用） | 1°〜10°（初期 6°）、staircase 対象 | 同時提示型のため、基準向きに対し ±角度差/2 を左右に割り当て |

### 6.2 キャリブレーション仕様

- **端末タイプ自動推定**：UA 文字列／端末識別 API から「iPhone（推定 dpi 約 460）／Android スマホ（既定 400）／タブレット（既定 264）／PC ブラウザ（既定 110）」の 4 区分に振り分ける
- **視聴距離スライダー**：30cm（読書近め）／40cm（読書標準）／50cm（読書遠め・タブレット机上）の 3 ノッチ。デフォルト 40cm
- **cpd → 物理サイズ換算**：cpd を `1 / (2 * tan(0.5° × distance / pixel_size_in_cm))` で正弦波周期画素数に変換し、Gauss 窓 SD も同様に物理 → 画素変換
- **動作要件**：自動推定が外れた場合の手動補正は V1 では実装しない（V2 検討）。代わりに「パッチが小さすぎる／大きすぎる」と感じたら視聴距離スライダーで調整するよう案内

### 6.3 staircase アルゴリズム仕様

- **方式**：3-down/1-up（正答率 ~79.4% 収束）
- **step size**：初期 = 4 unit（パラメータ依存。例 Game 2 角度差なら 4°）、最初の reversal で 2 unit、3 reversal 目以降で 1 unit に縮小
- **終了条件**：1 ゲーム = 60 秒経過、または最大試行数（Game 1: 1 試行のみ／Game 2: 最大 30 試行（60 秒経過で強制終了）／Game 3: 最大 40 試行）
- **閾値推定**：最終 6 reversals の対象パラメータ平均値を「今回の閾値」とする。reversal が 6 未満なら最終値を採用
- **状態の永続化**：ゲームごとに最終 step／現在のパラメータ値を保存し、次回セッションは前回値からスタート
- **リセット**：設定 →「staircase をリセット」で初期 step・初期パラメータに戻す
- **クランプ**：パラメータが定義された min/max を超える場合はクランプ

## 7. 3 ゲームの詳細仕様

### 7.1 Game 1：変化察知（Morphing Detection）

#### 仕様
- **グリッドサイズ**：3×3（易しい）／4×4（普通）／5×5（難しい）。staircase で自動切替
- **同時変化数**：1 個（易）／2 個（中）／3 個（難）
- **変化タイプ**：角度（°）変化のみ（V1 では size 変化は使わず、安定的な orientation discrimination に集約）
- **変化量（最大角度差）**：易 8°／中 5°／難 3°（staircase 対象）
- **変化スピード**：60 秒のゲーム時間で線形に最大角度差まで変化
- **試行時間**：1 試行 = 60 秒固定（じっくり見る設計、早期終了不可）
- **タイマー仕様**：
  - 60 秒の固定タイマーを **早期終了不可** とする。所定の 60 秒は必ず経過する（OPT-11）
  - ガボール盤は 60 秒間ずっと表示・モーフィングを継続する
  - 画面上部に「**残り N 秒**」のカウントダウンを 22pt 以上で常時表示する（ユーザーが残り時間を把握できるようにするため、ただし急かす演出は伴わない／OPT-7）
  - 「完了」ボタンは設けない
- **回答方法**：
  - 試行中、ユーザーは「変化したと思うパッチ」をタップで選択／再タップで解除を **自由に何度でも** 行える
  - 60 秒経過で自動採点。採点時点でタップ済み（選択中）のパッチが回答とみなされる
  - タップ無しで 60 秒経過した場合は「未挑戦」として記録され、staircase は up（易方向）に動く
- **採点**：60 秒経過時点の選択状態に対し以下を集計
  - True Positive（正タップ）= +1
  - False Positive（誤タップ）= -1
  - False Negative（取り逃がし）= 0（減点なし）
  - 1 試行の合計値が 0 未満なら 0 にクランプ
- **採点後の表示**：実際の変化箇所を拡大ハイライト 1.5 秒
- **緊急脱出**：画面右上の × ボタン（中断、未完了試行扱い）は維持する。これは緊急脱出用であり、通常完了の手段ではない

#### 受け入れ基準
- [ ] 1 試行 60 秒で自動終了し、採点画面に進む
- [ ] **60 秒タイマーは早期終了できない**（完了ボタンは存在しない）。所定の 60 秒は必ず経過する
- [ ] **画面上部に「残り N 秒」のカウントダウンが 22pt 以上で常時表示される**
- [ ] **60 秒経過時点で自動採点**され、その時点でタップ済みのパッチが回答とみなされる
- [ ] **試行中、ユーザーはパッチのタップによる選択／再タップによる解除を自由に何度でもできる**
- [ ] グリッドのパッチは 48pt 以上のタップ領域を持つ
- [ ] 変化開始時と終了時の角度差は staircase で決まった最大角度差に一致する
- [ ] 変化はスムーズ（30fps 以上）で、ジャンプしない
- [ ] 採点後に「変化していたパッチ」が拡大ハイライトで提示される
- [ ] タップ済みパッチは選択状態が視覚的に分かる（色／枠）

### 7.2 Game 2：左右並び傾き判別（Side-by-Side Tilt Discrimination）

#### 仕様
- **提示シーケンス（同時提示型）**：
  1. **固視点 500ms**（画面中央に十字）
  2. **左右並び 5 秒同時提示**：画面左右に 2 つのガボールパッチを 5000ms 連続表示（点滅なし、マスクなし、フェードなし）。サイズ・空間周波数・コントラストは左右で同一、向きのみ角度差を持つ。ユーザーは 5 秒間どちらの注視・両者の交互注視も自由
  3. **回答受付（最大 3000ms）**：5 秒経過後、「左 / 右」の 2AFC 回答ボタンが表示される（回答 UI 出現時の点滅感を軽減するための具体的な提示位置と表示・非表示の切り替え方は Designer 判断とする。ガボールの下に重畳しても、ガボール消去後に出してもよい）
  4. **タイムアウト判定**：3 秒以内にタップが無ければ「不正解」として記録し、staircase は up（易方向）に動く
  5. **フィードバック 1 秒**：正解／不正解アイコンと、正解側（左 / 右）を 1 秒間ハイライト → 次試行
- **2AFC 設問**：「**どちらが時計回りに傾いていますか？**」を「左」「右」の 2 択で回答。staircase が左右の角度差をコントロールする
- **角度差（staircase 対象）**：1°〜10°（初期 6°、min 1°、max 10°）。staircase は左右ペアの角度差（degrees diff）を制御
- **左右の傾き決定**：基準向き（試行ごとにランダム選択、例 0°）に対し、片方を `+角度差/2`（時計回り）、もう片方を `-角度差/2`（反時計回り）に設定。時計回り側がランダムに左 or 右に配置される
- **コントラスト**：左右のパッチともに 0.3 固定（V1 では角度差のみ staircase 対象）
- **試行回数**：最大 30 試行（1 試行 ≈ 9.5 秒：固視 0.5s + 提示 5s + 回答 3s + フィードバック 1s）。**所定試行数または 60 秒経過で自動終了**
- **セッション終了条件**（いずれか先に到達したもの）：
  - **60 秒経過**で強制終了（残り試行スキップ、結果サマリへ）
  - **30 試行達成**で終了
  - **× ボタン（緊急脱出）**による中断（未完了試行扱い）
- **採点後の表示**：最終試行のフィードバック後、結果サマリへ自動遷移

#### 受け入れ基準
- [ ] 試行開始時、画面中央に固視点が 500ms 表示される
- [ ] **2 つのガボールパッチが画面左右に同時表示される**（同サイズ・同 cpd・同コントラスト、向きのみ異なる）
- [ ] **提示中は 5 秒間連続表示**（点滅なし、マスクなし、フェードなし）
- [ ] 5 秒間の提示中、ユーザーは左右両方を自由に注視・交互注視できる
- [ ] 設問「どちらが時計回りに傾いていますか？」が文言として表示される
- [ ] 回答ボタンは「左」「右」の 2 個、それぞれ 56pt 以上
- [ ] **回答制限時間は 3 秒**。3 秒以内にタップが無ければ「不正解」として記録され、staircase は up（易方向）に動く
- [ ] 角度差は staircase に従い、3-down/1-up で動く
- [ ] 時計回り側は試行ごとにランダムに左／右に配置される（位置バイアス防止）

## 9. 進捗・統計仕様

### 9.1 V1 スコア算出方法

各ゲームの閾値を 0〜100 の正規化スコアに変換し、3 ゲームの平均を「V1 スコア」とする。

| ゲーム | 閾値パラメータ | 正規化式（参考） | スコア意味 |
|---|---|---|---|
| Game 1 | 最大角度差（°）、易 8° → 難 3° | `score = clamp((8 - threshold) / (8 - 3) × 100, 0, 100)` | 小さい角度差で見分けられるほど高得点 |
| Game 2 | 最小判別角度差（°）、初期 6° → 難 1° | `score = clamp((10 - threshold) / (10 - 1) × 100, 0, 100)` | 小さい角度差を判別できるほど高得点 |
| Game 3 | 最小角度差（°）、初期 30° → 難 5° | `score = clamp((45 - threshold) / (45 - 5) × 100, 0, 100)` | 周辺視で小さい差を見つけられるほど高得点 |
| **V1 スコア** | — | 3 ゲームのスコアの算術平均（整数四捨五入） | 0〜100 の整数 |

- **欠損ゲームがある日**：その日のスコアは「実施したゲームのみの平均」。3 ゲーム全てがない日はスコア無し（グラフでは欠損点）
- **同日複数セッション**：その日の最良スコア（max）を採用

### 9.2 週次グラフ表示仕様

- 過去 4 週間（28 日）を折れ線グラフで表示
- 横軸：日付（月／日）、縦軸：V1 スコア（0〜100）
- 当日の点は強調表示（円を大きく）
- データが 7 日未満：「もう少しデータが集まると傾向が見えます」のメッセージ
- データが 0 日：「まずは 1 セッション完了させましょう」のメッセージ
- 日次ベスト閾値（ゲーム別）は別タブまたは下部リスト表示

### 9.3 ストリーク・日次ベスト・バッジ仕様

#### ストリーク

## 16. 用語集

| 用語 | 説明 |
|---|---|
| **ガボールパッチ（Gabor patch）** | ガウシアン窓で振幅変調された正弦波の縞模様。V1 ニューロンの受容野と数学的に一致する刺激 |
| **V1（一次視覚野）** | 大脳後頭葉、視覚情報処理の最初の段階。orientation・空間周波数を弁別するニューロンが多い |
| **cpd（cycles per degree）** | 視野角 1° あたりの縞のサイクル数。空間周波数の単位。値が大きいほど細かい縞 |
| **CSF（Contrast Sensitivity Function）** | コントラスト感度関数。各空間周波数におけるコントラスト検出閾値の逆数 |
| **staircase 法** | 正答／誤答に応じて刺激パラメータを上下させ、知覚閾値を効率的に推定する手続き |
| **3-down/1-up** | 3 連続正解で難しく、1 つでも誤答で易しくする staircase ルール。正答率約 79.4% に収束 |
| **2AFC（Two-Alternative Forced Choice）** | 二択強制選択。「A か B か」の 2 択で必ず答えさせる課題形式 |
| **odd one** | 周辺視野ハントで、他と 1 つだけ異なるパッチ。pop-out 課題の標的 |
| **固視点（fixation point）** | 視線を留める目印。中心視を強制するために中央に表示する小さい十字 |
| **離心角（eccentricity）** | 固視点（中心視）から刺激位置までの視野角。値が大きいほど周辺視 |
| **perceptual learning** | 知覚学習。反復課題により知覚弁別能が向上する現象。V1 レベルでも起きる |
| **reversal** | staircase で難易度の上下方向が切り替わる点。閾値推定はこの reversal 平均で行う |
| **マスク（visual mask）** | 刺激の残像を消すための直後提示画像。本仕様ではランダム向きの高コントラストガボール |
| **V1 スコア** | 本アプリ独自の 0〜100 整数スコア。3 ゲームの閾値を正規化合成。医療数値ではない |
| **ストリーク** | 連続日数。おまかせコース完了で +1 |
| **片眼ガイダンス** | 「片目を手で覆ってください」と表示するだけのガイダンス。機械的オクルージョンは行わない |

## 17. 変更履歴

### 2026-04-30 — Game 2 を左右並び 5 秒同時提示型に再設計（amendment 2）
- **背景**：ユーザーフィードバック「Game 2 がガボールの点滅のように見え、本来の目的である『注視させること』からずれている」「60 秒経っても次に進まず左右の選択を待ち続ける（試行タイムアウトが無い）」に対応
- **設計判断の根拠**：旧設計の「短時間提示（1s）＋マスク（200ms）＋短時間提示（1s）→ 2AFC」は perceptual learning の古典的文献（Polat et al. ほか）に基づくが、**本アプリ『ガボールアイ』の主目的は「ガボール刺激を一定時間しっかり注視させる」訓練**であり、注視持続を阻害するマスクと短時間連続提示は本アプリのコンセプトと整合しない。Game 1（60 秒注視）と一貫させ、Game 2 も「左右並びを 5 秒間しっかり見る」設計に統一する
- **変更点**：
  - §5 F-08 の名称を「二重表裏判別」→「**左右並び傾き判別**（Side-by-Side Tilt Discrimination）」に変更。目的・主要な振る舞いを同時提示型に書き換え
