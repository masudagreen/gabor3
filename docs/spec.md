## 0. 改訂履歴

### 0.1 v3.0 → v3.1 改訂履歴（本質的変更点）

v3.1 は v3.0 の進捗モデル（レベル管理・mixed-radix オドメータ・クリア/失敗 2 値判定）を維持したまま、**ゲームの 1 単位を「1 ラウンド」から「指定時間まで反復するセッション」へ拡張**し、ラウンドの締め方・レベル昇降の適用単位・設定値の選択肢を更新する。2026-06-10 ユーザー確認済み。データモデルは一部拡張のうえ **v3.1 で再凍結**する。

| ID | 改訂対象 | 旧（v3.0） | 新（v3.1） | 理由 |
|---|---|---|---|---|
| **A** | **ゲーム構造の単位** | 1 ゲーム = 1 ラウンド = 1 レベル挑戦。終わるとホームに結果 | **1 セッション = 指定時間（`sessionMinutes`）を超えるまでラウンドを反復**。各ラウンド完了時に累積経過時間が `sessionMinutes`×60 秒未満なら次ラウンドを開始（最後のラウンドは指定時間を超えても完走） | じっと見続ける時間を「分」で確保したいニーズに合わせ、1 回のプレイを時間ベースのセッションにする |
| **A2** | **セッション時間設定** | なし | 新グローバル設定 **`sessionMinutes`**（1〜15 分、1 分刻み、**初期値 5 分**）を設定タブに追加。**レベル変数ではなく全プレイ共通の設定** | プレイ 1 回の長さをユーザーが選べるようにする |
| **B** | **ラウンドの締め方** | 全問正解になった時点で**即時締め切り**（v2.0 方式③相当）or 時間切れ | **全問正解での即時終了を廃止**。各ラウンドは「時間」秒だけ続き、**締切（時間到達）時にクリア/失敗を判定**（v2.0 自動採点①スタイルへ回帰） | 「正解した瞬間に終わる」を避け、注視時間を一定に保つ。連続プレイのリズムを安定させる |
| **B2** | **ラウンド間の開示** | 締め切り後 ✅/❌ 開示。次への遷移は短いインターバル（具体秒数 Designer 判断） | 締切後 ✅/❌・総合クリア/失敗を**ゲーム画面内（in-game）で開示**し、**3 秒のカウントダウン**（F-12 準拠、白→黄→赤）を表示。3 秒後に**自動で次ラウンドへ** | ラウンド反復のテンポを確定値（3 秒）で固定し、開示→次の流れを観察可能にする |
| **C** | **レベル昇降の適用単位** | 1 ゲーム（=1 ラウンド）結果ごとにレベル増減 | **各ラウンドのクリア/失敗で即レベル増減**（セッション内で連続的に上下）。昇降ロジック（§4.4 applyResult 相当）は不変、適用単位が「1 ゲーム」→「1 ラウンド」に変わるだけ | レベルがセッション内でも実力に追従して動く |
| **C2** | **永続の単位** | currentLevel / consecutiveFailures は永続 | 同左に加え、**連続失敗カウントはラウンド跨ぎ・セッション跨ぎ・再起動跨ぎで永続**。次ラウンドは更新後レベルで生成 | セッション制でも昇降状態が連続する |
| **D** | **設定値（振れ幅）の選択肢** | §4.1 の固定値集合のみ | 各変数の値集合を拡張（易→難順維持）。**追加値はデフォルト OFF**、既定の有効集合（`defaultVariableRanges`）は従来の値のみ。選択でレベル総数が増える | より易しい／より難しい域をテスト・調整できるようにする |
| **D2** | **個数×サイズの制約** | 暗黙（個数 1〜4、サイズ 3x3/4x4 で過密化せず） | 個数は格子セル数を超えられない。生成時に有効個数を格子容量でクランプ（具体規則は Generator 余地）。§4.6 / NF に明記 | 個数 5/6・サイズ 3x3 等の組み合わせで過密・破綻しないようにする |
| **E** | **結果表示・記録の粒度** | ゲーム終了ごとにホームへ結果。GameRecord は 1 ゲーム単位 | セッション末に**セッション要約**をホーム表示。記録は**セッション単位の `SessionRecord`**（ラウンド詳細は永続化しない）。累計プレイ回数は**セッション単位**で数える | セッション制に合わせて記録粒度を整理し、データ肥大を避ける |
| **E2** | **中断時の扱い** | 中断ゲームはレベル・記録に一切影響しない | **進行中の未完ラウンドは破棄**（記録・レベルに反映しない）。一方、**それ以前に完了済みのラウンドで起きたレベル変化は正当なプレイとして永続保持**（巻き戻さない） | セッション中の中断で、すでに確定したラウンド結果を不当に巻き戻さない |

> v3.0 → v3.1 で**不変**：レベルの 5 変数構成・mixed-radix オドメータ・クリア/失敗の 2 値判定ロジック・レベル昇降ルール（+1 / 2 連続失敗 −1 / クランプ）・バッジ 3 軸・名前空間 `gaboreye:v3:*`・横断規範（a11y / 老眼配慮 / ガボール描画品質 / 音・ハプティクス）。

### 0.2 v2.0 → v3.0 改訂履歴（本質的変更点）

v3.0 は「スコア管理 → レベル管理」への**作り直し（リブート）** である。ゲーム刺激の核（n×n ガボール格子の変化検出）は維持しつつ、進捗の管理単位・難易度の決定方法・採点モデルを根本から差し替える。

| 改訂対象 | 旧（v2.0） | 新（v3.0） | 理由 |
|---|---|---|---|
| **進捗の管理単位** | 日次スコア（正答率ベース 0〜100） | **レベル（その人が正解できる難易度）**。最高到達レベル・日次到達レベルで進捗管理 | 「自分が今どこまでできるか」をレベルという 1 つの整数で直感的に把握させる |
| **難易度の決定** | n/m/r/a/b を設定タブで手動固定 | **レベルが 5 変数の mixed-radix 組み合わせを自動決定**（個数/時間/回転方向/サイズ/回転速度）。手動スライダー廃止 | 難易度を「レベル」に一元化。ユーザーは個別変数を意識しない |
| **難易度の上下** | 手動。staircase は廃止のまま | **クリアで +1、2 回連続失敗で −1**（オドメータ式の自動昇降） | レベル制の核。実力に追従して難易度が自動調整される |
| **1 単位の構成** | 1 ラウンド m 秒 × r ラウンド = 1 セッション | **1 ゲーム = 1 ラウンド = 1 レベル挑戦**。r ラウンドは廃止 | レベル挑戦を 1 回単位に単純化 |
| **採点モデル** | TP+1/FP−1/FN 0 の部分点 → 0〜100 セッションスコア | **クリア / 失敗の 2 値判定**（全問正解=クリア、誤答・取りこぼし・時間切れ=失敗）。部分点・0〜100 スコアは全廃 | レベル昇降の入力は「クリアできたか」のみで十分 |
| **採点方式の切替** | scoringMode ①②③（自動採点・確定・全問正解遷移）を設定で切替 | **廃止**。クリア判定 1 本に統一 | レベル制では締め方の差異が不要 |
| **変化の種類** | 回転 + 空間周波数変化が同時混在 | **回転のみ**（一方向 / 振動＝往復回転）。空間周波数変化（b）は廃止 | レベル 5 変数に空間周波数が含まれず、b は死にパラメータ化したため |
| **回転方向** | 時計/反時計をパッチ独立ランダム（常に一方向） | **一方向 / 振動（往復回転）** をレベルで決定。振動=難 | 難易度軸の 1 つとして方向を追加 |
| **個数の扱い** | ラウンドごとランダム、UI 非表示 | **レベルで確定し、画面に明示**（例「3 個探せ！」） | レベルが個数を決めるため、隠す必然性がなくなった |
| **設定タブ** | n/m/r/a/b の手動スライダー + 採点方式 | **5 変数の振れ幅（範囲）設定 + 変化順設定**（テスト用）。手動スライダー・採点方式・b は削除 | レベル制に合わせ、設定は「レベルの定義域」を調整するものへ |
| **履歴グラフ** | 日次スコア（正答率 0〜100）の折れ線 | **日次到達レベル / 最高到達レベル**の推移。連続日数・累計プレイ回数は維持 | スコアからレベルへ指標を読み替え |
| **バッジ** | 継続日数 / 高難度クリア / 高スコア（正答率） | **継続日数 / 高難度到達 / 高レベル到達**。旧スコア依存条件は無効化、要再定義 | 進捗単位がレベルに変わったため |
| **データ** | `gaboreye:v2:*`（スコア管理スキーマ） | **`gaboreye:v3:*` で再設計**（現在レベル / 連続失敗カウント / レベル履歴 / 範囲設定 / 変化順）。起動時に旧データ扱いを定義（F-11） | スキーマ非互換のため |
| **継承する横断規範** | OPT-1〜12、F-19 音/ハプティクス、クリッピング品質、セーフエリア、a11y | **継承**（採点フィードバックは「クリア/失敗」に読み替え） | 老眼配慮と品質基準は引き続き必須 |
| **アプリ名・アイコン** | GaborEye 維持・アイコン現状維持 | **維持**（差し替えはスコープ外） | 今回は進捗モデルのリブートに集中 |

## 1. 想定（Assumptions）

`docs/questions-v3.md` の確定回答（2026-06-09）と v3.1 変更要望（2026-06-10 確認済み）を AS として明示する。明示回答のない事項は合理的仮定とし、その旨を併記する。v2.0 から不変の横断的前提も再掲する。

### 1.0 v3.1 確定事項由来の前提（2026-06-10）
- **AS-22（セッション制）**：1 回のプレイは「セッション」単位。**1 セッション = `sessionMinutes`×60 秒を超えるまでラウンドを反復**する。各ラウンド完了時に累積経過時間を確認し、`sessionMinutes`×60 秒**未満**なら次ラウンドを開始する。最後のラウンドが指定時間を超えてもそのラウンドは完走する（途中で打ち切らない）。1 ラウンドの長さ = 現在レベルの「時間」変数（seconds）。
- **AS-23（sessionMinutes 設定）**：`sessionMinutes` は 1〜15 分・1 分刻み・**初期値 5 分**。設定タブに置く全プレイ共通の設定であり、**レベル変数ではない**（レベルの梯子・総レベル数に影響しない）。
- **AS-24（締切時判定・即時終了廃止）**：全問正解でも即時終了しない。各ラウンドは「時間」秒だけ続き、**締切（時間到達）時点でクリア/失敗を判定**する。クリア=回転パッチを過不足なく選択、失敗=誤選択（FP>0）・選び逃し（FN>0）。「全問正解で即時遷移」は廃止（§11 スコープ外）。
- **AS-25（3 秒開示カウントダウン）**：締切後、各パッチに ✅/❌、刺激領域直下に総合クリア/失敗を**ゲーム画面内**で開示する。開示中に **3 秒のカウントダウン**（F-12 準拠、白→黄→赤）を表示し、3 秒後に**自動で次ラウンド**へ進む。3 秒は確定値。
- **AS-26（ラウンド単位の昇降）**：レベル昇降は**各ラウンドのクリア/失敗で即適用**する。昇降ロジック自体（§4.4）は v3.0 から不変で、適用単位が「1 ゲーム」→「1 ラウンド」に変わるだけ。連続失敗カウント・currentLevel はラウンド跨ぎ・セッション跨ぎ・再起動跨ぎで永続。次ラウンドは更新後レベルで生成する。
- **AS-27（設定値拡張・既定 OFF）**：§4.1 の値集合を拡張する（易→難順を維持）。追加値は**デフォルト未選択（OFF）**であり、既定の有効集合 `defaultVariableRanges` は従来の値のみとする。追加値を ON にするとレベル総数が増える。
- **AS-28（個数×サイズのクランプ）**：レベル定義上は個数とサイズは独立だが、生成時に有効個数を格子容量でクランプする（個数 < gridSize² となるよう、例 min(count, セル数−1)）。具体クランプ規則は Generator 余地（§4.6・NF-28d）。
- **AS-29（セッション要約・記録粒度）**：セッション末にホームへセッション要約を表示する（ラウンド数 / クリア数・失敗数 / 現在レベル / 今セッション最高到達レベル / 今日のストリーク）。記録は**セッション単位の `SessionRecord`**（ラウンド詳細は永続化しない）。累計プレイ回数は**セッション単位**で数える（「もう一度」で +1）。Streak は日次（セッション完了日）。LevelState はラウンドごとに連続更新・永続、DailyStats.highestLevelReached は「その日にクリアした最高レベル」を維持（§7 で再凍結）。
- **AS-30（中断時の扱い）**：中断時、**進行中の未完ラウンドは破棄**（記録・レベルに反映しない）。一方、**それ以前に完了済みのラウンドで起きたレベル変化は正当なプレイとして永続保持**（巻き戻さない）。中断したセッションは `SessionRecord` として記録するか否かを §7 / F-07 に定義する。

### 1.1 確定事項由来の前提（questions-v3.md Q1〜Q4、v3.0 ベース）

### 1.1 確定事項由来の前提（questions-v3.md Q1〜Q4）
- **AS-1（レベル = 難易度）**：履歴はスコアではなくレベルで管理する。レベルはその人が正解できる難易度を表す整数（Q 前文）。
- **AS-2（5 変数）**：レベルは次の 5 変数の組み合わせで決まる（易 → 難）：
  - 回転速度：6, 5.5, 5, 4.5, 4, 3.5, 3, 2.5, 2（deg/sec、9 値。速い=易 / 遅い=難）
  - 個数：1, 2, 3, 4（4 値）
  - 時間：40, 35, 30, 25, 20（秒、5 値）
  - 回転方向：一方向, 振動（2 値。振動 = パッチが回転を往復する＝難）
  - サイズ：3x3, 4x4（2 値）
- **AS-3（デフォルト変化順とオドメータ）**：デフォルトの変化順（最内側 LSB → 最外側 MSB）は **個数 → 時間 → 回転方向 → サイズ → 回転速度**。レベルが 1 上がると最内側変数が 1 段難化し、最内側が最難値に達したら次の変数を 1 段難化して最内側をリセットする（mixed-radix オドメータ）。組合せ総数 = 4×5×2×2×9 = **720 レベル**。L1 = 全変数最易（個数1, 時間40, 一方向, 3x3, 速度6）（Q1）。
- **AS-4（例は説明用）**：ユーザー提示の「例」（L2=時間35 等）はルール文と食い違うため**説明用**扱い。実際のデフォルト梯子は個数最内側に従う（Q1）。
- **AS-5（クリア/失敗判定）**：制限時間（ラウンド締切）時点で回転パッチを**過不足なく全選択 = クリア**。誤選択（FP>0）・選び逃し（FN>0）= 失敗。採点 3 方式（scoringMode ①②③）、TP/FP/FN 部分点、0〜100 セッションスコアは**全廃**。クリア/失敗の 2 値に統一（Q2）。**v3.1 で全問正解での即時終了は廃止**し、締切時判定に統一（AS-24）。
- **AS-6（ラウンドとセッション）**：**v3.0 の「1 ゲーム = 1 ラウンド = 1 レベル挑戦」を v3.1 でセッション制に拡張**（AS-22）。1 ラウンド = 1 レベル挑戦は維持しつつ、1 セッション = 指定時間まで複数ラウンドを反復する。複数ラウンド設定（roundCount = r、固定回数）は引き続き廃止。ラウンドごとのクリア/失敗が即レベル増減に反映（Q3・AS-26）。
- **AS-7（レベル増減）**：クリア → 次ラウンドで +1（v3.1。v3.0 では「次ゲーム」）。**2 回連続失敗 → −1**。連続失敗カウントは永続（端末ローカルに保持）（Q2/Q3、前文・AS-26）。
- **AS-8（個数明示）**：ゲーム画面に現在のレベルを表示し、探すべき回転パッチの個数を明示する（例「3 個探せ！」）。v2.0 の「個数を伏せる」挙動は廃止（前文）。
- **AS-9（設定の整理）**：レベルで自動決定される 5 変数の手動スライダーは削除。代わりに各変数の取りうる範囲（振れ幅）をテスト用に設定可能化し、変化順を設定画面で組み替え可能化する。残す既存設定：ダークモード / 音 ON-OFF / 振動 ON-OFF / 片眼ガイダンス / 視聴距離 / 免責事項再閲覧 / 全データ削除。削除する既存設定 UI：sfChangeSpeed(b) / scoringMode / roundCount(r) / gridSize(n) / roundSeconds(m) / rotationSpeed(a) の手動設定（Q4）。

### 1.2 合理的仮定（questions-v3.md に明示なし）
- **AS-10（静止パッチ）**：変化（回転）しないパッチは**ランダムな初期角度で静止固定**。これにより「回転中か静止か」の弁別が課題となる（v2.0 AS-12 を回転のみに読み替え）。
- **AS-11（振動の挙動）**：「振動」回転は、パッチが一定振幅で時計回り↔反時計回りを往復する回転。一方向回転より変化方向の手がかりが乱れるため難しい。具体的な振幅・周期は Designer/Generator が体感調整（AS-21）。
- **AS-12（回転速度の意味）**：回転速度が遅い（数値が小さい）ほど単位時間あたりの角度変化が小さく、「回転中か静止か」の弁別が難しくなる。ゆえに速度 6=最易、速度 2=最難。
- **AS-13（旧データ扱い）**：起動時、v2 以前の名前空間（`gaboreye:v1:*` / `gaboreye:v1.1:*` / `gaboreye:v1.2:*` / `gaboreye:v2:*`）のデータは v3 のレベル管理スキーマと非互換のため、**全リセット**し新名前空間 `gaboreye:v3:*` で初期化する。レベルへの変換移行は行わない（v2 にレベル情報が存在しないため）。F-11 参照。
- **AS-14（ラウンド締切後の開示）**：締切判定後（クリア/失敗確定後）、刺激画面にとどまったまま、回転していたパッチ上に ✅、ユーザーが誤って選んだ静止パッチ上に ❌ を重畳し、刺激領域直下に総合 クリア/失敗 を 1 個表示する。**v3.1：開示中に 3 秒のカウントダウン（F-12 準拠）を表示し、3 秒後に自動で次ラウンドへ進む（AS-25）。** v2.0 AS-16 をクリア/失敗判定に読み替え。
- **AS-15（初期レベル）**：初回起動時または全データ削除後の現在レベルは **L1**（全変数最易）から開始する。
- **AS-16（範囲外レベルのクランプ）**：レベルは設定された有効レベル範囲（変化順・各変数範囲から導かれる全レベル数）でクランプされる。最高レベルでクリアしてもそれ以上は上がらず、L1 で失敗してもそれ以下には下がらない。

### 1.3 横断的前提（v2.0 から継承）
- **AS-17**：本アプリは医療機器ではない。医療効果は断定せず、薬機法を意識した文言で運用する。
- **AS-18**：ターゲットプラットフォームは iOS / Android（Expo Go / Web）一級サポート。Web ブラウザは最小幅 360px〜最大 1280px でレスポンシブ・キーボード操作・スクリーンリーダーを担保。
- **AS-19**：日本語ロケールのみ。日付・週は ISO 週（月曜開始）。i18n キー基盤は維持。
- **AS-20**：完全オフライン動作。ネットワーク通信なし。年齢申告は 40 代/50 代/60 代/70 代以上（任意）。70 代以上選択時は警告と医師相談推奨。
- **AS-21**：レベル定義の 5 変数の**値集合は本書で確定**（AS-2、上記の通り）。一方で、振動の振幅・周期、静止パッチの初期角度ばらつき、回転の最小可知差など**体感に関わる具体値は spec で断定せず**、Designer/Generator が表示サイズ・cpd・端末性能から仮置きし、ユーザーが実機で試して確定する。各変数の範囲設定の初期値（=フル範囲か絞るか）も Designer/Generator 仮置き。

## 2. ビジョン / 課題 / ターゲット

### 2.1 ビジョン
**GaborEye v3.0** は、ガボールパッチの回転変化検出ゲームを土台に、**「自分のレベル」を上げていく**ことを継続のモチベーションにするアプリ。1 ゲーム = 1 レベル挑戦。クリアすればレベルが 1 つ上がり、難しくなる。失敗が続けば自動で下がる。老眼層が「ゆっくり回るものをじっと見て見つける」体験を、自分の到達レベルという 1 つの分かりやすい指標とともに、自助セルフケアとして継続する。

医療効果は断定せず、「自助セルフケア」として位置づける。

### 2.2 課題
- v2.0 の「0〜100 スコア」は、正答率ベースで毎セッション上下するため「自分が成長したか」が直感的に伝わりにくかった。
- 難易度を n/m/r/a/b の手動設定に委ねていたため、ユーザーは「自分に合った難しさ」を自分で探す必要があり、迷いが生じた。
- v3.0 は **レベル**という単調増加しうる単一指標で「今どこまでできるか」を可視化し、難易度調整を自動化することで、これらを解消する。

### 2.3 ターゲットユーザー
v2.0 から不変。

- **メインターゲット**：老眼層（40 代以上）。手元の文字が見えづらい等の主観的不満を持つが医療機関には行くほどでない自助セルフケア指向のユーザー。「じっと一枚絵を見つめる時間を確保したい人」を意識する。
- **対象外**：子ども（小学生以下）/ 70 代以上（任意申告で警告）/ 重度視覚障害者 / 強度近視（-6D 以上）/ 目に違和感のあるユーザー。

## 3. ユーザーストーリー / ユースケース

- **US-1（初回起動）**：初めて起動したユーザーとして、簡素なオンボーディングで免責に同意し視聴距離を設定したうえで、すぐに L1 を体験したい。何のゲームかを 90 秒以内に理解できること。
- **US-2（日常プレイ）**：継続中のユーザーとして、アプリを開いたら距離リマインドの後すぐホームタブで現在のレベルのラウンドが始まり、設定した時間（例 5 分）が経つまでラウンドを続けて回転しているパッチをタップしていきたい。今いくつ探せばよいかを画面で知りたい。
- **US-3（レベルアップ）**：ラウンドをクリアしたユーザーとして、次のラウンドでレベルが 1 つ上がり、少し難しくなる手応えを得たい。
- **US-4（レベルダウン）**：難しすぎて 2 回連続で失敗したユーザーとして、自動でレベルが 1 つ下がり、無理なく続けられるようにしてほしい。
- **US-5（中断）**：プレイ中に用事ができたユーザーとして、× ボタンまたは他タブを押したときに確認ダイアログで中断でき、進行中の未完ラウンドは記録・レベルに影響しないでほしい（それまでに完了したラウンドの結果は残ってよい）。
- **US-6（進捗確認）**：続けているユーザーとして、履歴タブで日次到達レベル・最高到達レベルの推移、連続日数、累計プレイ回数（セッション数）とバッジを見て達成感を得たい。
- **US-9（セッション要約）**：セッションを終えたユーザーとして、今回のラウンド数・クリア数・失敗数・現在レベル・今セッションの最高到達レベル・今日のストリークを見て、「もう一度」で新しいセッションを始めたい。
- **US-10（プレイ時間の選択）**：ユーザーとして、1 回のプレイ時間（1〜15 分）を設定で選び、自分の都合に合わせて続けたい。
- **US-7（テスト調整）**：開発者/テスターとして、設定タブで各変数の範囲と変化順を組み替え、レベルの梯子の作り方を試したい。
- **US-8（個別配慮）**：片眼トレーニングをしたいユーザーとして、設定で片眼ガイダンスを有効にしたい。音・振動も個別に切り替えたい。

## 4. レベルシステム・セッション詳細（v3.x の核）

> 本章はレベル制の振る舞いを定義する。各機能（F-01 等）の受け入れ基準はここを参照する。値集合は AS-2 で確定済み。

### 4.1 5 変数とその値集合（易 → 難、v3.1 拡張・既定 ON / 追加 OFF）

v3.1 で各変数の**全集合**を拡張する（易→難順を維持）。**既定の有効集合（`defaultVariableRanges`）は v3.0 と同じ値のみ**で、追加値はデフォルト OFF。設定タブ（F-13）で追加値を ON にすると、その変数の段数が増え、総レベル数が増える（AS-27）。

| 変数 | キー | 全集合（易 → 難） | 既定 ON（v3.0 と同一） | v3.1 追加（既定 OFF） | 全段数 / 既定段数 |
|---|---|---|---|---|---|
| 個数 | count | 1, 2, 3, 4, 5, 6 | 1, 2, 3, 4 | 5, 6 | 6 / 4 |
| 時間 | seconds | 60, 55, 50, 45, 40, 35, 30, 25, 20, 15, 10 | 40, 35, 30, 25, 20 | 60, 55, 50, 45（長い=易）/ 15, 10（短い=難） | 11 / 5 |
| 回転方向 | direction | 一方向, 振動 | 一方向, 振動 | （追加なし） | 2 / 2 |
| サイズ | gridSize | 3x3, 4x4, 5x5, 6x6 | 3x3, 4x4 | 5x5, 6x6 | 4 / 2 |
| 回転速度 | rotationSpeed | 7, 6.5, 6, 5.5, 5, 4.5, 4, 3.5, 3, 2.5, 2, 1.5, 1 | 6, 5.5, 5, 4.5, 4, 3.5, 3, 2.5, 2 | 7, 6.5（速い=易）/ 1.5, 1（遅い=難） | 13 / 9 |

- **既定（`defaultVariableRanges`）の総レベル数** = 4×5×2×2×9 = **720**（v3.0 と不変）。
- **全集合をすべて ON にした場合の総レベル数** = 6×11×2×4×13 = **6864**（理論上限）。実際の各レベルの中身は変化順（§4.2）と有効集合に依存する。
- **「時間」は両側に拡張**：60〜45 秒（v3.0 より長い＝易）と 15〜10 秒（v3.0 より短い＝難）。易→難の連続性を保つため、全集合は 60（最易）→10（最難）の降順で並ぶ。
- **「回転速度」も両側に拡張**：7, 6.5（v3.0 より速い＝易）と 1.5, 1（v3.0 より遅い＝難）。全集合は 7（最易）→1（最難）の降順。
- **「個数」「サイズ」は難側に拡張**：個数 5/6、サイズ 5x5/6x6 を追加。個数とサイズの組み合わせ制約は §4.6 を参照。

### 4.2 変化順とオドメータ（mixed-radix）
- **変化順**は最内側（LSB、最も頻繁に変化）から最外側（MSB）への並び。デフォルトは **個数 → 時間 → 回転方向 → サイズ → 回転速度**（AS-3）。
- レベル番号 L（1 始まり）から 5 変数の組を決める規約：
  - L が 1 上がると、最内側変数が 1 段難化する。
  - 最内側変数がその最難値に達した状態でさらに 1 上がると、最内側を最易にリセットし、次（1 つ外側）の変数を 1 段難化する。以下、桁上がりが連鎖する（オドメータ）。
- **総レベル数**は各変数の**有効段数**の積。既定（`defaultVariableRanges`）では 4×5×2×2×9 = **720**。全集合を ON にした理論上限は 6×11×2×4×13 = 6864（§4.1）。
- **デフォルト梯子の例**（説明用、AS-3。既定有効集合 = 720 レベル）：
  - L1 = (個数1, 時40, 一方向, 3x3, 速6)
  - L2 = (個数2, 時40, 一方向, 3x3, 速6)
  - L3 = (個数3, …) / L4 = (個数4, …)
  - L5 = (個数1, 時35, …) … L20 = (個数4, 時20, …)
  - L21 = (個数1, 時40, 振動, 3x3, 速6) …
  - L720 = (個数4, 時20, 振動, 4x4, 速2)（全変数最難）

### 4.3 クリア / 失敗判定（AS-5・AS-24、v3.1 改訂：締切時判定）
- **締め切りの契機**：そのレベルの「時間」変数値（seconds）が経過した時点（**ラウンド締切**）の 1 本のみ。**v3.1 で全問正解での即時締め切りは廃止**（§11 スコープ外）。各ラウンドは必ず「時間」秒だけ続く。確定ボタン・部分点は存在しない。
- **クリア**：ラウンド締切時点で、回転しているパッチを**過不足なく全選択**している（FP=0 かつ FN=0）。
- **失敗**：ラウンド締切時点で、誤選択（静止パッチを選んでいる、FP>0）または選び逃し（回転パッチを選んでいない、FN>0）がある。
- 締切判定後の開示・3 秒カウントダウン・次ラウンド自動遷移は §4.6（B2）/ F-03 を参照。

### 4.4 レベル増減ルール（AS-7, AS-16, AS-26, AS-30）

> **v3.1：昇降ロジック（applyResult 相当）は v3.0 から不変。適用単位が「1 ゲーム」→「1 ラウンド」に変わるだけ。** 各ラウンドの締切判定（§4.3）ごとに以下を即適用する。

- ラウンド結果が**クリア** → 連続失敗カウントを 0 にリセットし、次ラウンドのレベルを +1（上限でクランプ）。
- ラウンド結果が**失敗** → 連続失敗カウントを +1。
  - 連続失敗カウントが **2 に達したら** レベルを −1（下限 L1 でクランプ）し、連続失敗カウントを 0 にリセットする。
  - 連続失敗 1 回目ではレベルは変わらない。
- 連続失敗カウントは**永続**（**ラウンド跨ぎ・セッション跨ぎ・アプリ再起動・日跨ぎでも保持**）。クリアおよびレベルダウン発火でリセット。
- currentLevel も永続。**次ラウンドは更新後のレベルで生成**する（セッション内でレベルが連続的に上下する）。
- **中断時の扱い（AS-30）**：
  - **進行中の未完ラウンド**（締切前に中断したラウンド）は、クリアでも失敗でもなく、レベル・連続失敗カウント・記録のいずれにも反映しない（破棄）。
  - 一方、**それ以前に完了済み（締切判定済み）のラウンド**で起きたレベル変化・連続失敗カウント変化は、**正当なプレイとして永続保持**する（中断を理由に巻き戻さない）。
- レベルは有効レベル範囲 [1, 総レベル数] でクランプ（AS-16）。

### 4.5 範囲設定との関係（F-13）
- 設定で各変数の有効値部分集合（振れ幅）と変化順を変えると、**総レベル数と各レベルの中身が変わる**。
- 範囲・変化順を変更したときの現在レベルの扱い：新しい有効レベル範囲に収まるようクランプする（範囲が縮んで現在レベルが新上限を超える場合は新上限に丸める）。連続失敗カウントは 0 にリセットする（梯子が変わったため）。

### 4.6 セッションループ・ラウンド反復・3 秒開示（v3.1 新設、AS-22 / AS-24 / AS-25）
- **セッション = ラウンドの反復**：1 セッションは以下を繰り返す。
  1. 現在レベルからラウンド（n×n 格子・回転パッチ・「時間」秒の制限時間）を生成。
  2. ユーザーがタップ選択。「時間」秒経過で**締切**、クリア/失敗を判定（§4.3）。
  3. レベル昇降を即適用（§4.4）。
  4. 締切後、ゲーム画面内で ✅/❌・総合クリア/失敗を開示し、**3 秒のカウントダウン**（F-12 準拠、白→黄→赤）を表示。
  5. 3 秒後、**累積経過時間**（セッション開始からの合計）を確認：`sessionMinutes`×60 秒**未満**なら更新後レベルで次ラウンドを開始（1 に戻る）。**到達または超過していればセッション終了**。
- **最後のラウンド**：あるラウンドの途中で累積時間が `sessionMinutes`×60 秒を超えても、そのラウンドは打ち切らず完走する。完走後に累積時間が指定時間以上ならセッション終了。
- **累積経過時間の定義**：各ラウンドの実プレイ時間（「時間」秒）の合計とする。3 秒の開示カウントダウン分を累積に含めるか否かは Generator 余地（体感に影響しないため断定しない）。判定の基準点は「各ラウンド完了時」で固定。
- **3 秒は確定値**：開示→次ラウンドの間隔は 3 秒で固定（Designer/Generator が変更しない）。カウントダウンの色変化（白→黄→赤）と太字補強は F-12 に従う。
- **セッション末**：セッション終了後、ホームタブにセッション要約（F-08）を表示する。

### 4.7 個数 × サイズのクランプ（v3.1 新設、AS-28）
- レベル定義上、個数（count）とサイズ（gridSize）は独立変数だが、**個数は格子のセル数（gridSize²）を超えてはならない**。さらに静止パッチが最低 1 つ存在するよう、**有効個数 < gridSize²** とする。
- 生成時、レベルが決めた count が格子容量を超える場合、有効個数を格子容量でクランプする（例 `min(count, gridSize² − 1)`）。具体クランプ規則（過密回避のための上限の決め方）は Generator 余地。
- 例：個数 6 × 3x3（9 セル）は許容（6 < 9）だが過密。個数 6 × サイズ次第ではクランプが働く。クランプはレベル番号 ⇄ 5 変数の梯子定義そのものは変えず、**ラウンド生成時にのみ**適用する（梯子の総レベル数は変えない）。

## 5. 機能一覧（F-01〜F-14）

> 各機能の受け入れ基準は**ユーザーから観察できる挙動**で記述する。レベル 5 変数の値集合は §4 で確定。体感に関わる具体値は Designer/Generator 判断（AS-21）。

### F-01 回転変化検出ゲーム（コア・セッション制）
- **目的**：n×n のガボールパッチ格子の中で、ゆっくり回転しているパッチをすべて見つけてタップさせる。難易度は現在レベルが決定する。1 セッションは指定時間（`sessionMinutes`）までラウンドを反復する。
- **主要な振る舞い**：
  - 現在レベルが決める n×n（3x3〜6x6、有効サイズ集合に依存）のガボールパッチを格子状に表示する。
  - そのうち現在レベルが決める個数のパッチが**ゆっくり回転**する。回転は現在レベルが決める速度（deg/sec）・方向（一方向 / 振動）で行う。個数は格子容量でクランプされる（§4.7）。
  - 回転しないパッチはランダムな初期角度で静止固定（AS-10）。
  - ユーザーは回転していると思うパッチを**直接タップ**して選択（トグル）。
  - 制限時間は現在レベルが決める秒数。**制限時間が経過した時点（締切）でクリア/失敗を判定**する（全問正解での即時終了はしない、§4.3）。
  - 締切後、開示と 3 秒カウントダウンを経て、累積時間が指定時間未満なら次ラウンドへ自動遷移する（§4.6）。
- **受け入れ基準**：
  - [ ] 現在レベルに対応する n×n のガボールパッチ格子が表示される（有効サイズ集合の範囲内、既定は 3x3 / 4x4）
  - [ ] 現在レベルに対応する個数のパッチが回転し、残りは静止する（個数は格子容量でクランプされ、必ず静止パッチが 1 つ以上存在する）
  - [ ] 回転は現在レベルに対応する速度・方向（一方向 / 振動）で行われる。振動は時計回り↔反時計回りの往復として観察できる
  - [ ] 静止パッチはランダムな初期角度で静止している
  - [ ] ユーザーはパッチを直接タップして選択/選択解除でき、選択中のパッチが視覚的に明示される（控えめな枠など、ガボール本体の視認を阻害しない形）

### F-03 ラウンド締切判定・結果開示・3 秒カウントダウン
- **目的**：ラウンド締切後に、クリア / 失敗の結果と、回転していたパッチ・ユーザーの誤選択を刺激画面にとどまったまま開示し、3 秒のカウントダウン後に自動で次ラウンドへ進める（AS-5, AS-14, AS-24, AS-25）。

- **受け入れ基準**：

  - [ ] 結果開示中（3 秒カウントダウン中）はパッチのタップ操作が無効化される

### F-04 レベル昇降（ラウンドごと：クリアで +1 / 2 連続失敗で −1）
- **目的**：**各ラウンドの結果**に応じて現在レベルを自動で増減し、セッション内でも実力に追従させる（AS-7, AS-26, §4.4）。
- **主要な振る舞い**：ラウンドのクリアで連続失敗カウントを 0 にして +1、失敗で連続失敗カウント +1（2 到達で −1 しカウント 0）。クランプ・永続は §4.4 の通り。昇降ロジック自体は v3.0 から不変で、適用単位がラウンドに変わる。
- **受け入れ基準**：
  - [ ] **ラウンドをクリアすると、次のラウンドのレベルが 1 つ上がる（上限を超えない）**
  - [ ] 失敗 1 回目ではレベルは変わらない
  - [ ] 失敗が 2 回連続したとき（ラウンド跨ぎ・セッション跨ぎ含む）にレベルが 1 つ下がり（下限 L1 を下回らない）、連続失敗カウントがリセットされる
  - [ ] クリアすると連続失敗カウントがリセットされる（失敗→クリア→失敗では下がらない）
  - [ ] 連続失敗カウント・現在レベルは**ラウンド跨ぎ・セッション跨ぎ・アプリ再起動でも保持される**
  - [ ] **進行中の未完ラウンドを中断した場合は、そのラウンドはレベル・連続失敗カウントに影響しない**
  - [ ] **中断より前に完了済みのラウンドで起きたレベル変化は保持される（中断で巻き戻らない）**
  - [ ] 上限レベルでクリアしてもそれ以上は上がらず、L1 で失敗が続いても L1 を下回らない
  - [ ] レベル上昇/下降がユーザーに観察できる形（次ラウンドのレベル表示変化、または開示中の告知）で示される

  - [ ] **追加値を ON にすると、その変数の有効段数が増え、総レベル数が増える（梯子に反映される）**
  - [ ] 各変数は少なくとも 1 値を有効にする必要があり、全値を無効にはできない
  - [ ] 変化順（5 変数の最内側〜最外側）を組み替えられ、結果がレベルの梯子に反映される
  - [ ] 範囲・変化順を変更すると、現在レベルが新しい有効範囲にクランプされ、連続失敗カウントがリセットされる（§4.5）
  - [ ] レベル自動決定 5 変数の**手動スライダー（個別に値を固定する UI）は存在しない**
  - [ ] 旧設定 UI（sfChangeSpeed(b) / scoringMode / roundCount(r) / gridSize(n) / roundSeconds(m) / rotationSpeed(a) 手動設定）はアプリ内に一切存在しない
  - [ ] 視聴距離（30/40/50cm、デフォルト 40cm）を選択でき、即座に保存される
  - [ ] ダークモード（OS 連動/明/暗）を選択できる
  - [ ] 音 ON/OFF と振動 ON/OFF を個別のトグルで切り替えられ、ON/OFF が一目で判別できる
  - [ ] 片眼ガイダンス（off/左/右/交互）を設定できる
  - [ ] 「免責事項を読む」から免責を再閲覧できる
  - [ ] 「全データ削除」は 2 段階確認（タップ → 確認ダイアログ）で実行され、実行後は現在レベル L1・連続失敗カウント 0 にリセットされる
  - [ ] 各項目行は 56pt 以上の高さ
  - [ ] バージョン情報に `v3.1.x` と免責同意日時が表示される

### F-14 音・ハプティクスフィードバック（v2.0 F-14 継承・クリア/失敗に適合）
- **目的**：視覚以外のチャネル（聴覚・触覚）でクリア/失敗/重要イベントを補強する。
- **主要な振る舞い**：
  - **音**：(1) クリア時、(2) 失敗時、(3) カウントダウン残り 3・2・1 秒の毎秒ティック、(4) レベルアップ時、(5) バッジ獲得時。
  - **ハプティクス**：(1) クリア時、(2) 失敗時、(3) レベルアップ時、(4) バッジ獲得時。
  - 設定で音 ON/OFF と振動 ON/OFF を個別に切替。サイレントモード時は音を鳴らさずハプティクスのみ。
- **受け入れ基準**：
  - [ ] 各ラウンド締切時にクリア/失敗に応じた音が再生される（音 OFF 時は無音）
  - [ ] カウントダウン残り 3/2/1 秒の各時点でティック音が再生される（音 OFF 時は無音）
  - [ ] レベルアップ時に専用音が再生される（音 OFF 時は無音）
  - [ ] バッジ獲得時に獲得音が再生される（音 OFF 時は無音）
  - [ ] クリア/失敗/レベルアップ/バッジ獲得時にハプティクスが発火する（振動 OFF 時は無振動）
  - [ ] サイレントモード（OS 設定）のとき音は鳴らさず、ハプティクスは発火する（振動 OFF でない限り）
  - [ ] 試行中（締め切り前の注視中）は結果フィードバック以外の音/振動を発火しない

## 7. データモデル（v3.1、`gaboreye:v3:*` 名前空間）

**v3.1 で本データモデルを再凍結する。** 以降の変更にはユーザー再承認が必要。名前空間は v3.0 と同じ `gaboreye:v3:*`（v3.1 はスキーマ拡張であり名前空間は変えない）。型表記はスキーマの意図を示すもので、実装の型名・構造の細部は Generator 判断。

> **v3.1 のスキーマ差分（v3.0 比）**：
> - `Settings` に **`sessionMinutes`**（1〜15、既定 5）を追加。`variableRanges` の各要素が取りうる値域を §4.1 拡張後の全集合に拡大（既定値は v3.0 同一）。

### スプリント間の原則
- S2（レベル中核）と S4（ゲームコア）は実装規模が大きいため、必要ならデータ/ロジック/UI に分割して進める。
- 各スプリント完了時に `npm test` の件数が増えていること。
- マルチファイル変更後はフルテストを走らせてから完了宣言する。
- Web と Android（Expo Go）の両方で動作確認する（特にゲーム描画・振動回転）。

## 9. 非機能要件（v2.0 から継承・v3.0/v3.1 向け調整）

### 9.1 パフォーマンス
- **NF-1** 刺激描画 60fps を目標。回転（一方向/振動）アニメーションは 30fps 最低許容
- **NF-2** 画面遷移は 200ms 以内
- **NF-3** アプリ起動から距離リマインド表示まで 3 秒以内
- **NF-4** 中位スマホ（iPhone 12 / Pixel 5 相当）でフレームドロップを起こさない

### 9.2 オフライン
- **NF-5** ネットワーク通信を行わずに全機能が動作。**NF-6** 初回ダウンロード後オフラインで起動可能

### 9.3 アクセシビリティ
- **NF-7** 主要操作要素に aria-label / accessibility label を付与
- **NF-8** UI テキストとボタン背景のコントラスト比 7:1 以上
- **NF-9** Web 版でキーボード操作：Tab で全インタラクティブ要素にフォーカス、Enter/Space で起動。focus outline 3px 以上
- **NF-10** スクリーンリーダーで主要要素が読み上げられる。レベル・個数・クリア/失敗が読み上げられる
- **NF-11** 点滅エフェクトは使用しない（てんかん配慮）。カウントダウンの色変化補強は太字化のみ
- **NF-12** 色のみで情報を伝えない（形・テキストでも判別可能）。タブ選択状態・カウントダウン色変化・クリア/失敗を補強する
- **NF-13** prefers-reduced-motion 対応：UI 装飾アニメーションを抑制（回転検出ゲーム本体は課題上アニメーションが必須のため、ゲーム刺激そのものは抑制対象外）
- **NF-14** Skip link（Web 版）
- **NF-15** 選択状態は aria-checked / aria-pressed で明示

### 9.4 老眼配慮（OPT-1〜12 継承・v3 調整）
- **NF-16** 本文最小フォント 18pt。**NF-17** タップ領域 48pt 以上、推奨 56pt。**NF-18** ボタン自動消失なし
- **OPT-11/12 改訂（v3.1）**：採点 3 方式は廃止。締め方は「**制限時間到達での締切**」の 1 本（全問正解での即時締め切りは廃止、AS-24）。共通項は「制限時間のラウンド・直接タップ選択・試行中は正誤フィードバックなし・締切後 ✅/❌ 開示＋3 秒カウントダウン・レベル/個数の明示・指定時間までラウンド反復（セッション制）」

### 9.5 プラットフォーム
- **NF-19** iOS 15 以上 / Android 8（API 26）以上。**NF-20** Web は最新 2 メジャーの Safari/Chrome/Edge/Firefox
- **NF-21** スマホ縦持ち固定、PC 横画面（最小 360px、最大 1280px）。**NF-22** タブレットは横レイアウトで PC と同等

### 9.6 セキュリティ
- **NF-23** ローカルストレージは端末標準の暗号化に従う。**NF-24** リリースビルドでログ出力しない

### 9.7 国際化
- **NF-25** 日本語のみ。i18n キー基盤は維持

### 9.8 ガボール描画品質（継承）
- **NF-26** 角度の最小可動単位は Designer/Generator が表示サイズ・cpd・端末性能から導出（具体値は spec に書かない）
- **NF-27** パッチの枠（矩形/円形マスク）と縞模様の間に背景色が見えてはならない
- **NF-28** 角度を最小単位ずつ変化させても、パッチ枠と縞模様の間に隙間や欠けが生じてはならない
- **NF-28b 回転/静止の弁別性**：静止パッチの初期角度のばらつきと、回転パッチの単位時間あたり角度変化量を、ユーザーが「回転中」と「静止」を区別できるよう調整する。各回転速度値（6〜2）でこの弁別が成立すること（最小可知差の具体値は Designer/Generator が体感調整）
- **NF-28c 振動の視認性**：振動（往復回転）が一方向回転と区別して観察でき、かつ静止とも区別できること（振幅・周期は Designer/Generator が体感調整）
- **NF-28d 個数 × サイズの過密回避（v3.1）**：有効個数は格子セル数（gridSize²）未満になるよう生成時にクランプし、静止パッチが必ず 1 つ以上残ること。拡張サイズ（5x5/6x6）でもパッチが視認可能なサイズで描画されること（過密でガボール本体の弁別が破綻しない。具体クランプ規則は Generator 余地、§4.7）

### 9.9 ステータスバー回避
- **NF-29** ゲーム刺激提示中はフルスクリーン許容（OS ステータスバー領域も使用してよい）
- **NF-30** それ以外の画面（距離リマインド・ホーム結果・履歴・設定・ダイアログ）はセーフエリア準拠

### 9.10 音・ハプティクス品質
- **NF-31** 音再生レイテンシ 100ms 以内。**NF-32** ハプティクス強度は端末標準をデフォルト。**NF-33** サイレントモード尊重（音は鳴らさずハプティクスのみ）

## 11. スコープ外（v3.x では実装しない）

- **スコア管理（0〜100 / 部分点 TP/FP/FN）**：全廃。レベル管理に一本化（AS-5）
- **採点 3 方式（scoringMode ①②③）**：廃止。クリア判定 1 本（AS-5）
- **全問正解での即時ラウンド終了**：**v3.1 で廃止**。各ラウンドは必ず制限時間まで続き、締切時に判定する（AS-24・§4.3）
- **固定回数の複数ラウンド設定（roundCount = r）**：廃止。ラウンド反復は固定回数ではなく**指定時間（`sessionMinutes`）まで**反復するセッション制で行う（AS-6・AS-22）
- **空間周波数変化（sfChangeSpeed = b）**：廃止。変化軸は回転のみ
- **5 変数の手動スライダー固定**：廃止。難易度はレベルが決定（AS-9）
- **v1〜v2 データの後方互換・レベルへの変換移行**：行わない。全リセット（AS-13 / F-11）
- **漸進難化モード / staircase の別建てモード**：レベル制自体が自動昇降を担うため、別モードは作らない
- **変化種類の区別回答**：回転のみのため該当 UI なし
- **アプリアイコン差し替え**：現状維持
- **眼精疲労クールダウン画面**：引き続き廃止
- **マルチプレイヤー / 対戦 / フレンド / ランキング / SNS シェア**
- **医療機関連携 / 医師レビュー / HealthKit・Google Fit 連携**
- **Eye Tracking / ARKit・ARCore**
- **クラウド同期 / 複数端末ログイン / アカウント・認証・課金**
- **多言語（英語含む）**：将来対応可能な i18n キー構造のみ維持
- **医療・薬機法的な「視力◯◯回復」表記**
- **BGM / キャラクター演出**
